## 2. cell_infos 解析规则

### 结构
```json
{
  "1": {
    "timeStamp": 1436089616273,
    "isConnected": 1,
    "type": "lte",
    "cell_identity": { "Ci": 5066627, "Tac": 6766, "mno": "46001", ... },
    "signal_strength": { "rsrp": -95, "rsrq": -10, "rssi": -65, "Dbm": -95, ... }
  },
  "2": { ... }
}
```

### 提取字段映射

| JSON 路径 | Layer0 列名 | 提取规则 | 说明 |
|-----------|------------|---------|------|
| key | cell_seq | JSON 对象的 key（"1","2",...） | 同一记录内的基站序号 |
| `.timeStamp` | cell_ts_raw | 原样 | 基站扫描时间戳 |
| `.isConnected` | is_connected | 1=当前连接, 2=邻区 | 连接状态 |
| `.type` | tech | lte→4G, nr→5G, gsm→2G, wcdma→3G, cdma→2G | 制式标准化 |
| `.cell_identity.Ci` 或 `.Nci`/`.nci`/`.cid` | cell_id_raw | COALESCE 取第一个非空 | Cell ID 原始值 |
| `.cell_identity.Tac` 或 `.tac`/`.lac`/`.Lac` | lac_raw | COALESCE 取第一个非空 | LAC/TAC 原始值 |
| `.cell_identity.mno` 或 `mccString+mncString` | operator_id_raw | 优先 mno，否则拼接 | 运营商标识 |
| `.cell_identity.Pci`/`.psc` | pci | 物理小区标识（LTE）/主扰码（WCDMA） | PCI |
| `.cell_identity.Earfcn`/`.earfcn`/`.ChannelNumber`/`.arfcn`/`.uarfcn` | freq_channel | 频点号 | 频率通道 |
| `.cell_identity.Bwth` | bandwidth | 带宽（kHz） | 仅 LTE 主服务区有 |
| `.signal_strength.rsrp` 或 `.SsRsrp` | sig_rsrp | LTE 用 rsrp，NR 用 SsRsrp | 参考信号接收功率 |
| `.signal_strength.rsrq` 或 `.SsRsrq` | sig_rsrq | LTE 用 rsrq，NR 用 SsRsrq | 参考信号接收质量 |
| `.signal_strength.rssnr` 或 `.SsSinr` | sig_sinr | LTE 用 rssnr，NR 用 SsSinr | 信噪比 |
| `.signal_strength.rssi` | sig_rssi | 仅 LTE（约 45% 有值） | 接收信号强度 |
| `.signal_strength.Dbm` | sig_dbm | 通用信号强度 | dBm |
| `.signal_strength.AsuLevel` | sig_asu_level | 0~99 正常 | ASU 等级 |
| `.signal_strength.Level` | sig_level | 0~4 正常 | 信号等级 |
| `.signal_strength.TimingAdvance` | timing_advance | 仅部分主服务区有 | 时间提前量 |
| `.signal_strength.CsiRsrp` | sig_csi_rsrp | 仅少量 NR 有（~3.5%） | CSI-RSRP |
| `.signal_strength.CsiRsrq` | sig_csi_rsrq | 仅少量 NR 有 | CSI-RSRQ |
| `.signal_strength.CsiSinr` | sig_csi_sinr | 仅少量 NR 有 | CSI-SINR |
| `.signal_strength.cqi` | sig_cqi | 仅少量 LTE 有（<0.5%） | 信道质量指示 |
| — | parsed_from | 固定 = 'cell_infos' | 解析来源标签 |

## 3. ss1 解析规则

### 格式
```
{信号块}+&{时间}&{GPS}&{基站块}+&{AP块};...
```

### 元素解析

| 位置 | 提取字段 | 解析方法 |
|------|---------|---------|
| 元素1（信号块） | tech, sig_ss, sig_rsrp, sig_rsrq, sig_sinr | 按 `+` 拆多条；每条 5 值：`{tech},{SS},{RSRP},{RSRQ},{RSSNR}`；**按制式匹配基站**；-1 视为无效置 NULL |
| 元素2（时间） | cell_ts_raw | unix 秒级时间戳 |
| 元素3（GPS） | ss1_lon, ss1_lat, ss1_gps_ts | 逗号分 3 值；可能为 `0` 表示无 GPS |
| 元素4（基站块） | tech, cell_id_raw, lac_raw, operator_id_raw | 按 `+` 拆多条；首字符 `l`/`n`/`g`；逗号分 4 值 |
| 元素5（AP） | — | WiFi bssid,ssid，当前阶段不提取 |
| — | parsed_from | 固定 = 'ss1' | 解析来源标签 |

### ss1 与 cell_infos 的关系（已更新）

> **不做合并。** cell_infos 和 ss1 各自独立拆行，全部保留。
> 同一条原始记录的 cell_infos 行和 ss1 行通过 `原始记录ID` 关联，但在 L0 层不合并。
> 数据补齐（GPS、信号、时间的互补）留到后续层处理。
>
> 详细 ss1 解析规则见 `docs/05_ss1解析规则.md`。
>
> **关键变更（V2）**：
> - 邻区行（is_connected=2）已去除，不进入 L0
> - ss1 信号按制式匹配基站（l信号→l基站，n信号→n基站）
> - is_connected 列已移除
> - match_status 字段已移除

## 6. Layer0 目标表结构

```sql
CREATE TABLE rebuild2.layer0_cell_records (
    -- 主键与来源
    id              bigserial   PRIMARY KEY,
    record_id       varchar     NOT NULL,       -- 原始记录标识
    source_table    varchar     NOT NULL,       -- 'gps' / 'lac'
    parsed_from     varchar     NOT NULL,       -- 'cell_infos' / 'ss1'
    match_status    varchar,                    -- CELL_INFOS_ONLY / MATCHED / SS1_ONLY / SS1_UNMATCHED
    cell_seq        int,                        -- 同一记录内的基站序号

    -- 时间
    ts_raw          varchar,                    -- 原始上报时间文本
    ts_std          timestamptz,                -- 标准化上报时间
    cell_ts_raw     varchar,                    -- 基站扫描时间原始值
    gps_ts_raw      varchar,                    -- GPS 上报时间

    -- 网络标识
    operator_id_raw varchar,                    -- PLMN 原始值
    operator_id_cn  varchar,                    -- 运营商中文
    tech            varchar,                    -- 原始制式
    tech_norm       varchar,                    -- 标准化制式 (2G/3G/4G/5G)
    lac_raw         varchar,                    -- LAC/TAC 原始文本
    lac_dec         bigint,                     -- LAC 十进制
    cell_id_raw     varchar,                    -- Cell ID 原始文本
    cell_id_dec     bigint,                     -- Cell ID 十进制
    bs_id           bigint,                     -- 基站 ID
    sector_id       bigint,                     -- 扇区 ID
    pci             int,                        -- 物理小区标识
    freq_channel    int,                        -- 频点号
    bandwidth       int,                        -- 带宽(kHz)
    is_connected    int,                        -- 1=主服务/2=邻区

    -- 位置
    gps_info_type   varchar,
    gps_raw         varchar,                    -- 原始 GPS 字符串
    lon_raw         float8,                     -- 经度
    lat_raw         float8,                     -- 纬度
    loc_source      varchar,                    -- 北京来源标记

    -- 信号强度（统一提取，不区分 LTE/NR 列名）
    sig_rsrp        int,                        -- RSRP (dBm)
    sig_rsrq        int,                        -- RSRQ (dB)
    sig_sinr        int,                        -- SINR (dB)
    sig_rssi        int,                        -- RSSI (dBm)，仅 LTE
    sig_dbm         int,                        -- 通用 Dbm
    sig_asu_level   int,                        -- ASU 等级
    sig_level       int,                        -- 信号等级 (0-4)
    sig_ss          int,                        -- ss1 原始信号值
    timing_advance  int,                        -- 时间提前量
    sig_csi_rsrp    int,                        -- CSI-RSRP（仅少量 NR）
    sig_csi_rsrq    int,                        -- CSI-RSRQ
    sig_csi_sinr    int,                        -- CSI-SINR
    sig_cqi         int,                        -- CQI（仅少量 LTE）

    -- 设备元数据
    did             varchar,
    plmn_main       varchar,                    -- 主卡运营商
    brand           varchar,
    model           varchar,
    sdk_ver         varchar,
    oaid            varchar,
    data_source     varchar,                    -- DNA/DAA

    -- 合规标记（解析后填充）
    is_valid_operator  boolean,                 -- 运营商合规
    is_valid_lac       boolean,                 -- LAC 合规
    is_valid_cellid    boolean,                 -- Cell ID 合规
    is_valid_gps       boolean,                 -- GPS 合规
    is_valid_signal    boolean                  -- 信号字段合规
);
```

## 7. 信号字段专项评估（之前遗漏的部分）

| 字段 | 异常值 | 出现原因 | 处理 |

| sig_rsrq | 90, >10 | SDK 上报错误 | 标记无效 |
