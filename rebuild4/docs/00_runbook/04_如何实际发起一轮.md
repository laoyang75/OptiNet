# 如何实际发起一轮

这份说明是给你实际操作 3 个外部 agent 用的。

## 1. 先确定当前阶段

按顺序执行：

1. `round1_plan`
2. `round2_detail`
3. `round3_execution`
4. `finalization`

不要跳阶段。

## 2. 每一轮你需要发给 3 个 agent 的内容

以某一轮 `<round>` 为例，你需要使用：

- `rebuild4/docs/02_rounds/<round>/prompts/00_共享约束.md`
- `rebuild4/docs/02_rounds/<round>/prompts/01_codex_prompt.md`
- `rebuild4/docs/02_rounds/<round>/prompts/02_claude_prompt.md`
- `rebuild4/docs/02_rounds/<round>/prompts/03_gemini_prompt.md`

实际发法：

- 给 Codex：`00_共享约束.md` + `01_codex_prompt.md`
- 给 Claude：`00_共享约束.md` + `02_claude_prompt.md`
- 给 Gemini：`00_共享约束.md` + `03_gemini_prompt.md`

## 3. 你对他们的要求

你要明确要求他们：

- 使用 prompt 指定的固定输出文件名
- 输出必须写到该轮自己的 `outputs/<agent>/` 目录
- 不要修改别的 agent 的目录
- 不要直接产出 merged 文档

## 4. 页面检查提醒

### `round1_plan`

- 不要求看页面
- 不要访问页面

### `round2_detail`

- 不要求看页面
- 不要访问页面

### `round3_execution`

- Codex 必须用 Playwright
- Claude 必须用 Antigravity 浏览器
- Gemini 必须用 Antigravity 浏览器
- 三者都必须用 `PG17 MCP`

### `finalization`

- 不重新访问页面
- 如果需要页面证据，只引用 round3 codex 输出目录中的截图和 round3 merged 的页面结论
- 三者都必须用 `PG17 MCP`

## 5. 三个 agent 都完成后怎么做

完成后，你不用自己合并。

下一步交给我：

1. 我读取 `outputs/` 下的 3 份结果
2. 我按 `90_merge_prompt.md` 合并到 `merged/`
3. 我按 `91_decision_prompt.md` 生成 `decisions/01_待裁决问题清单.md`
4. 如果是 `finalization`，再按 `92_taskbook_generation_prompt.md` 编制到 `03_final/`
5. 如果需要你裁决，我只给你问题文件

## 6. 你什么时候回复我

你只需要在 2 种情况下回复：

1. 告诉我“这一轮 3 个 agent 都输出完了”
2. 或者直接回复裁决结果，例如：`D-001: A` / `F-001: B`
