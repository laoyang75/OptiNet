# 统一映射矩阵

状态：统一稿
更新时间：2026-04-06
说明：本矩阵统一了页面主语、关键 API、逻辑表/视图、允许的 `data_origin`、最小可用数据与验收方式。

## 1. 使用规则

- 页面实现前，必须先冻结该页所在行
- API 字段变更前，必须先回写本矩阵
- 页面验收时必须同时查：PG17 SQL、API 返回、Playwright 页面结果

## 3. 页面级统一验收规则

### 3.1 必显元素

每个页面必须至少显示：

- 页面主语
- `data_origin`
- 关键主键或筛选上下文
- 时间上下文
- 如为 `synthetic/fallback/empty`，必须有原因说明

### 3.2 不允许的行为

- 静默使用 `synthetic`
- 用 `fallback` 覆盖 honest empty
- 页面主语与 API/表主语不一致
- 只改 UI 文案，不改 API/数据来源合同
