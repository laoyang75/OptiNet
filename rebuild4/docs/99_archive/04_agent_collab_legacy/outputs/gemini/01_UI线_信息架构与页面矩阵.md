# UI 线信息架构与页面矩阵（Gemini 草案）

状态：可合并草案  
更新时间：2026-04-06

## 1. UI 主逻辑

rebuild4 的 UI 不应再被理解成 dashboard 或 replay 工具。  
它的主链路必须是：

1. 进入流转总览，理解当前批次状态
2. 发现变化、观察项或问题项
3. 下钻对象列表与对象详情
4. 查看 baseline、画像、治理证据
5. 回到总览确认变化是否符合预期

## 3. 页面矩阵

| 页面 | 主语 | 主要 API | 主要表/读模型 | 允许来源 | 验收前提 |
| --- | --- | --- | --- | --- | --- |
| `/flow/overview` | `batch` | `/flow/overview` | `batch`, `batch_flow_summary`, 四分流事实层汇总 | `real` 主验收，`synthetic` 仅评估 | 至少 1 个真实 batch |
| `/flow/snapshot` | `batch + snapshot_timepoint` | `/flow/snapshot` | `batch_snapshot`, timepoint 摘要 | `real` 主验收，`synthetic` 仅评估 | 至少 1 条真实 snapshot 链 |
| `/runs` | `run / batch` | `/runs` | `run`, `batch` | `real` / `synthetic` 均可，但需标记 | 至少 1 个 run |
| `/objects` | 对象集合 | `/objects` | 对象读模型 | `real` 主验收 | 对象表已落地 |
| `/object/:kind/:id` | 单个对象 | `/objects/:kind/:id` | 对象详情读模型 | 跟随对象来源 | 对象详情可追溯 |
| `/observation` | 待观察对象/记录 | `/observation` | `fact_pending_observation` + 对象视图 | `real` 主验收 | 观察流已产生数据 |
| `/anomalies` | 问题对象/记录 | `/anomalies` | `fact_pending_issue` + 异常读模型 | `real` 主验收 | 问题流已产生数据 |
| `/baseline` | `baseline_version` | `/baseline` | `baseline_version`, baseline 摘要 | `real` 主验收 | 至少 1 个基线版本 |
| `/lac-profile` | `lac` 画像 | `/profiles/lac` | LAC 画像读模型 | 跟随对象来源 | LAC 画像已落地 |
| `/bs-profile` | `bs` 画像 | `/profiles/bs` | BS 画像读模型 | 跟随对象来源 | BS 画像已落地 |
| `/cell-profile` | `cell` 画像 | `/profiles/cell` | Cell 画像读模型 | 跟随对象来源 | Cell 画像已落地 |
| `/compare` | 对照任务 | `/compare` | compare 读模型 | `real` / `synthetic` / `fallback`，必须显式 | 对照语义已冻结 |
| `/initialization` | 数据准备状态 | `/initialization` | 初始化控制读模型 + 继承真相 | `real` 主验收 | P0/P1 准备完毕 |
| `/governance` | 治理证据 | `/governance` | 字段质量、ODS、trusted 损耗读模型 | `real` 优先，`fallback` 必须显式 | 继承真相已入库 |

## 4. 页面优先级

### 第一优先级

- `/runs`
- `/flow/overview`
- `/flow/snapshot`
- `/objects`
- `/object/:kind/:id`
- `/baseline`

原因：
- 这是主流程链的最短闭环

### 第二优先级

- `/observation`
- `/anomalies`
- `/initialization`
- `/governance`

原因：
- 这是支撑决策与解释的主辅助层

### 第三优先级

- `/lac-profile`
- `/bs-profile`
- `/cell-profile`
- `/compare`

原因：
- 画像需要依托对象与基线
- compare 不是主入口

## 5. 矩阵使用规则

1. 没有主语，就不允许建页面
2. 没有 API/表绑定，就不允许验页面
3. 没有 `data_origin`，就不允许称为正式模块
4. 页面实现顺序必须服从矩阵，不再允许“哪个先好做就先做哪个”
