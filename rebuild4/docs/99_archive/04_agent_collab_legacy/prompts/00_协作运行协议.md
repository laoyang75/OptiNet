# 多 Agent 协作运行协议

适用对象：Codex / Claude / Gemini 三条独立文档生成线。

## 1. 任务目标

本轮不是直接写代码，而是为 `rebuild4` 重构准备一套可执行、可校验、可合并的文档基线。

三条线分别关注：

1. 数据清洗整理规则线
2. 流转处理流程线
3. 操作 UI 线

每条线都需要生成自己负责范围内的正式文档草案，并为最终统一任务书提供输入。

## 2. 唯一输入边界

优先只使用 `rebuild4/docs` 目录内资料：

- `rebuild4/docs/01_foundation/`
- `rebuild4/docs/02_research/`
- `rebuild4/docs/03_workstreams/`
- `rebuild4/docs/UI_v2/`
- `rebuild4/docs/reference/`

禁止把 `rebuild2/`、`rebuild3/` 目录当作新的并行主真相源；如果需要引用历史内容，优先使用 `rebuild4/docs/reference/` 中的副本。

## 3. 输出要求

每条线都必须：

- 产出指定文件
- 明确事实、约束、冲突、缺口、风险
- 不得假装已经解决未解决的问题
- 给出可验证的检查方式
- 为最终统一任务书提供结构化输入

## 4. 强制校验要求

你写出的任何“步骤”都必须写出对应校验要求：

- 数据库事实必须通过 `PG17 MCP` 查询，不使用 shell 输出或人工估算替代
- 文档校验
- SQL / 数据校验
- API 校验（如涉及接口）
- 页面 Playwright 校验（如涉及页面）
- 未通过时的处理动作

## 5. 输出风格

- 用中文
- 面向执行，不写泛泛而谈的方案书
- 尽量结构化，少写空话
- 遇到冲突时必须把裁决依据写出来
- 能冻结就冻结，不能冻结就明确标成待确认
