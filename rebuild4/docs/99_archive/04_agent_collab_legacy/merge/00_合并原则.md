# 合并原则

## 1. 合并顺序

1. 先看数据真相
2. 再看流程语义
3. 再看 UI 表达
4. 最后生成统一执行任务书

## 2. 冲突裁决

- 数据真实性冲突：以数据库与正式元数据为准
- 流程语义冲突：以 Tier 0 和统一流程线结论为准
- UI 表达冲突：以 UI_v2 和页面映射矩阵为准，但不得伪造数据主语

## 3. 必须保留的记录

合并时必须保留：

- 冲突点
- 裁决理由
- 被舍弃方案
- 需要人工确认的问题

## 4. 最终输出约束

统一任务书中的每一步都必须包含：

- 输入
- 输出
- 数据校验
- API 校验
- 页面 Playwright 校验（如涉及页面）
- 未通过时的回退动作
