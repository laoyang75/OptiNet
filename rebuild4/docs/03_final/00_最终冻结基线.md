结论：**已具备。**

1. round1 已冻结唯一真相源、主链方向、governance / compare 边界与先合同后编码顺序。

### 1.3 冻结期页面依据与执行期页面验收的区分

1. **冻结期页面依据**：只承接 round3 已归档截图与结论，用于冻结页面主语、边界与成功标准；冻结阶段不再用页面重新裁方向。

## 4. 正式冻结与版本规则

### 4.1 正式冻结执行规则

1. **唯一真相包**：`03_final/00_05` 六文件共同构成正式冻结包。
2. **唯一数据库事实来源**：计数、比例、差集、缺口统一以 PG17 为准；结构确认统一以 PG17 结构检查为准。
3. **唯一主链来源**：rebuild4 正式主链只接受 `rebuild2.l0_gps + rebuild2.l0_lac` 重跑生成的 real 数据；`rebuild3*` 永远只读参考。
4. **G0 只读**：G0 只负责冻结包校验，不负责任何数据库写入；G1 的首个原子动作必须是 raw 时间跨度预检，首个数据库写动作只能发生在该预检通过后的 bootstrap schema 阶段。
5. **G0 正式 passed 先以 artifact 闭合**：在数据库控制表尚不存在前，`rebuild4/runtime_artifacts/gate/{package_id}/G0/attempt_001/index.json` 是 G0 唯一正式启动凭证；G1 只能在该文件存在、可读且声明 `status=passed` 时启动，之后再由 `P1-G1-04` 原样镜像进 `gate_run_result`，不得口头补记。
6. **执行期只读六文件**：执行期不得把包外决策文件、历史 merged 文稿或页面截图当作新的运行时合同来源。
7. **Gate 证据统一介质**：Gate 证据一律写入工作区相对路径 `rebuild4/runtime_artifacts/gate/{package_id}/{gate_code}/{attempt_id}/`；不同 attempt 的证据禁止混放；数据库只登记 repo-relative `evidence_ref`，不登记绝对路径、对象存储 URI 或 DB blob。
8. **current 只认显式指针**：`/api/flow-overview`、`/api/runs/current`、`/api/flow-snapshot/timepoints` 的 current 只允许读取正式控制层 `current_pointer`；仅因“数据库里已经出现 completed rolling run”不得自动切换。
9. **进入验收后冻结 current**：`G3` 一旦签发进入 `G4-G8` 的验收轮次，`current_pointer` 在该轮验收结束前不得再推进到新的 rolling batch；若必须继续 rolling，只能回到 `G3` 重新签发新的验收轮次。
10. **唯一上抛条件**：只有规则 seed 不可用、real rolling 窗口缺失、页面一级主语冲突三类异常真正发生时，才允许新增裁决文件。

### 4.2 package_id 升版规则

1. 六文件任一内容发生实质变化，都必须生成新的 `package_id` 与新的 per-file version。
2. 新版本写入 `contract_version` 后，旧版本必须退出 current，而不是被静默覆盖。
3. 执行期禁止在不升版的情况下“顺手改正式文件”。
4. 当前生效冻结包版本为 `rebuild4-final-freeze-2026-04-06-v7`。

### 4.3 Gate owner 角色矩阵（冻结逻辑角色，不冻结人名）

| Gate | 冻结 `gate_owner` | `backup_role` | 交接目标 |
|---|---|---|---|
| `G0` | `release_owner` | `seed_owner` | `G1` |
| `G1` | `seed_owner` | `release_owner` | `G2` |
| `G2` | `pipeline_owner` | `seed_owner` | `G3` |
| `G3` | `pipeline_owner` | `api_owner` | `G4` |
| `G4` | `api_owner` | `pipeline_owner` | `G5` |
| `G5` | `governance_owner` | `api_owner` | `G6` |
| `G6` | `baseline_owner` | `api_owner` | `G7` |
| `G7` | `compare_owner` | `release_owner` | `G8` |
| `G8` | `frontend_owner` | `api_owner` | 发布收口 |

冻结说明：

1. `gate_definition.gate_owner` 只允许写入上表角色名，不允许运行时自由命名。
2. 单线程执行时，同一人可以兼任多个角色，但库内仍必须使用冻结角色名；不得把“当前执行者名字”直接写成新的 `gate_owner`。
3. seed 审批优先要求职责分离；若当前只有 1 名执行者，则 `approved_by` 固定写 `single_owner_exception`，并在 `approval_note` 明示这是单线程自审降级。
