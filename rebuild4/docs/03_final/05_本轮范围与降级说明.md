
# 本轮范围与降级说明

## 1. 本轮正式范围

### 1.1 计入正式完成标准的 7 个模块

| 模块 | 交付内容 | 关键 Gate / 验收 |
|---|---|---|
| `M0` 合同与 Gate 冻结 | `03_final/` 六文件冻结包、manifest、`contract_version`、`gate_definition`、`gate_check_item` | `G0` + `G1` |
| `M1` 数据准备链路 | real `full_initialization`、real `rolling`、`batch_snapshot`、baseline 刷新链 | `G2` + `G3` |
| `M2` 解释层与治理底座 | 快照、规则 seed、seed manifest、结构资产目录、asset 语义映射、trusted loss | `G1` + `G5` |
| `M3` 主流程读模型 API | 12 个核心 API 家族 + 统一来源 envelope | `G4` |
| `M4` baseline / profile | baseline current、diff、history 与三类 profile | `G6` |
| `M5` compare 降级链路 | compare 的辅助验证身份、banner 与空 fallback；不计入 12 页，但计入正式放行 | `G7` + `P5-GPW-11` |
| `M6` 前端工作台 | 12 个正式页面通过 Playwright 验收；compare 另作辅助验证页复核 | `G8`（`P5-GPW-00` 到 `P5-GPW-11`） |

### 1.2 本轮必须继承的完成定义

1. `governance` 计入正式完成标准，不能继续停留在 mock 或空壳状态。
2. compare 只保留降级身份，不计入 12 个正式页面，但仍属于正式辅助验证模块并计入放行。
3. baseline 与 profile 必须诚实展示历史与缺口，不得伪造上一版。
4. 主流程页面必须真实消费来源语义，不允许 `synthetic` / `fallback` 悄悄顶上主流程。
5. `G1` 只阻塞主链起跑必须项：可执行 seed current、结构资产目录、控制层与快照表；`asset_usage_map / asset_migration_decision` 的人工语义补齐移到 `G5`，不再前置阻塞开发。

### 1.3 12 个正式页面的计数口径

本轮 `12 个正式页面` 的唯一解释空间如下：

1. `/flow-overview`
2. `/flow-snapshot`
3. `/runs`
4. `/objects`
5. `/observation-workspace`
6. `/anomaly-workspace`
7. `/baseline`
8. `/profiles/lac`
9. `/profiles/bs`
10. `/profiles/cell`
11. `/initialization`
12. `/governance`

补充规则：

- `/objects/:key` 是 `/objects` 的详情态，不单独计页。
- `/api/batches/:id/detail` 是 `/runs` 的子资源，不对应独立页面。
- `/validation/compare` 是辅助验证页，不计入上述 12 个正式页面。
- `P00` 是全局导航与信息架构验收，不单独计入 12 页。

## 2. 明确降级范围

### 2.1 compare 降级边界

| 项 | 冻结边界 |
|---|---|
| 模块身份 | 辅助验证模块，不是日常主工作流 |
| 允许来源 | `real / synthetic / fallback` |
| 页面要求 | 首屏必须显示“辅助验证”身份；`synthetic` / `fallback` 必须出现 banner |
| 数据要求 | `fallback` 只允许空 envelope，不展示伪结果 |
| 完成定义 | `G7` 只验降级身份成立，不要求 compare 成为主链日常能力；但 compare 模块本身仍计入正式放行 |
| 页面计数 | 不计入 12 个正式页面 |

### 2.2 legacy 信息降级边界

| 项 | 冻结边界 |
|---|---|
| legacy 字段 | 只允许通过 API `legacy_ref` 嵌套返回 |
| 页面展示 | 只允许灰色参考区 / 详情参考区 |
| 明确禁止 | 不进入主状态模型、不进入主筛选条件、不参与 Gate 通过判断 |

## 3. 明确排除范围

以下内容不计入本轮正式完成标准：

1. **launcher / 运维健康检查**：本轮只冻结接口保留名和边界，不进入 `G0-G8`、12 个正式页面与统一数据合同成功标准。
2. **`rebuild3_sample_meta` 等历史兼容资产**：只作历史参考，不进入 rebuild4 主链。
3. **`rebuild2` / `rebuild3*` 的 current 化、转正化、回写行为**：全部禁止。
4. **旧前端整体 fork**：禁止复用 rebuild3 项目骨架、路由、Store、axios 与隐式 fallback 逻辑。
5. **`scenario_replay`、mock 数据、手动拼装结果**：禁止作为主链正式完成证据。

## 4. 阶段顺序与停机线

### 4.1 严格串行的阶段顺序

正式执行顺序冻结为：

`Phase 0 (G0) -> Phase 1 (G1) -> Phase 2A (G2) -> Phase 2B (G3) -> Phase 3 (G4 -> G5 -> G6) -> Phase 4 (G7) -> Phase 5 (G8)`

冻结含义：

1. 不允许跳过前置 Gate 提前展开后续阶段。
2. G0 只做冻结包只读校验；G1 的第一个原子动作必须是双输入 raw `上报时间` 预检；只有预检通过后才允许进入 bootstrap 写库。
3. 当前已确认 `rebuild2.l0_lac` 与 `rebuild2.l0_gps` 都覆盖 `2025-12-01T00:00:00Z -> 2025-12-07T23:59:59Z`，约 `167.9997` 小时。
4. 不允许在 `G2` 之前启动 rolling；也不允许在 `P2A-G2-06` 完成 standardized `event_time` 覆盖检查前启动 rolling。
5. 只有在 `P2B-G3-05` 已把 `G3` 签发为 `passed`、且 `rebuild4/runtime_artifacts/gate/{package_id}/G3/{attempt_id}/index.json` 存在时，才允许启动 `G4`。
6. 不允许在 `G4` 之前让主流程页面接正式 API；`G5/G6` 只能在 `G4` 的合同测试通过后继续补完整性。
7. 不允许在 `G7` 之前把 compare 当成主导航主语或正式完成证据。
8. 不允许在 `G8` 之前把页面 Playwright 结果记作正式通过证据；`G8` 只有在 `P5-GPW-00` 到 `P5-GPW-11` 全部通过且 `G8/{attempt_id}/index.json` 汇总完成后才算正式通过，其中 `P5-GPW-11` 不计入 12 个正式页面计数、但计入 compare 模块放行。
9. `G3` 一旦签发本轮验收批次，`G4-G8` 期间不得继续推进 `current_pointer`；若要继续 rolling，只能回到 `G3` 重新签发新的验收轮次。

### 4.2 launcher 与主链的关系

launcher 可以独立实现、独立上线、独立维护，但不得影响以下冻结事实：

- 主链成功标准仍由 `G0-G8` 决定。
- launcher 不承担主链数据主语。
- launcher 不改变 API envelope、数据来源语义与页面矩阵。
- 本轮执行/验收闭环不包含 launcher；是否实现 launcher，不影响主链冻结完成判定。

## 5. 仅在异常发生时才允许新增裁决

当前无需新增人类裁决问题。只有以下 3 类异常真正发生时，才允许重新开启裁决文件：

| 异常类型 | 触发条件 | 届时才允许上抛的内容 |
|---|---|---|
| 规则 seed 不可用 | 按 `canonical_seed_contract`、`seed_builder` 构建指南与 current `seed_artifact_manifest` 仍无法生成非空、可审计 canonical seed | 是否放宽 seed 来源或接受临时占位值 |
| real rolling 窗口缺失 | 客观上无法形成符合 `rolling_window_contract` 的 completed real batch `> 1` | 是否允许人工切片或接受 `G3` 暂缓 |
| 页面一级主语冲突 | 同一页面在冻结矩阵下仍出现不可消解的一级主语冲突 | 是修矩阵还是拆页面 |

在以上异常没有发生前，禁止为了实施细节、措辞差异或局部工程便利重新上抛方向问题。

## 6. 设计文档未实现能力的处理

以下为 UI 设计规范中裁决后确认不做或调整实现方式的能力项：

| 设计能力 | 裁决结果 | 说明 |
|---|---|---|
| 流转总览页批次切换器（对比功能） | 不做 | 流转总览只加 run 下拉框回看；跨 run 对比职责由流转快照承担 |
| 画像展开行口径说明 | 做 | 初始方案为不做，裁决后反转：画像展开行必须包含口径说明区域（灰色背景块，根据字段 metadata 动态生成） |
| 批次趋势折线图 | 不做 | 使用 3 列 metric 趋势摘要卡片替代折线图 |
| 观察工作台虚拟滚动 | 不做 | 使用分页替代虚拟滚动 |
| 全局状态条 GlobalStatusBar | 做 | 裁决确认实现全局状态条（正常隐藏、处理中蓝色脉冲、重跑紫色、验收灰蓝） |
