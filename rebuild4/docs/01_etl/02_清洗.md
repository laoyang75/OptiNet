# ETL Step 2: 清洗

## 做什么

检查解析后的每一行数据，把无效值清理掉。

清洗有两种方式：
- **置空（nullify）**：把无效值改成 NULL，保留这行（比如运营商编码无效，但基站信息可能有用）
- **删除行**：CellID 无效的行直接删掉（没有小区标识的数据无法归属，没有分析价值）

## 规则总览

共 16 条规则，分 5 个类别：

```
运营商（2 条）→ 无效编码置空
LAC（3 条）  → 无效值置空
CellID（3 条）→ 无效值置空 + 最终删除行
信号（4 条）  → 越界值置空
位置（2 条）  → GPS 越界标记无效
```

## 规则详解

### 运营商清洗

| ID | 规则 | 条件 | 动作 |
|----|------|------|------|
| ODS-001 | 垃圾编码 | '00000', '0', '000000', '(null)(null)', '' | 置空 |
| ODS-002 | 非白名单 | 不在 46000~46020 白名单 | 置空 |

白名单：46000(移动), 46001(联通), 46002(移动), 46003(电信), 46005(电信), 46006(联通), 46007(移动), 46009(联通), 46011(电信), 46015(广电), 46020(铁路)

### LAC 清洗

| ID | 规则 | 条件 | 说明 |
|----|------|------|------|
| ODS-003 | LAC=0 | lac = 0 | 未获取到 LAC |
| ODS-004 | 4G 保留值 | lac IN (65534,65535) AND 4G | 16bit 保留值 |
| ODS-005 | 溢出值 | lac = 268435455 | 0xFFFFFFF |

### CellID 清洗

| ID | 规则 | 条件 | 说明 |
|----|------|------|------|
| ODS-006 | CellID=0 | cell_id = 0 | 无小区 |
| ODS-007 | 5G 溢出 | cell_id = 268435455 AND 5G | 28bit 溢出 |
| ODS-008 | 4G 溢出 | cell_id = 2147483647 AND 4G | Integer.MAX_VALUE |

### 信号清洗

| ID | 规则 | 有效范围 | 说明 |
|----|------|----------|------|
| ODS-009 | RSRP | -156 ~ -1 | 0 和正数是无效值 |
| ODS-010 | RSRQ | -34 ~ 10 | |
| ODS-011 | SINR | -23 ~ 40 | |
| ODS-012 | Dbm | < 0 | 0 和正数是无效值 |

### 位置清洗

| ID | 规则 | 条件 | 动作 |
|----|------|------|------|
| ODS-013 | 非原生GPS标记无效 | gps_info_type 不是 'gps' 或 '1' | 标记 gps_valid=false |
| ODS-014 | 非原生GPS经纬度置空 | gps_valid=false 且有坐标 | lon_raw/lat_raw 置 NULL |
| ODS-015 | 经度越界 | 经度 < 73 或 > 135 | 标记 gps_valid=false |
| ODS-016 | 纬度越界 | 纬度 < 3 或 > 54 | 标记 gps_valid=false |

**GPS 来源说明**：gps_info_type 的值含义：
- `'gps'` 或 `'1'`：原生 GPS 定位（可信，保留）
- `'2'`：WiFi 定位（不可信，清除）
- `'4'`：基站定位（不可信，清除）
- `'5'`：其他来源（不可信，清除）
- `NULL`：ss1 行（无此字段，ss1 的 GPS 来自 gps_block，单独判断）

### 最终行过滤

置空完成后，删除 CellID 为 NULL 或 0 或溢出值的行。这些行没有有效的小区标识，无法用于后续分析。

## 派生字段

清洗完成后计算：

| 字段 | 公式 | 说明 |
|------|------|------|
| bs_id | 4G: cell_id/256, 5G: cell_id/4096 | 基站 ID |
| sector_id | 4G: cell_id%256, 5G: cell_id%4096 | 扇区 ID |
| operator_cn | 46000→移动, 46001→联通, 46011→电信... | 运营商中文名 |
