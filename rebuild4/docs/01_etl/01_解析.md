# ETL Step 1: 解析（炸开）

## 做什么

把原始记录里的 `cell_infos`（JSON）和 `ss1`（文本）解析成结构化的行。

一条原始记录包含手机扫描到的**多个基站**和**多组信号**，解析后每个基站变成独立的一行。

## cell_infos 解析

### 输入格式

JSON 对象，key 是序号，value 是一个基站的信息：

```json
{
  "1": {
    "isConnected": 1,
    "type": "nr",
    "cell_identity": {
      "mccString": "460", "mncString": "01",
      "Nci": 563347713, "Tac": 65546, "Pci": 515,
      "ChannelNumber": 633984
    },
    "signal_strength": {
      "SsRsrp": -88, "SsRsrq": -11, "SsSinr": 8,
      "Dbm": -88, "AsuLevel": 52, "Level": 2
    }
  }
}
```

### 提取字段

| 目标字段 | 来源 | 说明 |
|----------|------|------|
| tech_norm | cell.type | lte→4G, nr→5G, gsm→2G, wcdma→3G |
| operator_code | cell_identity.mno 或 mccString+mncString | 运营商编码 |
| lac | cell_identity.Tac / tac / lac / Lac | 位置区编码 |
| cell_id | cell_identity.Ci / Nci / nci / cid | 小区标识 |
| pci | cell_identity.Pci | 物理小区标识 |
| freq_channel | cell_identity.Earfcn / ChannelNumber / arfcn | 频点号 |
| bandwidth | cell_identity.Bwth | 带宽(kHz) |
| rsrp | signal_strength.rsrp / SsRsrp | 信号强度 |
| rsrq | signal_strength.rsrq / SsRsrq | 信号质量 |
| sinr | signal_strength.rssnr / SsSinr | 信噪比 |
| rssi | signal_strength.rssi | 接收信号强度 |
| dbm | signal_strength.Dbm | 分贝毫瓦 |
| asu_level | signal_strength.AsuLevel | 0~99 |
| sig_level | signal_strength.Level | 0~4 |
| timing_advance | signal_strength.TimingAdvance | 时间提前量 |
| csi_rsrp/rsrq/sinr | signal_strength.CsiRsrp/Rsrq/Sinr | CSI 信号（5G） |
| cqi | signal_strength.cqi | 信道质量 |
| lon_raw, lat_raw | 原始记录 `原始上报gps` | GPS 经纬度 |

### 直传字段（从原始记录直接带入）

data_source, data_source_detail, dev_id, ip, plmn_main, brand, model, sdk_ver, oaid, pkg_name, wifi_name, wifi_mac, cpu_info, pressure, ts_raw, gps_ts_raw, gps_info_type

### 处理规则

1. `jsonb_each()` 拆开每个 cell
2. **只保留 isConnected=1** 的主服务基站
3. GPS 标记 `gps_filled_from = 'raw_gps'`

## ss1 解析

### 输入格式

分号分隔文本，每组 4 段（`&` 分隔）：

```
信号段&时间段&GPS段&基站段;信号段&时间段&GPS段&基站段;...
```

### 每段含义

| 段 | 格式 | 示例 |
|----|------|------|
| 信号段 | 逗号分隔，首字母=制式 | `l,-85,-10,15` |
| 时间段 | unix 秒 | `1765011706` |
| GPS段 | `经度,纬度` 或空 | `116.397,39.908` |
| 基站段 | `制式,cell_id,lac,plmn` 或 `1` | `l,12345,4176,46000` |

### 基站继承（forward-fill）

```
组1: ... & l,12345,4176,46000    ← 有基站
组2: ... & 1                      ← 继承组1的基站
组3: ... & n,67890,73733,46001   ← 新基站
组4: ... & 1                      ← 继承组3的基站
```

### ss1 特有字段

| 字段 | 说明 |
|------|------|
| sig_ss | ss1 原始信号值（信号段第 2 个值） |
| cell_ts_raw | 组时间戳（时间段，unix 秒） |

### ss1 没有的字段（置 NULL）

pci, freq_channel, bandwidth, rssi, dbm, asu_level, sig_level, timing_advance, csi_*

| 字段 | 覆盖率 | 说明 |
|------|--------|------|
| operator_code | ~99.8% | 极少数缺失 |
| lac | ~99.9% | |

| rssi | ~16% | 仅部分 LTE |
| wifi_name | ~46% | 原始数据部分有 |
| pressure | ~12% | 仅部分设备有气压传感器 |
