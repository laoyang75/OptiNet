# ETL Step 1: 解析（炸开）

## 做什么

把原始记录里的 `cell_infos`（JSON）和 `ss1`（文本）解析成结构化的行。

一条原始记录包含手机扫描到的**多个基站**和**多组信号**，解析后每个基站变成独立的一行。

## cell_infos 解析

### 输入格式

JSON 对象，key 是序号，value 是一个基站的信息：

```json
{
  "1": {
    "isConnected": 1,
    "type": "nr",
    "cell_identity": {
      "mccString": "460", "mncString": "01",
      "Nci": 563347713, "Tac": 65546, "Pci": 515
    },
    "signal_strength": {
      "SsRsrp": -88, "SsRsrq": -11, "SsSinr": 8
    }
  },
  "2": { "isConnected": 2, ... }
}
```

### 处理规则

1. 用 `jsonb_each()` 拆开每个基站
2. **只保留 isConnected=1 的主服务基站**（isConnected=2 是邻区，暂不用）
3. 提取字段：

| 目标字段 | 来源 | 说明 |
|----------|------|------|
| tech_norm | cell.type | lte→4G, nr→5G, gsm→2G, wcdma→3G |
| operator_code | cell_identity.mno 或 mccString+mncString | 运营商编码 |
| lac | cell_identity.Tac / tac / lac / Lac | 位置区编码 |
| cell_id | cell_identity.Ci / Nci / nci / cid | 小区标识 |
| rsrp | signal_strength.rsrp / SsRsrp | 信号强度 |
| lon_raw, lat_raw | 原始记录的 `原始上报gps` | GPS 经纬度 |

4. GPS 标记 `gps_filled_from = 'raw_gps'`

### 代码位置

`pipeline.py → _parse_cell_infos()`

## ss1 解析

### 输入格式

分号分隔的文本，每组 4 段（`&` 分隔）：

```
信号段&时间段&GPS段&基站段;信号段&时间段&GPS段&基站段;...
```

### 每段含义

| 段 | 格式 | 示例 |
|----|------|------|
| 信号段 | 逗号分隔，首字母标识制式 | `l,-85,-10,15` (l=LTE) |
| 时间段 | 毫秒时间戳 | `1701234567890` |
| GPS段 | `经度,纬度` 或空 | `116.397,39.908` |
| 基站段 | `制式,cell_id,lac,plmn` 或 `1`(继承) | `l,12345,4176,46000` |

### 处理规则

1. 去尾部 `;`，按 `;` 分组
2. 每组按 `&` 拆成 4 段
3. **基站继承**：如果基站段 = `1`，使用上一组的基站（同一条报文内向前查找）
4. 从基站段解析 cell_id、lac、运营商
5. 从信号段解析 RSRP/RSRQ/SINR，按制式匹配到基站
6. GPS 来自本组 GPS 段，标记 `gps_filled_from = 'ss1_own'`

### 基站继承（forward-fill）

```
组1: 信号A & 时间1 & GPS1 & l,12345,4176,46000    ← 有基站
组2: 信号B & 时间2 & GPS2 & 1                       ← 继承组1的基站
组3: 信号C & 时间3 & GPS3 & n,67890,73733,46001    ← 新基站
组4: 信号D & 时间4 &      & 1                       ← 继承组3的基站
```

使用 SQL 窗口函数实现：
```sql
MAX(cb_own) OVER (PARTITION BY record_id ORDER BY grp_idx
                  ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)
```

### 代码位置

`pipeline.py → _parse_ss1()`
