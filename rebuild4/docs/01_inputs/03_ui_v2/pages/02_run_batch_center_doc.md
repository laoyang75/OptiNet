# 运行/批次中心 设计说明

## 页面目标
帮助用户诊断批次级变化，了解每个批次的四分流结果、晋升降级情况、趋势走向。

## 这个页面主要回答什么问题
1. 当前 run 包含多少个批次，哪些已完成
2. 哪个批次的四分流比例异常
3. 哪个批次的 waiting/issue 激增
4. 趋势是好转还是恶化
5. 哪个批次触发了 baseline 刷新
6. 哪些批次是重跑的（区分正常和修复后重跑）

## 用户主要操作
1. 浏览批次列表，识别异常批次
2. 点击批次查看详情（四分流、晋升/降级）
3. 观察近 N 批趋势图
4. 从晋升/降级列表下钻到对象详情
5. 识别重跑批次及其影响

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 运行上下文 | 5% | 当前 run 信息 | run_id/type/status/versions | 否 | 否 |
| 批次列表 | 35% | 批次一览+异常发现 | batch_id/窗口/类型/输入量/四分流/delta | 是 | 是(→批次详情) |
| 批次趋势图 | 20% | 趋势判断 | 近N批各指标折线 | 是 | 否 |
| 批次详情区 | 40% | 选中批次深度查看 | 四分流明细/晋升降级列表/级联更新 | 是 | 是(→对象详情) |

## 筛选器与切换器
- 批次类型筛选：全部 / 正常 / 重跑
- 趋势图时间范围：近6批 / 近12批 / 近24批
- 批次状态筛选：全部 / 运行中 / 完成 / 失败

## 状态表达规则
- 批次状态：运行中(蓝色脉冲) / 完成(绿色) / 失败(红色)
- 重跑批次：紫色虚线左边框 + "重跑" 徽标
- Delta: 改善=绿色，恶化=红色
- 四分流条形：governed(绿) / pending_obs(黄) / pending_issue(橙) / rejected(红)

## 下钻路径
- 批次行 → 展开批次详情
- 晋升列表中对象 → 对象详情
- 降级列表中对象 → 对象详情
- 四分流某层 → 该层事实明细（对象浏览页筛选）

## 组件边界建议
- RunBatchCenterView.vue (页面)
- RunContextBar.vue (运行上下文)
- BatchTable.vue (批次列表表格)
- BatchTrendChart.vue (趋势图)
- BatchDetailPanel.vue (选中批次详情)
- FlowDistributionBar.vue (四分流条形图)
- PromotionList.vue (晋升/降级列表)

## 读模型建议
- GET /api/runs/current 当前运行上下文
- GET /api/batches?run_id=xxx 批次列表（预聚合四分流、delta）
- GET /api/batches/:id/detail 批次详情（晋升/降级/级联列表）
- GET /api/batches/trend?last=7 趋势数据

## 空状态 / 错误状态 / 重跑中状态
- 空状态：无批次时显示"当前 run 尚未产生批次"
- 重跑中：重跑批次行显示紫色标记，详情区显示"此批次为修复后重跑"

## 开发注意事项
- 批次列表可能很长（24批/天），需要分页或虚拟滚动
- 四分流百分比可用迷你内联条形图展示
- 重跑批次需要后端标记 is_rerun 和 rerun_source_batch
