# 基础数据治理 — 设计说明文档

## 1. 页面目标

提供 rebuild3 数据资产的全景视图，让开发者和数据工程师在一个页面内完成以下工作：

- 了解系统中有哪些表和字段、各自归属哪个层级
- 判断每个数据资产的迁移结论（直接复用 / 重组迁移 / 仅参考 / 可淘汰）
- 追踪字段和表的实际使用位置，识别僵尸资产
- 为 rebuild2 → rebuild3 迁移提供决策依据

## 2. 回答什么问题

| 问题 | 对应区域 |
|------|----------|
| 系统里一共有多少张表、多少个字段？ | 顶部概览卡 |
| 某个字段属于哪个层级、被哪些页面使用？ | Tab 1 字段目录 |
| 某张表的粒度、主键、上下游关系是什么？ | Tab 2 表目录 |
| 某张表被哪些 UI 页面 / API / 批处理任务使用？ | Tab 3 实际使用 |
| 哪些资产可以直接复用、哪些需要改造？ | Tab 4 迁移状态 |
| rebuild2 遗留了哪些资产？是否可以淘汰？ | Tab 4 迁移状态（仅参考 + 可淘汰分组） |

## 3. 用户操作

### 概览卡
- 纯展示，无交互。6 个卡片分两行展示登记和使用统计。

### Tab 切换
- 4 个 Tab：字段目录（默认）、表目录、实际使用、迁移状态。
- 点击 Tab 按钮切换面板，当前 Tab 高亮显示。

### Tab 1 字段目录
- **搜索**：输入字段名或中文说明，实时过滤表格行。
- **筛选器**：层级 / 数据类型 / 是否核心 / 迁移状态，四个下拉独立筛选，可组合使用。
- 表格支持排序（后续迭代）。

### Tab 2 表目录
- **搜索**：输入表名或中文说明。
- **筛选器**：表类型 / 迁移状态。

### Tab 3 实际使用
- 左侧列表点击切换右侧详情面板。
- 左侧搜索框过滤表名。
- 右侧展示使用方列表、上下游关系链、使用说明。

### Tab 4 迁移状态
- 按迁移结论分 4 个色彩区块：绿（直接复用）、蓝（重组迁移）、橙（仅参考）、红（可淘汰）。
- 每个区块内列出具体资产和操作建议。

## 4. 区块说明表

| 区块 | 位置 | 数据来源 | 说明 |
|------|------|----------|------|
| 概览卡（上行） | 页面顶部 | 元数据统计 | 已登记表 / 核心依赖 / 历史兼容 / 研究表 |
| 概览卡（下行） | 页面顶部 | 元数据统计 | 已登记字段 / 实际使用字段 / 待迁移确认 |
| 字段目录表格 | Tab 1 | 字段元数据注册表 | 15 行 mock，覆盖源/标准、对象、画像、元数据层级 |
| 表目录表格 | Tab 2 | 表元数据注册表 | 12 行 mock，覆盖事实/对象/画像/基线/元数据/研究表 |
| 实际使用面板 | Tab 3 | 使用方注册 + 血缘分析 | 左右布局，选中 obj_cell 为默认展示 |
| 迁移状态分组 | Tab 4 | 迁移决策表 | 4 组：直接复用 8 项、重组迁移 6 项、仅参考 4 项、可淘汰 2 项 |

## 5. Tab 切换机制

- 使用原生 JS `switchTab(name)` 函数。
- 每个 Tab 面板 ID 格式：`tab-{name}`（field / table / usage / migration）。
- 切换时移除所有 `.active`，为目标按钮和面板添加 `.active`。
- 默认显示 Tab 1 字段目录。

## 7. 组件边界

| 组件 | 职责 | 输入 | 输出 |
|------|------|------|------|

| TableCatalog | 表目录表格 + 搜索筛选 | 表列表 + 筛选条件 | 纯展示 |

| MigrationStatus | 按结论分组展示 | 迁移决策列表 | 纯展示 |

## 8. 读模型建议

本页面需要以下读模型 / API：

| API | 方法 | 说明 |
|-----|------|------|
| `GET /api/governance/overview` | GET | 返回概览统计（表数、字段数、使用数、待迁移数） |
| `GET /api/governance/fields` | GET | 返回字段目录，支持 `?tier=&type=&core=&migration=&q=` 筛选 |
| `GET /api/governance/tables` | GET | 返回表目录，支持 `?table_type=&migration=&q=` 筛选 |
| `GET /api/governance/usage/:table_name` | GET | 返回指定表的使用方列表、上下游关系、使用说明 |
| `GET /api/governance/migration` | GET | 返回按迁移结论分组的资产列表 |

所有 API 为只读，不涉及写入操作。数据源来自元数据注册表（手动维护或从 DDL / 代码扫描自动生成）。

## 9. 侧边栏分组

侧边栏导航分三组，用分隔线区分：

- **主流程层**：流转总览、流转快照、运行/批次中心、对象浏览、等待/观察工作台、异常工作台、基线/画像
- **画像视角层**：LAC 画像、BS 画像、Cell 画像
- **支撑治理层**：基础数据治理（当前页面，active）、验证/对照、初始化数据
