# 等待/观察工作台 设计说明

## 页面目标
帮助用户判断候选对象的**三层资格推进节奏**，发现堆积和停滞，决定继续观察、晋升还是转问题。

## 核心概念：三段式资格进度

每个候选对象不再用单一阈值衡量晋升距离，而是分三层依次推进：

| 层级 | 名称 | 含义 | 关键指标 | 前置条件 |
|------|------|------|---------|---------|
| L1 | **存在资格** | 最小注册门槛，证明这个对象确实存在 | 合规主键稳定、GPS有效点>=N、设备数>=N、时间跨度>=N天 | 无 |
| L2 | **锚点资格 (anchorable)** | 满足锚定条件，可以确定位置 | GPS有效点>=10、设备>=2、P90半径<1500m、最低跨度>=1天、无禁锚异常 | 存在资格达成 |
| L3 | **基线资格 (baseline_eligible)** | 满足基线建模条件 | 更高样本量/活跃天数、足够原始信号占比、稳定性 | 锚点资格达成 |

**重要**：`10 GPS / 2 设备 / 3 天` 等数值不再直接作为对象晋升门槛，它们是某一层资格的参考阈值。

## 这个页面主要回答什么问题
1. 等待池中有多少对象，是否在堆积
2. 每个候选当前缺哪一层资格（存在/锚点/基线）
3. 每层资格内的具体指标缺什么
4. 最近几个批次有推进吗（前进/停滞/回退）
5. 建议动作对应到具体缺失的资格层和指标
6. 堆积的主要原因按资格层分布

## 用户主要操作
1. 浏览卡片扫描三层资格进度和趋势
2. 按缺失资格层/推进状态筛选
3. 点击卡片下钻到对象详情
4. 查看堆积分析（按缺失资格类型分布）

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 汇总条 | 5% | 总量+delta+告警 | waiting数/observing数/晋升数/转问题数 | 是 | 否 |
| 筛选栏 | 5% | 过滤排序 | 缺失资格层/推进状态/排序 | 否 | 否 |
| 候选卡片区 | 70% | 逐对象三层资格推进判断 | 主键/三层资格进度/趋势/质量/建议 | 是 | 是(→详情) |
| 堆积分析 | 20% | 系统性堆积诊断（按缺失资格类型） | 趋势图/缺失资格类型分布 | 是 | 否 |

## 卡片结构设计

每张候选对象卡片包含：

1. **头部**：Cell 主键 + lifecycle badge (waiting/observing)
2. **三层资格进度区**：
   - 存在资格：综合进度 XX%（各指标明细可折叠）
   - 锚点资格：综合进度 XX%（需存在资格达成后才能评估，否则显示锁定状态）
   - 基线资格：综合进度 XX%（需锚点资格达成后才能评估，否则显示锁定状态）
3. **推进趋势**：前进中 / 停滞 / 回退 / 新进入 + 近批 delta
4. **建议动作**：明确说明"缺存在资格的XX"还是"缺锚点资格的XX"
5. **质量指标**：质心坐标、P90半径
6. **底部**：首次发现日期、停滞批次数

### 资格层视觉状态
- **达成**：绿色边框 + 绿色勾号图标 + 100% 文字
- **进行中**：正常边框 + 对应颜色图标 + 展开指标明细
- **锁定**：虚线边框 + 半透明 + "需XX资格达成后才能评估"

## 筛选器与切换器
- Tab: 等待中/观察中/全部
- 缺失资格层: 全部/缺存在资格/缺锚点资格/缺基线资格/全部达标
- 推进状态: 全部/前进中/停滞/回退
- 排序: 进度降序/首次发现/停滞批次数

## 堆积分析区

改为按"缺失资格类型"分布：
- 缺存在资格 38%
- 缺锚点资格 42%
- 缺基线资格 15%
- 全部达标(待晋升) 5%

保留近 6 批次 waiting 数量趋势柱状图。

## 状态表达规则
- waiting: Amber-100 底色 | observing: 深 Amber 底色
- 资格层达成: 绿色勾号 + 绿色边框 + "100%"
- 资格层进行中: 对应颜色图标（存在=indigo, 锚点=amber, 基线=purple）
- 资格层锁定: 虚线边框 + 半透明 + "--"
- 趋势: 前进↗绿 / 停滞→灰 / 回退↙红 / 新进入 蓝
- 建议: 继续观察(灰) / 即将晋升(绿边) / 建议关注(黄边) / 建议转问题(橙边)

## 下钻路径
- 卡片 → 对象详情
- 汇总条晋升数 → 本批晋升对象列表

## 组件边界建议
- ObservationWorkspaceView.vue
- CandidateCard.vue（含三层 QualificationLayer 子组件）
- QualificationLayer.vue（含 met/in-progress/locked 三态）
- QualificationDetailRow.vue（指标明细行）
- TrendIndicator.vue
- SuggestedActionBadge.vue（动作文案需包含缺失资格层信息）
- AccumulationAnalysis.vue（按缺失资格类型分布）

## 读模型建议
- GET /api/observation-workspace?lifecycle=waiting&missing_qual=anchorable&sort=progress_desc
- 后端预计算字段:
  - existence_progress, anchorable_progress, baseline_progress
  - current_missing_layer (存在/锚点/基线/无)
  - missing_detail (具体缺失指标描述)
  - trend_direction, stall_batches
  - suggested_action (需包含缺失资格层信息)

## 空状态 / 错误状态 / 重跑中状态
- 空: "当前没有等待或观察中的对象"
- 重跑: 提示"部分对象状态可能因重跑更新"

## 开发注意事项
- 卡片数量可达1000+，需虚拟滚动或分页
- 三层资格进度由后端预计算，前端不应重复计算阈值逻辑
- 建议动作由后端推荐，必须包含缺失资格层和具体指标信息
- 卡片改为 2 列布局（三层资格区需要更多纵向空间）
- 资格层指标明细默认折叠，仅当前缺失层展开
