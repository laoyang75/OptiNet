# 第 1 轮：总体评估与计划 - 共享约束

## 1. 任务模式

本轮采用“同题独立评估”。

- Codex / Claude / Gemini 做的是同一个任务
- 目标不是分工写作，而是互补、查漏、互证
- 输出必须使用完全相同的文件名

## 2. 唯一输入范围

只允许使用：

- `rebuild4/docs/00_runbook/`
- `rebuild4/docs/01_inputs/`
- 当前轮之前已经统一的 `merged/` 文档
- 当前轮之前已经裁决完成的 `decisions/` 文档
- `PG17 MCP`

禁止：

- 把 `rebuild4/docs/99_archive/` 当作新的主真相源
- 自己发明新的需求
- 把旧文档里的冲突数字并列当真而不裁决

## 3. 数据库事实要求

- 所有计数、比例、差异、缺口，都必须通过 `PG17 MCP execute_sql` 验证
- `search_objects` 只可用于确认表结构、字段、索引
- 不允许引用 shell 输出代替数据库事实

## 4. 页面规则

本轮不要求看页面；禁止为了凑内容而访问页面。只允许使用 `rebuild4/docs/01_inputs/*` 和 `PG17 MCP`。

## 5. 输出硬约束

必须产出：

- `01_总体评估与计划.md`
- `02_主要风险与缺口.md`
- `03_候选裁决问题.md`

每份文档都必须：

- 讲清结论
- 讲清依据
- 讲清风险
- 讲清校验方式
- 不得把未验证推测写成事实
