## 3. 仅在以下异常情况下才新增裁决

以下 3 类异常发生前，禁止新建裁决文件、禁止暂停执行：

**异常类型 1（规则 seed 可用性）**  
`parse_rule` / `compliance_rule` 在遍历所有可用资料（文档、SQL、Python）后，仍无法形成非空、可审计的正式 seed，导致 G1 中断。  
→ 届时需上抛：是否放宽 seed 来源范围，或接受初始 seed 为占位值進入 G2。

**异常类型 2（rolling 输入窗口缺失）**  
客观上无法获得满足 `completed batch > 1` 的 real rolling 数据窗口，导致 G3 无法通过。  
→ 届时需上抛：是否允许以 `rebuild2.l0_*` 的人工切片代替，或接受 G3 暂缓。

**异常类型 3（页面主语新冲突）**  
同一页面在已冻结的矩阵下出现不可消解的一级主语冲突（例如同一页面同时需要 `run` 与 `batch` 作为一级主语）。  
→ 届时需上抛：以矩阵修订或页面拆分哪种方式解决。
