# 统一最终校验清单

状态：round3 主控合并稿  
更新时间：2026-04-06  
适用范围：rebuild4 从 `G0` 到最终冻结的逐步校验、停机、回退与验收。  
强制工具：数据库事实用 `PG17 MCP execute_sql`；结构确认用 `PG17 MCP search_objects`；页面验收用 Playwright。  
统一结论：**所有计数以 PG17 为准，所有完成声称必须同时具备 SQL / API / Playwright 证据。**

---

## 1. 使用规则

1. **先文档 / SQL / 结构，再 API，再 Playwright。** 任一前置失败，都不得进入后续验收。
2. **所有计数一律回到 PG17。** 页面数字、shell 输出、日志摘要都不能替代数据库事实。
3. **主流程页无 real 数据时也必须验收。** 验的是诚实空状态，不是“有数据才测”。
4. **每一项检查都要落证据。** 至少记录：检查 ID、执行时间、结果、证据路径、是否阻断下一步。
5. **每个 Gate 都必须有停机动作。** 未定义“失败后怎么办”的检查项，不算可执行检查项。
6. **`trusted_loss_*` 在 G1 只验表结构，在 G2/G5 才验数据。** 不允许把 G2 的数据要求提前挪到 G1。
7. **compare 的 synthetic / fallback 只能在 compare 生效。** 任何主流程页面出现这两种来源都视为阻断级失败。

## 3. SQL 与结构核验

### 3.1 SQL 核验语句

以下 SQL 为执行草案。涉及 `$1` 的语句，执行时替换为真实 `run_id` 或 `batch_id`。

```sql
-- Q01: schema 就绪
SELECT COUNT(*) AS schema_count
FROM information_schema.schemata
WHERE schema_name IN ('rebuild4', 'rebuild4_meta');

-- Q02: 关键控制表存在性
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_schema = 'rebuild4_meta'
  AND table_name IN (
    'contract_version', 'rule_set_version', 'source_adapter', 'run', 'batch',
    'initialization_step_log', 'batch_snapshot', 'baseline_version',
    'gate_definition', 'gate_check_item', 'gate_run_result',
    'compare_job', 'compare_result', 'trusted_loss_summary', 'trusted_loss_breakdown'
  )
ORDER BY table_name;

-- Q03: 关键业务表存在性
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_schema = 'rebuild4'
  AND table_name IN (
    'fact_standardized', 'fact_governed', 'fact_pending_observation',
    'fact_pending_issue', 'fact_rejected', 'obj_cell', 'obj_bs', 'obj_lac',
    'baseline_cell', 'baseline_bs', 'baseline_lac'
  )
ORDER BY table_name;

-- Q04: field_audit / target_field snapshot 基线
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.field_audit_snapshot) AS field_audit_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.target_field_snapshot) AS target_field_rows;

-- Q05: ODS 定义 / 执行基线
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.ods_rule_snapshot WHERE is_active) AS ods_rule_rows,
  (SELECT COUNT(DISTINCT rule_code) FROM rebuild4_meta.ods_execution_snapshot) AS ods_execution_rule_codes;

-- Q06: ODS 差集
SELECT rule_code
FROM rebuild4_meta.ods_rule_snapshot
WHERE is_active
EXCEPT
SELECT DISTINCT rule_code
FROM rebuild4_meta.ods_execution_snapshot
ORDER BY rule_code;

-- Q07: parse / compliance 是否非空
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.parse_rule) AS parse_rule_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.compliance_rule) AS compliance_rule_rows;

-- Q08: asset 注册表是否非空
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.asset_table_catalog) AS asset_table_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_field_catalog) AS asset_field_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_usage_map) AS asset_usage_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_migration_decision) AS asset_migration_rows;

-- Q09: trusted_loss_* 表存在后的行数（G1 允许为 0，G2/G5 需要 > 0）
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.trusted_loss_summary) AS trusted_summary_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.trusted_loss_breakdown) AS trusted_breakdown_rows;

-- Q10: real full initialization run
SELECT run_id, run_type, status, data_origin
FROM rebuild4_meta.run
WHERE run_type = 'full_initialization'
ORDER BY created_at DESC;

-- Q11: full init logical batch 语义
SELECT run_id, COUNT(*) AS batch_count, MIN(batch_seq) AS min_seq, MAX(batch_seq) AS max_seq
FROM rebuild4_meta.batch
WHERE run_id = $1
GROUP BY run_id;

-- Q12: 四分流守恒
SELECT
  (SELECT COUNT(*) FROM rebuild4.fact_standardized WHERE batch_id = $1) AS standardized_rows,
  (SELECT COUNT(*) FROM rebuild4.fact_governed WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_pending_observation WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_pending_issue WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_rejected WHERE batch_id = $1) AS routed_rows;

-- Q13: 初始化 11 步
SELECT run_id, COUNT(*) AS step_count
FROM rebuild4_meta.initialization_step_log
WHERE run_id = $1
GROUP BY run_id;

-- Q14: baseline current 唯一性
SELECT COUNT(*) AS current_baseline_rows
FROM rebuild4_meta.baseline_version
WHERE is_current = true;

-- Q15: trusted loss 最新快照与 breakdown 类型
SELECT total_rows, trusted_rows, filtered_rows, filtered_pct, filtered_with_rsrp, filtered_with_gps
FROM rebuild4_meta.trusted_loss_summary
ORDER BY created_at DESC
LIMIT 1;

SELECT breakdown_type, COUNT(*) AS row_count
FROM rebuild4_meta.trusted_loss_breakdown
GROUP BY breakdown_type
ORDER BY breakdown_type;

-- Q16: rolling run 完成情况
SELECT run_id, run_type, status, data_origin
FROM rebuild4_meta.run
WHERE run_type = 'rolling'
ORDER BY created_at DESC;

-- Q17: completed rolling batch 数量
SELECT r.run_id, COUNT(*) AS completed_batches
FROM rebuild4_meta.batch b
JOIN rebuild4_meta.run r USING (run_id)
WHERE r.run_type = 'rolling'
  AND b.status = 'completed'
GROUP BY r.run_id
ORDER BY completed_batches DESC;

-- Q18: completed batch 的 11 metric 覆盖（期望 0 行）
SELECT b.batch_id, COUNT(s.*) AS snapshot_metrics
FROM rebuild4_meta.batch b
LEFT JOIN rebuild4_meta.batch_snapshot s ON s.batch_id = b.batch_id
WHERE b.status = 'completed'
GROUP BY b.batch_id
HAVING COUNT(s.*) != 11
ORDER BY b.batch_id;

-- Q19: completed batch 的 flow summary / refresh log 覆盖
SELECT
  b.batch_id,
  COUNT(DISTINCT f.fact_layer) AS flow_layers,
  COUNT(DISTINCT r.baseline_version) AS refresh_rows
FROM rebuild4_meta.batch b
LEFT JOIN rebuild4_meta.batch_flow_summary f ON f.batch_id = b.batch_id
LEFT JOIN rebuild4_meta.baseline_refresh_log r ON r.batch_id = b.batch_id
WHERE b.status = 'completed'
GROUP BY b.batch_id
ORDER BY b.batch_id;

-- Q20: baseline 历史版本链
SELECT baseline_version, run_id, batch_id, is_current, created_at
FROM rebuild4_meta.baseline_version
ORDER BY created_at DESC;
```

### 3.2 `PG17 search_objects` 结构检查

结构检查至少执行以下 3 项：

1. `search_objects(schema='rebuild4_meta', object_type='table')`：确认控制层、Gate、compare、trusted_loss 表全部存在。
2. `search_objects(schema='rebuild4', object_type='table')`：确认 `fact_standardized`、四分流、`obj_*`、`baseline_*` 全部存在。
3. `search_objects(schema='rebuild4_meta', object_type='column', table='batch_snapshot')`：确认 `stage_name`、`metric_name`、`metric_value`、`created_at` 等关键字段齐全。
