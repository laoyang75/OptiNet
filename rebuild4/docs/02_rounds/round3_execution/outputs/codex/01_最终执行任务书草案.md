状态：codex 独立草案

## 1. 执行结论

rebuild4 可以进入实施阶段，但 **不能直接并行开工“全量 DDL + 全量 API + 全量前端”**。  
本轮应按 **G0 -> G7** 的停机线串行推进，先把 `rebuild4` / `rebuild4_meta`、governance 元数据承接、real initialization、real rolling 和来源合同落地，再做页面联调。

本草案冻结 6 条执行结论：

1. **唯一执行入口先冻结，再写代码。**
   - round3 合并完成后，只认 `rebuild4/docs/03_final/01_rebuild4_最终执行任务书.md` 与 `rebuild4/docs/03_final/02_rebuild4_最终校验清单.md`。
2. **正式主链必须从 `rebuild2.l0_gps + rebuild2.l0_lac` 重跑。**
   - `rebuild3*` 只作为参考，不作为 rebuild4 正式主链写入源。
3. **主流程页、对象页、baseline、画像、initialization、governance 只允许 `real`。**
   - 没有 real，就进入诚实空状态；不得再用 fallback 伪装“已有结果”。
4. **compare 继续保留，但严格降级为辅助验证模块。**
   - 允许 `synthetic`/`fallback`，但必须显式 banner，且不计正式完成标准。
5. **先做元数据承接，再做 real run，再做 API，再做页面。**
   - 执行顺序不得倒置，否则会再次落入“页面先渲染、语义后补齐”的 rebuild3 偏航路径。
6. **所有步骤必须同时具备执行动作、验收动作、回退动作。**
   - 任一步骤未给出 SQL/API/Playwright 验证或未给出回退方式，视为任务书未完成。

**验证**

- 只回退失败模块，不回退已通过 Gate 的历史证据

## 5. 执行约束与禁止事项

1. `rebuild2` / `rebuild2_meta` / `rebuild3*` 只读，不允许把旧结果回写成 rebuild4 正式当前态。
2. `fallback` 不得进入主流程页；主流程页无 real 数据时必须空状态。
3. 不允许在 `batch_snapshot`、四分流名称、`run_type`、`data_origin` 上做静默扩容。
4. 不允许把 UI mock 数字写入正式 API 或正式页面。
5. 页面联调前必须先通过 API 来源合同；不允许“先把页面跑起来再补 `data_origin`”。
6. `classification_v2`、`gps_confidence`、`signal_confidence` 不得回到对象主状态，只能作为 `legacy_ref`。
7. launcher / 运维健康页不计入本轮正式完成标准，但可以独立存在。
8. compare 只做验证；不允许反向写回主链表，也不允许据此判断主流程已完成。

---

## 6. 完成定义（Definition of Done）

只有同时满足下面 8 条，rebuild4 才算完成首轮正式交付：

1. `rebuild4` / `rebuild4_meta` 结构完整、约束生效、Gate 可执行。
2. governance 快照与 `parse_rule` / `compliance_rule` 已正式落表且可查询。
3. 至少 1 条 completed real `full_initialization` 与 baseline v1 已生成。
4. 至少 1 条 completed real `rolling` run 且 completed batch > 1。
5. 主流程 API 全部返回统一来源 envelope，主流程页只消费 real 或诚实空状态。
6. baseline 历史与差异接口诚实可追溯；仅 1 个 baseline 时明确 `previous_available=false`。
7. governance 正式可用；compare 明确降级但不阻断主流程。
8. SQL、API、Playwright 三类证据齐备，`03_final` 文档冻结完成。
