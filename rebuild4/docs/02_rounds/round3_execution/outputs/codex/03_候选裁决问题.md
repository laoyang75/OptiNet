## 4. 仅在以下异常情况下再上抛

如果实施中出现下面 3 类异常，再单独生成人类裁决文件：

1. **规则 seed 无法产生非空结果。**
   - 例如 `parse_rule` / `compliance_rule` 从文档、SQL、Python 提取后仍无法形成可审计正式表。
2. **real rolling 输入源客观缺失。**
   - 例如无法获得满足 `completed batch > 1` 的 rolling 数据窗口，导致 `G3` 无法通过。
3. **主流程页出现新的主语冲突。**
   - 例如同一页面同时需要 `run` 与 `batch` 作为一级主语，且无法通过矩阵唯一化。

在未出现上述异常前，本轮建议按 `01_最终执行任务书草案.md` 与 `02_逐步校验与验收清单草案.md` 直接执行，不新增人类裁决环节。
