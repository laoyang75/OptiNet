# 逐步校验与验收清单草案

状态：codex 独立草案  
生成时间：2026-04-06  
适用范围：rebuild4 从 Phase 0 到最终冻结的逐步校验、验收、停机与回退。  
强制工具：数据库事实用 `PG17 MCP execute_sql`；结构确认用 `PG17 MCP search_objects`；页面验收用 Playwright。

---

## 1. 使用规则

1. **先 SQL / 结构，再 API，再 Playwright。** 任一步骤如果数据库事实未通过，不得进入页面验收。
2. **所有计数一律以 PG17 为准。** shell 输出、页面数字、日志摘要都不能替代数据库事实。
3. **Playwright 只验“已通过前置 Gate 的页面”。** 主流程页没有 real 数据时，应验空状态，而不是跳过。
4. **每一项检查都要落证据。** 建议最少记录：检查 ID、执行人、时间、结果、证据路径、是否阻断下一步。
5. **回退动作必须和检查项一一对应。** 未写回退方案的步骤，不得算“可执行”。

## 3. SQL 核验语句

以下 SQL 为执行草案。涉及 `$1` 的语句，执行时替换为真实 `batch_id` 或 `run_id`。

```sql
-- Q01: schema 就绪
SELECT COUNT(*) AS schema_count
FROM information_schema.schemata
WHERE schema_name IN ('rebuild4', 'rebuild4_meta');

-- Q02: 关键控制表存在性
SELECT table_schema, table_name
FROM information_schema.tables
WHERE table_schema = 'rebuild4_meta'
  AND table_name IN (
    'contract_version', 'rule_set_version', 'source_adapter', 'run', 'batch',
    'batch_snapshot', 'baseline_version', 'gate_definition', 'gate_check_item', 'gate_run_result'
  )
ORDER BY table_name;

-- Q03: `batch_snapshot` 关键字段检查
SELECT column_name
FROM information_schema.columns
WHERE table_schema = 'rebuild4_meta'
  AND table_name = 'batch_snapshot'
ORDER BY ordinal_position;

-- Q04: field_audit / target_field snapshot 基线
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.field_audit_snapshot) AS field_audit_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.target_field_snapshot) AS target_field_rows;

-- Q05: ODS 定义 / 执行基线
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.ods_rule_snapshot WHERE is_active) AS ods_rule_rows,
  (SELECT COUNT(DISTINCT rule_code) FROM rebuild4_meta.ods_execution_snapshot) AS ods_execution_rule_codes;

-- Q06: ODS 差集
SELECT rule_code
FROM rebuild4_meta.ods_rule_snapshot
WHERE is_active
EXCEPT
SELECT DISTINCT rule_code
FROM rebuild4_meta.ods_execution_snapshot
ORDER BY rule_code;

-- Q07: parse / compliance 是否非空
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.parse_rule) AS parse_rule_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.compliance_rule) AS compliance_rule_rows;

-- Q08: trusted 损耗 summary 基线
SELECT total_rows, trusted_rows, filtered_rows, filtered_pct, filtered_with_rsrp, filtered_with_gps
FROM rebuild4_meta.trusted_loss_summary
ORDER BY created_at DESC
LIMIT 1;

-- Q09: real full initialization run
SELECT run_id, run_type, status, data_origin
FROM rebuild4_meta.run
WHERE run_type = 'full_initialization'
ORDER BY created_at DESC;

-- Q10: full init logical batch 语义
SELECT run_id, COUNT(*) AS batch_count, MIN(batch_seq) AS min_seq, MAX(batch_seq) AS max_seq
FROM rebuild4_meta.batch
WHERE run_id = $1
GROUP BY run_id;

-- Q11: 四分流守恒
SELECT
  (SELECT COUNT(*) FROM rebuild4.fact_standardized WHERE batch_id = $1) AS standardized_rows,
  (SELECT COUNT(*) FROM rebuild4.fact_governed WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_pending_observation WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_pending_issue WHERE batch_id = $1)
+ (SELECT COUNT(*) FROM rebuild4.fact_rejected WHERE batch_id = $1) AS routed_rows;

-- Q12: 初始化 11 步
SELECT run_id, COUNT(*) AS step_count
FROM rebuild4_meta.initialization_step_log
WHERE run_id = $1
GROUP BY run_id;

-- Q13: baseline current 唯一性
SELECT COUNT(*) AS current_baseline_rows
FROM rebuild4_meta.baseline_version
WHERE is_current = true;

-- Q14: rolling run 完成情况
SELECT run_id, run_type, status, data_origin
FROM rebuild4_meta.run
WHERE run_type = 'rolling'
ORDER BY created_at DESC;

-- Q15: completed rolling batch 数量
SELECT r.run_id, COUNT(*) AS completed_batches
FROM rebuild4_meta.batch b
JOIN rebuild4_meta.run r USING (run_id)
WHERE r.run_type = 'rolling'
  AND b.status = 'completed'
GROUP BY r.run_id
ORDER BY completed_batches DESC;

-- Q16: completed batch 的 11 metric 覆盖
SELECT b.batch_id, COUNT(*) AS snapshot_metrics
FROM rebuild4_meta.batch b
LEFT JOIN rebuild4_meta.batch_snapshot s ON s.batch_id = b.batch_id
WHERE b.status = 'completed'
GROUP BY b.batch_id
ORDER BY b.batch_id;

-- Q17: governance 核心表非空检查
SELECT
  (SELECT COUNT(*) FROM rebuild4_meta.asset_table_catalog) AS asset_table_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_field_catalog) AS asset_field_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_usage_map) AS asset_usage_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.asset_migration_decision) AS asset_migration_rows,
  (SELECT COUNT(*) FROM rebuild4_meta.trusted_loss_breakdown) AS trusted_breakdown_rows;

-- Q18: baseline 历史版本链
SELECT baseline_version, run_id, batch_id, is_current, created_at
FROM rebuild4_meta.baseline_version
ORDER BY created_at DESC;
```

## 4. API 验收动作

| 编号 | API | 关键断言 |
|---|---|---|
| A01 | `GET /api/flow-overview` | 顶层 envelope 完整；`subject_scope=batch`；四分流字段使用全称；主流程仅 `real` 或诚实空状态 |
| A02 | `GET /api/flow-snapshot/timepoints` | 只列 real completed batch；不混入 synthetic scenario |
| A03 | `GET /api/flow-snapshot` | 只返回 4 stage / 11 metric；`context.batch_id` 非空 |
| A04 | `GET /api/runs/current` | `run_type` 只能是 `full_initialization / rolling / scenario_replay` |
| A05 | `GET /api/batches` + `GET /api/batches/:id/detail` | 列表按 run/batch 组织；`batch_seq` 单调；异常/刷新/transition 明细分开 |
| A06 | `GET /api/objects` + `GET /api/objects/:key/detail` | 返回 `lifecycle_state`、`health_state`、资格字段；旧字段只在 `legacy_ref` |
| A07 | `GET /api/observation-workspace` | 必含存在/锚点/基线三段式进度或等价结构；不能只给单一 10/2/3 阈值 |
| A08 | `GET /api/anomaly-workspace` | object/record 双视角 payload 分开；含 `severity`、`downstream_impact` |
| A09 | `GET /api/baseline/current`、`/current/diff`、`/history` | 仅 1 个 baseline 时 `previous_available=false`；current 唯一 |
| A10 | `GET /api/profiles/lac|bs|cell` | 主状态使用新模型；旧分类/可信度只走 `legacy_ref` 或参考字段 |
| A11 | `GET /api/initialization/latest`、`/initialization/:run_id` | 返回 real full init；11 步详情来自正式表，不得输出 mock 数字 |
| A12 | `GET /api/governance/*` 与 `GET /api/validation/compare` | governance 多 endpoint 可用；compare 只做辅助验证，`fallback` 不返回伪结果 |
