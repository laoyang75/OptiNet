SELECT COUNT(*) FROM rebuild4_meta.run
WHERE run_type = 'full_initialization' AND status = 'completed' AND data_origin = 'real';

  (SELECT COUNT(*) FROM rebuild4.fact_governed WHERE batch_id = $1)

### Phase 2B：real rolling（M1 后半部分）

**输入**：上一 Phase 产出的 baseline + 新增数据窗口。

**完成标准**：Gate G3 通过。

**动作**：
1. 创建 1 条 `run_type = rolling`、`data_origin = real` 的 run。
2. 在该 run 下完成 > 1 个 completed batch。
3. 每个 completed batch 写入 11 个 snapshot metric + flow summary + transition summary + anomaly summary + baseline_refresh_log。
4. 至少产生 1 次 baseline 刷新（或诚实记录未刷新原因）。

**Gate G3 SQL 校验**：
```sql
-- completed rolling run
SELECT COUNT(*) FROM rebuild4_meta.run
WHERE run_type = 'rolling' AND status = 'completed';
-- 期望: >= 1

-- completed batch > 1
SELECT COUNT(*) FROM rebuild4_meta.batch
WHERE run_id = $1 AND status = 'completed';
-- 期望: > 1

-- 每个 completed batch 都有 11 个 snapshot metric
SELECT b.batch_id, COUNT(s.*) AS metric_count
FROM rebuild4_meta.batch b
LEFT JOIN rebuild4_meta.batch_snapshot s ON s.batch_id = b.batch_id
WHERE b.status = 'completed'
GROUP BY b.batch_id
HAVING COUNT(s.*) != 11;
-- 期望: 0 行（即所有 completed batch 都恰好 11 个 metric）
```

```json
{
  "data_origin": "real | synthetic | fallback",
  "origin_detail": "run_id=RUN-..., batch_id=BATCH-...",
  "subject_scope": "batch | run | object | baseline | governance | compare | initialization",
  "subject_note": null,

    "contract_version": "rebuild4-contract-v1",
    "rule_set_version": "rebuild4-rule-v1"
  },
  "data": {}
}
```

### 4.2 来源规则

| 页面层 | 允许 data_origin | 备注 |
|---|---|---|
| 主流程层 + 对象层 + baseline + 画像 + initialization + governance | `real` | 没有 real 就诚实空状态 |
| compare | `real` / `synthetic` | synthetic 必须 banner |
| compare 降级空响应 | `fallback` | `data = null`，不伪造结果 |

## 8. 关键约束总结

1. **唯一执行入口**：冻结后只有 `rebuild4/docs/03_final/01_rebuild4_最终执行任务书.md` 与 `02_rebuild4_最终校验清单.md` 具备执行效力。
2. **数据起点**：从 `rebuild2.l0_gps` + `rebuild2.l0_lac` 重跑（D-001 裁决 A）。
3. **前端起点**：全新 Vue 3 + TS + Vite 项目（D-002 裁决 C）。
4. **ODS 展示**：定义层 26 条与执行层 24 条分离为独立 API（D-003 裁决 C）。
5. **来源语义**：`data_origin` / `origin_detail` / `subject_scope` 必须出现在所有核心 API 顶层。
6. **禁止 fallback 伪装**：主流程页无 real 数据时必须诚实空状态。
7. **Gate 是停机线**：未通过的 Gate 必须阻断后续 Phase，不得绕过。
8. **legacy 字段降级**：`classification_v2` / `gps_confidence` / `signal_confidence` 只进 `legacy_ref`，不进对象主状态。
