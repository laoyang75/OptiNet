# Finalization - 共享约束

## 1. 任务模式

本轮采用“同题独立冻结编制”。

- Codex / Claude / Gemini 做的是同一个任务
- 目标不是重新讨论方向，而是检查前三轮结果与裁决是否真正一致，并形成最终冻结基线
- 输出必须使用完全相同的文件名

## 2. 唯一输入范围

只允许使用：

- `rebuild4/docs/00_runbook/`
- `rebuild4/docs/01_inputs/`
- `rebuild4/docs/02_rounds/round1_plan/merged/*`
- `rebuild4/docs/02_rounds/round2_detail/merged/*`
- `rebuild4/docs/02_rounds/round3_execution/merged/*`
- `rebuild4/docs/02_rounds/*/decisions/*`
- `rebuild4/docs/02_rounds/round3_execution/outputs/codex/` 中已经保存的截图和页面证据
- `PG17 MCP`

禁止：

- 重新访问页面
- 重新发明新范围或新方向
- 忽略已裁决结果
- 把 `99_archive/` 当当前主真相源

## 3. 数据库事实要求

- 所有计数、比例、差异、缺口，都必须通过 `PG17 MCP execute_sql` 验证
- `search_objects` 只可用于确认表结构、字段、索引
- 不允许引用 shell 输出代替数据库事实

## 4. 页面规则

本轮不重新浏览网页。

如果需要页面证据，只能引用：

- round3 merged 的页面结论
- round3 codex 输出目录下的截图与证据说明

## 5. 输出硬约束

必须产出：

- `01_最终冻结基线草案.md`
- `02_最终范围技术栈与数据策略草案.md`
- `03_候选裁决问题.md`

每份文档都必须：

- 讲清结论
- 讲清依据
- 讲清哪些裁决已回写
- 讲清仍存在的残留缺口
- 不得把未验证推测写成事实
