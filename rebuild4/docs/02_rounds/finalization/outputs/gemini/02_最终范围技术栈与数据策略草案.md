# 02_最终范围技术栈与数据策略草案

## 结论
技术栈冻结为前后端彻底分离的新架构（Vue 3 + Vite，严格禁止从 rebuild3 拷贝核心逻辑，后端统一依赖 PG17 和 RESTful JSON API）。启动器与调度框架不介入核心数据流控制，需明确划定为底层基础能力。数据生成策略彻底抛弃手动 mock 与 `scenario_replay`，必须通过 rebuild4 的 `full_initialization` 与 `rolling` 主链产出 real 数据。

## 依据（文件路径 / 截图路径 / PG17）
- **文件路径**：`rebuild4/docs/02_rounds/round3_execution/merged/01_统一最终执行任务书.md` 中的 WP3，WP4，WP6；`round3_execution/decisions/01_待裁决问题清单.md` 的 D-001/D-002 裁决。
- **截图路径**：P-E08（11步初始化流程页面证据，必须由真实数据取代原先的 mock 数据）。
- **PG17**：执行 `SELECT COUNT(*) FROM rebuild3.fact_standardized` 确保对旧体量的心智掌握；查询 `field_audit` 的 17/3/7 分布，确认必须使用真实数据库作为唯一来源而非人工补录的台账。

## 风险
- 开发时违背“禁止从 rebuild3 前端 fork”的裁决，直接拷贝带隐含 fallback 逻辑的代码，导致旧系统幽灵逻辑在 rebuild4 中浮现。
- 把启动器及任务编排当作具体业务功能，使得基础设施逻辑与主链数据生成严重耦合。

## 校验
- 启动项目时检查 `package.json`，确保它是全新基于 Vite 创建的 Vue 3 项目，无任何无用的历史依赖。
- 通过 `G2_REAL_INITIALIZATION_READY` 的数据库探针验证 `data_origin=real` 和 1 个 logical batch 是否生成成功，且 11 步日志必须写入 `rebuild4_meta.initialization_step_log`。

## 若失败如何处理
- 如果前端代码审计发现包含非纯展示的 rebuild3 代码（如自带 fetch 的组件），立即拒绝 PR，要求按 UI_v2 重写。
- 如果没有产出 baseline v1 及完整的 11 步日志，直接阻塞并禁止进入 Phase 2B (real rolling) 的开发和验证。

### 技术栈冻结
- **前端**：全新 Vue 3 + TypeScript + Vite 架构。必须重写路由、Pinia Store、Axios 请求层。纯展示组件需经 UI_v2 审计后复用。
- **数据库**：PG17。schema 必须新建 `rebuild4` 和 `rebuild4_meta`。禁止在旧 `rebuild3` 上直接迭代。
- **后端**：严格遵循带 `envelope`（包含 `data_origin`，`origin_detail`，`subject_scope`，`context` 等）的 RESTful JSON API 标准。

### 启动器/基础框架约束
- **基础角色**：作为纯基础设施，提供运维级别的接口和健康检查（Health Check）。
- **边界划分**：启动器不纳入 rebuild4 核心数据流业务逻辑，不出现在 API 统一来源合同中，也不受 G2-G7 Gate 中关于真实主语的约束。
- **强制拦截**：基础框架必须实现强制 `envelope` 的拦截器，无标准 envelope 输出的 API 直接报错。

### 数据生成与回灌策略如何进入最终任务书
- **数据起点（唯一真理）**：基于 `rebuild2.l0_gps` (38,433,729) 与 `rebuild2.l0_lac` (43,771,306) 重跑。
- **数据回灌/生成**：禁止通过手动 mock 脚本或 `scenario_replay` 伪装数据。必须严格执行 WP3 规定的 11 步初始化流程，跑通 real 的 `full_initialization`（包含事实清洗、对象派生、异常判定、Baseline 生成等）。
- **Baseline 链刷新**：通过第一个 initial batch 强制生成 baseline v1，随后的 real rolling 必须基于上一版冻结的 baseline 进行 diff 并登记是否刷新新版本。

### 本轮范围与降级说明
- **正式核心范围**：主流程概览、批次运行列表、对象/对象详情、工作台（观察/异常）、Baseline 画像中心、Governance 基础数据治理。
- **明确降级**：`compare`（验证/对照）面板被显式降级为辅助验证页，不进入主干完成标准，必须带顶层 banner，且空响应仅使用 fallback。
- **明确排除/历史遗留**：旧的 `rebuild3` 和 `rebuild2` 数据只作为历史参照（以 `legacy_ref` 嵌套返回），不作为当前展示的核心状态；`classification_v2` 等旧字段退出核心模型；废弃 `rebuild3_sample_meta`。
