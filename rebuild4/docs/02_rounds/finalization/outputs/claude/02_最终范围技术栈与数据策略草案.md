## 3. 数据生成策略冻结

### 3.1 real `full_initialization` 策略

| 维度 | 冻结值 |
|---|---|
| 输入源 | `rebuild2.l0_gps`（38,433,729 行）+ `rebuild2.l0_lac`（43,771,306 行） |
| run_type | `full_initialization` |
| data_origin | `real` |
| 批次语义 | 1 个 logical run + 1 个 logical batch（batch_seq=1） |
| 步骤 | 11 步，每步写入 `initialization_step_log` |
| 输出 | fact_standardized + 四分流 + obj_* + batch_snapshot + batch_flow/transition/anomaly + baseline v1 + trusted_loss_* |
| 守恒约束 | fact_standardized = 四分流合计 |
| baseline | 首个 real baseline_version，is_current=true 唯一 |
| trusted loss | 首次产出 summary + breakdown（至少 tech、source_combo 两类） |
| checkpoint | 允许作为执行引擎内部机制，不改变对外批次语义 |

### 3.2 real `rolling` 策略

| 维度 | 冻结值 |
|---|---|
| run_type | `rolling` |
| data_origin | `real` |
| 完成标准 | 至少 1 条 completed run + completed batch > 1 |
| 每个 completed batch 输出 | batch_snapshot（11 metric）+ batch_flow_summary + batch_transition_summary + batch_anomaly_* + baseline_refresh_log |
| baseline 时序 | 当前批只读取上一版冻结 baseline；新版仅供下一批使用 |
| timepoints | 只列 real completed batch；不混入 scenario_replay |
| 窗口切分 | 实施细节，由实施者决定；只要满足 data_origin=real + G3 通过 |

### 3.3 baseline 策略

| 维度 | 冻结值 |
|---|---|
| 存储模式 | 按 baseline_version 追加存储，主键 (baseline_version, object_id) |
| current view | 可以是 view，但正式历史从实体表读 |
| diff | baseline_diff_summary + baseline_diff_object |
| 诚实规则 | 仅 1 个 baseline 时 previous_available=false + subject_note 明示 |
| 禁止 | 用 rebuild2/rebuild3/synthetic baseline 冒充上一版 |

### 3.4 governance 数据策略

| 数据源 | 策略 | 目标表 |
|---|---|---|
| `rebuild2_meta.field_audit`（27 行） | 一次性快照冻结 | `rebuild4_meta.field_audit_snapshot` |
| `rebuild2_meta.target_field`（55 行） | 一次性快照冻结 | `rebuild4_meta.target_field_snapshot` |
| active `ods_clean_rule`（26 条） | 一次性快照冻结 | `rebuild4_meta.ods_rule_snapshot` |

| parse_rule | 从文档/SQL/Python 提取 seed | `rebuild4_meta.parse_rule` |

5. **建议直接进入最终冻结合并，无需新增人类裁决。**
