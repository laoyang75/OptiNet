# Step 02: 流式画像 vs rebuild2 批量画像偏差评估

## 核心概念

### 增量流程（从 rebuild3 §7 冻结）

每个批次（本实验中 = 每天）执行以下步骤：

```
1. 标准化事件（etl_filled 中当天数据）
2. 查对象注册表：已知 cell_id → 累积证据；未知 → 注册
3. 四分流（简化版，本实验只关注前三路）：
   - 已知且健康 → 正常累积
   - 已知但异常 → 标记待复核
   - 未知但合规 → 注册为 waiting
   - 结构不合规 → 拒绝
4. 批末晋升：根据累积数据判定 lifecycle 状态转换
5. 级联更新 BS 和 LAC
```

**关键原则**：批次内不改状态，批末统一更新。

### 流式处理的对象生命周期

```
Day 1: 新 cell_id 首次出现 → 注册，lifecycle = waiting
       ↓ 数据累积
Day N: 达到门槛 → 晋升到 observing
       ↓ 继续累积
Day M: 满足全部条件 → 晋升到 active
```

### 三层资格门槛（从 rebuild3 §9 + docs/03_final/02_数据生成与回灌策略.md）

| 层级 | 资格 | 条件 |
|------|------|------|
| L1 | existence_eligible | 首次出现可解析的 (operator, lac, bs_id, cell_id) |
| L2 | anchorable | GPS 点 >= 10, 设备 >= 2, P90 < 1500m, 观测跨度 >= 24h, 无禁锚异常 |
| L3 | baseline_eligible | 已 anchorable + 无禁基线异常 |

### 生命周期状态转换（从 docs/02_profile/01_cell.md）

| 状态 | 条件 |
|------|------|
| waiting | obs < 3 或 设备 < 2 |
| observing | 基本条件满足但不满足 active |
| active | obs >= 3, 设备 >= 2, P90 < 1500m, 观测跨度 >= 24h, 非碰撞 |

### Cell 画像核心算法（从 docs/02_profile/04_cell_research.md）

- **独立观测点**: `(cell_id, minute)` 去重，不含设备维度
- **质心**: 中位数（PERCENTILE_CONT 0.5），不用 AVG
- **漂移分类**: 基于日质心的 net_drift / max_spread 比率
  - collision: > 2.2km, ratio < 0.3（碰撞门槛 2.2km，有天然断层）
  - stable: max_spread < 500m
  - 取消 low_collision，< 2.2km 的归为 large_coverage

## 实验步骤

### Phase 1: 准备（只读，不修改现有表）

1. 读取 rebuild2 基准表字段结构，确定可对比字段（质心、P90、设备数、生命周期等）
2. 确认 etl_filled 的每日数据分布
3. 确定 rebuild2 的 cell_id 和当前 etl_filled 的 cell_id 匹配方式

### Phase 2: 流式模拟

对 N = 1, 2, ..., 7，用前 N 天的累积数据构建完整画像：

```
eval_stream_cell_day1  ← 只用 Day 1
eval_stream_cell_day2  ← 用 Day 1 + Day 2
...
eval_stream_cell_day7  ← 用 Day 1-7（应接近批量结果）
```

每个快照必须执行完整的画像流程：
1. 独立观测点去重
2. 中位数质心计算
3. P50/P90 半径
4. 信号统计
5. 生命周期状态判定
6. 漂移检测（Day 1-2 可能数据不足）

**重要**：不需要为每天都建物理表。可以用 CTE 或临时表完成，只保留对比结果。

### Phase 3: 对比分析

将每天的流式快照与 rebuild2 基准对比：

#### Cell 级（核心）

| 对比维度 | 计算方法 |
|----------|---------|
| 数量覆盖率 | 流式 Cell 数 / rebuild2 Cell 数 |
| 质心偏差 | 匹配 Cell 的质心距离（米），取中位数和 P90 |
| 生命周期一致率 | 流式 active 的 Cell 在 rebuild2 中也是 active 的比例 |
| P90 偏差 | 流式 P90 / rebuild2 P90 的比值分布 |

#### BS 级

| 对比维度 | 计算方法 |
|----------|---------|
| 数量覆盖率 | 流式 BS 数 / rebuild2 BS 数 |
| 质心偏差 | 匹配 BS 的质心距离（米） |

### Phase 4: 收敛曲线

输出每天的核心指标变化趋势：

```
Day | Cell 覆盖率 | 质心中位偏差 | Active 一致率 | BS 覆盖率
 1  |    ???     |    ???m    |    ???%     |   ???
 2  |    ???     |    ???m    |    ???%     |   ???
...
 7  |    ???     |    ???m    |    ???%     |   ???
```

### Phase 5: 结论

根据收敛曲线给出判断：

- **可以直接流式**：Day 3-4 后核心指标收敛，偏差在可接受范围
- **需要轻量初始化**：收敛慢，建议用 N 天批量做种子再流式续跑
- **偏差来源分析**：哪些类型的 Cell 偏差最大（低观测、碰撞、边缘 Cell）
