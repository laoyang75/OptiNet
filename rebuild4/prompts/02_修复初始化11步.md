## ETL 可复用模块设计

ETL 模块需要支持两种调用模式：
1. **初始化**：处理全量 L0 数据 → fact_standardized（scope 决定数据范围）
2. **滚动批次**：处理增量数据 → 追加到 fact_standardized，更新 Cell 累积指标

### 关键接口
```
输入：batch_id + scope + 数据范围（时间窗口或全量）
处理：JSON 解析 → ODS 清洗 → 标准化字段映射
输出：写入 fact_standardized，返回处理统计（input/output/rejected）
副作用：写入 pipeline_run 记录
```

### Cell 累积指标更新
每次 ETL 完成后，更新对应 Cell 的累积指标：
- gps_valid_count += 本批 GPS 有效数
- distinct_dev_id = 历史累计去重设备数
- record_count += 本批记录数
- observed_span_hours = MAX(event_time) - MIN(event_time) 历史最大跨度
- active_days = 历史累计活跃天数

当累积指标达到阈值时，触发晋升规则（waiting → observing → active）。

### 已有代码
- `backend/app/etl/__init__.py` + `pipeline.py`：初始框架，包含 ODS 规则函数和 Cell 累积更新 SQL
- 需要补充：实际的批量 SQL ETL 逻辑、与初始化 11 步的集成

## 11 步定义（来自冻结文档）

| step_seq | step_code | step_name | 需要的计数 | 复用模块 |
|---|---|---|---|---|
| 1 | BIND_VERSION | 冻结版本绑定 | 无 input/output | - |
| 2 | STANDARDIZE | 标准化事件 | input=L0 记录, output=fact_standardized | **ETL 模块** |
| 3 | COMPLIANCE_CHECK | 合规检查 | input=fact_standardized, output=通过数 | ODS 规则 |
| 4 | LAC_BOUNDARY | LAC 预边界 | input=distinct LAC, output=trusted LAC | 可信库 |
| 5 | CELL_PROMOTE | Cell 晋升 | input=total Cell, output=active Cell | Cell 累积规则 |
| 6 | DERIVE_BS | 派生 BS | input=active Cell, output=obj_bs | BS 聚合 |
| 7 | DERIVE_LAC | 派生 LAC | input=active BS, output=obj_lac | LAC 聚合 |
| 8 | GOVERNANCE | 事实治理 | input=fact_standardized, output=fact_governed | 四分流 |
| 9 | ANOMALY_DETECT | 异常检测 | input=active objects, output=anomaly count | - |
| 10 | REGRESSION | 回归 | input/output 对象总数 | - |
| 11 | BASELINE_V1 | 基线 v1 | input=baseline_eligible, output=baseline objects | - |

## 修复方法

### 2.1 确保列存在

```sql
ALTER TABLE rebuild4_meta.initialization_step_log 
  ADD COLUMN IF NOT EXISTS input_count BIGINT,
  ADD COLUMN IF NOT EXISTS output_count BIGINT,
  ADD COLUMN IF NOT EXISTS pass_rate NUMERIC;
```

### 2.2 从样本数据反算每步计数

每步的计数从已完成的 Step 00/01a/01b 的结果表中 COUNT：

```sql
-- Step 2: STANDARDIZE (复用 ETL)
-- input = sample_l0_gps(214865) + sample_l0_lac(206097)
-- output = 样本范围 fact_standardized 记录数

-- Step 5: CELL_PROMOTE (复用 Cell 累积规则)
-- input = 样本范围 obj_cell 总数
-- output = 样本范围 lifecycle_state='active' 的 Cell 数

-- Step 6: DERIVE_BS
-- input = 样本范围 active Cell 数
-- output = 样本范围 obj_bs 数（有 active Cell 的）

-- Step 8: GOVERNANCE (四分流)
-- input = 样本范围 fact_standardized
-- output = 样本范围 fact_governed
```

### 2.3 与 rebuild2 对比

```sql
-- 对比初始化链路关键数字
-- rebuild2 的完整链路：l0_gps(38M) + l0_lac(43M) → trusted(30M) → enriched(30M)
-- rebuild4 的完整链路：fact_standardized(82M) → four-way routing
-- 按样本 LAC 对比关键比例是否一致
SELECT 
  'r4_standardized' as layer, COUNT(*) as cnt
FROM rebuild4.fact_standardized
WHERE (operator_code = '46000' AND lac IN ('4176','2097233'))
   OR (operator_code = '46001' AND lac IN ('6402','73733'))
   OR (operator_code = '46011' AND lac IN ('6411','405512'))
UNION ALL
SELECT 'r2_l0_gps', COUNT(*) FROM rebuild4.sample_l0_gps
UNION ALL
SELECT 'r2_l0_lac', COUNT(*) FROM rebuild4.sample_l0_lac
UNION ALL
SELECT 'r2_enriched', COUNT(*) FROM rebuild4.sample_enriched;
```
