# Step 05: 修复基线与画像（样本模式）

## 前置条件
- `00`-`04` 已完成
- 阅读 `99_系统上下文.md`

## 工具要求
- 你必须使用 `PG17 MCP` 执行所有 SQL
- 你必须使用 Playwright 检查页面

## 目标
baseline_cell/bs/lac 有真实的 GPS 中心坐标和 radius，画像页面数据完整。
在样本模式下，仅处理 6 个样本 LAC 范围内的基线和画像。

## 修复内容

### 6.1 baseline_cell
- lon/lat 来自 obj_cell.gps_center_lon/lat
- radius_m 来自 obj_cell.p90_radius_m
- record_count 来自 obj_cell.record_count
- 只包含 baseline_eligible = true 的 Cell

### 6.2 baseline_bs
- lon/lat 来自 obj_bs.gps_center_lon/lat
- cell_count 来自 obj_bs.active_cell_count

### 6.3 baseline_lac
- lon/lat 来自 obj_lac.gps_center_lon/lat
- bs_count 来自 obj_lac.active_bs_count

### 6.4 画像页面数据补全
- Cell 画像：记录数、P90、GPS/信号率、事实路由分布
- BS 画像：P50/P90、分类、面积、GPS/信号率
- LAC 画像：位置名称、Cell/BS 数、面积、异常 BS 占比

### 6.5 画像展开行的口径说明
每个画像的展开行需要显示灰色口径说明块。

## 验证

对比 rebuild4 基线数据与 rebuild2 画像：
```sql
-- 基线 Cell 的坐标应与 rebuild2 画像一致
SELECT bc.operator_code, bc.lac, bc.cell_id,
  bc.lon, bc.lat, bc.radius_m,
  r2.gps_center_lon, r2.gps_center_lat, r2.gps_p90_dist_m
FROM rebuild4.baseline_cell bc
JOIN rebuild4.sample_cell_profile r2 
  ON bc.operator_code = r2.operator_code AND bc.lac = r2.lac AND bc.cell_id = r2.cell_id::text
WHERE (bc.operator_code = '46000' AND bc.lac IN ('4176','2097233'))
LIMIT 20;
```

用 Playwright 截图确认画像页面展示正确。
