## 修复内容

### 4.1 检查样本范围的分流分布

```sql
-- 样本范围在各分流表中的分布
SELECT 'fact_governed' as tbl, COUNT(*) FROM rebuild4.fact_governed
WHERE (operator_code = '46000' AND lac IN ('4176','2097233'))
   OR (operator_code = '46001' AND lac IN ('6402','73733'))
   OR (operator_code = '46011' AND lac IN ('6411','405512'))
UNION ALL
SELECT 'fact_pending_observation', COUNT(*) FROM rebuild4.fact_pending_observation
WHERE ... -- 同样条件
UNION ALL
SELECT 'fact_pending_issue', COUNT(*) FROM rebuild4.fact_pending_issue
WHERE ...
UNION ALL
SELECT 'fact_rejected', COUNT(*) FROM rebuild4.fact_rejected
WHERE ...;
```

### 4.2 分流逻辑对齐

按冻结文档，分流逻辑应该是：
- **fact_governed**: 对象 active + anchorable + 无异常 → 记录进入治理事实
- **fact_pending_observation**: 对象 waiting/observing → 记录等待观察
- **fact_pending_issue**: 对象有异常 → 记录标记为问题
- **fact_rejected**: 合规检查不通过 → 拒收

对样本范围的事实，根据其所属 Cell 的 lifecycle_state/health_state 重新判定分流。

### 4.3 更新 batch_flow_summary

确保样本范围内的 batch flow_summary 反映真实计数。
