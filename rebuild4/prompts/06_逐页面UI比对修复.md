# Step 06: 逐页面 UI 比对修复（样本模式）

## 前置条件
- `00`-`05` 全部完成，样本数据层已修复
- 阅读 `99_系统上下文.md`

## 工具要求
- 你必须使用 `PG17 MCP` 执行所有 SQL
- 你必须使用 Playwright 检查页面

## 目标
每个页面与 rebuild3 UI (http://127.0.0.1:47122) 并排对比，确保结构一致。
样本模式下，画像类页面筛选到样本 LAC 后应有数据展示。

## 验收方法
用 Playwright 同时截图两个系统的同一路由，逐项对比：
- 布局结构（区域划分）
- 组件类型（badge、progress bar、card）
- 数据字段（同一字段出现在同一位置）
- 交互模式（筛选器、标签切换、展开折叠）
- 配色系统（green=正常, amber=观察, red=拒绝, orange=异常）
