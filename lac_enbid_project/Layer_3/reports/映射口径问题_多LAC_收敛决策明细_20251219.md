# 映射/口径问题：多 LAC 小区“收敛到主 LAC”的决策明细 — 2025-12-19

目标：解决“同一个 `cell_id_dec` 在 7 天内出现多个 `lac_dec`”导致的口径不一致问题。  
策略：对“多 LAC 小区”，按**信号有效上报量**优先收敛到一个主 LAC；再用 LAC 区域置信度（LAC 库的规模指标）做 tie-break。

## 1) 决策规则（你已确认的思路落地为可执行规则）

对每个 `(operator_id_raw, tech_norm, cell_id_dec)`：

1. **过滤无效信息**：把 `sig_rsrp` 的无效值（`-110`、`-1`、以及非负数 `>=0`）视为“无效信号/占位值”。
2. **7 天上报量排序**：优先选择 `good_sig_cnt` 最大的 `lac_dec`（`good_sig_cnt` 仅统计“有效 RSRP”）。
3. **LAC 区域置信度**（tie-break）：若 `good_sig_cnt` 相同，则用 LAC 库（`Step04_Master_Lac_Lib`）中的 `valid_gps_count` 作为置信度，取更大的。
4. **最终 tie-break**：若仍相同，再按该 cell 在该 LAC 下的 `row_cnt`（Step06 行数）更大者；再按 `lac_dec` 更小者（保证稳定）。

说明：
- `1829638147` 那类“相邻 LAC 切换”通常会被收敛到上报量更大的主 LAC；下一批数据若主 LAC 改变，会自然切换。
- `126936197` 那类“某个 LAC 下信号全 -110”的，会被优先剔除并收敛到另一个 LAC。
