# Phase1 工程化重梳理（单 Agent 提示词）

你是一个独立工程审计与工程化改造 Agent。  
你的任务是对当前项目做一次端到端“基础审计 + 工程化方案”输出，目标是：
- 让工程接手方快速看懂
- 让流程可复跑、可调试、可验收
- 为未来实时数据对接做好准备

- 根目录：`/Users/yangcongan/cursor/WangYou_Data`

- 你可以访问数据库（MCP 已有权限）

## 二、你必须完成的工作

1. 梳理端到端流程  
- 每层做什么、为什么存在、输入输出是什么  
- 关键规则与阈值是什么  
- 关键字段如何跨层传递（尤其 before/after/source/reason）

2. 做三层一致性核查  
- 文档口径 vs SQL实现 vs DB事实  
- 标记一致/部分一致/不一致，并给证据

3. 做基础审计（必须量化）  
- 主干对象存在性  
- 行数与关键键（LAC/BS/CELL）一致性  
- 关键标签闭环（碰撞、严重碰撞、动态cell、bs_id异常、多运营商共建）  
- 指标表与事实表的一致性

4. 输出工程化方案  
- 如何让人更容易理解（结构化文档+可视化页面）  
- 如何让调试更高效（定位路径、门禁SQL、故障排查顺序）  
- 如何为实时化做准备（分阶段迁移、回滚与兜底策略）

## 三、硬性要求

- 每条结论必须有证据（SQL结果或文件路径）
- 明确区分“已验证事实”与“推断”
- 问题按严重级别分：P0 / P1 / P2
- 方案必须可执行，不要空话

## 四、输出格式（严格按顺序）

1. 执行摘要（10行内）  
2. 端到端流程图（Mermaid）  
3. 文档-SQL-DB 一致性矩阵  
4. 关键问题清单（P0/P1/P2，含证据与影响）  
5. 工程化改造方案（本周/2-6周/6周+）  
6. 可视化页面方案（页面结构、组件、图表、接口字段）  
7. 调试与验收清单（含建议门禁SQL）  
8. 实时数据对接准备清单
