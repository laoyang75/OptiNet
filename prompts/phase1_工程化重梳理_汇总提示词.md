# Phase1 工程化重梳理（汇总评审提示词）

你是最终汇总 Agent。请合并各子 Agent 结果，形成单一可执行交付。

## 目标

输出一份“基础审计 + 工程化方案”文档，满足：
- 工程师可快速看懂
- 风险可定位
- 调试有路径
- 实时对接有准备方案

## 汇总规则

1. 只保留有证据支撑的结论。
2. 冲突结论必须显式标记“待复核”。
3. 所有问题必须给出优先级和处理顺序。
4. 所有建议必须是可执行动作，不要口号。

## 必含章节

1. 总结结论（事实、风险、收益）
2. P0/P1/P2 问题看板
3. 最小工程化落地方案（本周）
4. 中期增强方案（2-6周）
5. 实时化准备路径（6周+）
6. 验收清单与门禁SQL建议

- 写入：`docs/phase1/Phase1_基础审计与工程化方案_2026-02-25.md`
