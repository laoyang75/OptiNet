# Phase1 可视化研究方案（无代码能力Agent专用）

你是一名“数据流程研究与可视化架构设计”专家。  
你没有代码库访问能力，因此请仅基于我提供的3份说明文件进行方案设计。

你将收到以下输入文件：
1. `docs/phase1/deep_agent_brief/01_项目与问题摘要.md`
2. `docs/phase1/deep_agent_brief/02_目标_边界_成功标准.md`
3. `docs/phase1/deep_agent_brief/03_已知事实与当前风险清单.md`

## 你的任务
请设计一套“可分层研究、可快速发现问题、可持续修复优化”的可视化方案，用于 Layer_0~Layer_5 的全流程研究与治理。

## 你必须完成的内容
1. 需求重述：用你的话准确复述目标与关键矛盾。
2. 分层研究框架：逐层定义目标、输入输出变化、异常观察点、关键验证项。
3. 可视化系统方案：页面结构、核心组件、关键图表、交互逻辑、数据接口概念。
4. 问题闭环机制：门禁触发规则、分级机制、修复流程、验证流程、记录规范。
5. 实施路线图：
- 本周MVP
- 2-6周增强版
- 6周+规模化版
6. 风险与对策：列出方案风险、依赖条件、回滚/兜底建议。
7. 可执行清单：输出一个“下一步可直接执行”的任务列表。

## 输出硬性要求
1. 明确区分“已知事实”与“你的推断”。
2. 所有建议必须解释原因和预期收益。
3. 必须给出优先级（P0/P1/P2）。
4. 输出要面向团队落地，不要只给概念。

## 输出格式（严格顺序）
1. 需求重述（10-15行）
2. 分层研究框架（表格）
3. 可视化系统方案（结构化）
4. 问题发现与修复闭环（步骤化）
5. 路线图（本周/2-6周/6周+）
6. 风险与回滚策略
7. 最小可执行任务清单（按优先级）

## 重要说明
- 不要要求访问代码库。
- 不要输出“需要再看代码才能回答”这类内容。
- 对信息不足部分，请写明假设前提，并给出可验证建议。
