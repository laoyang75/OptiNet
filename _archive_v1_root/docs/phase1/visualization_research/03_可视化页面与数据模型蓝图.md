# 可视化页面与数据模型蓝图

## 1. 页面结构
1. 全局总览页
- 链路健康状态
- Layer 行数守恒
- 核心异常指标
2. 分层研究页（L0~L5）
- 层目标/输入/输出
- 规则命中
- 前后变化
- 样本穿透
3. 问题与补丁页
- 问题池
- 修复进度
- 补丁效果
4. 决策与日志页
- 操作记录
- 关键决策
- 回滚记录

## 2. 核心组件
1. KPI卡片：行数、覆盖率、异常率、补齐收益。
2. Sankey：输入->规则->输出流转。
3. Diff表：修复前后指标差异。
4. Trace详情：按 `seq_id` 或主键回看完整过程。
5. 规则雷达图：每层规则复杂度与风险暴露。

## 3. 建议数据模型（最小）
1. `pipeline_layer_snapshot`
- `run_id/layer_id/input_rows/output_rows/pass_flag`
2. `pipeline_rule_hit`
- `run_id/layer_id/rule_code/hit_rows/hit_ratio`
3. `pipeline_quality_metric`
- `run_id/layer_id/metric_code/before_value/after_value/delta`
4. `pipeline_issue`
- `issue_id/severity/layer_id/title/evidence_sql/status`
5. `pipeline_patch_log`
- `patch_id/issue_id/change_type/change_summary/owner/verified_flag`
6. `pipeline_decision_log`
- `decision_id/context/options/chosen/rationale/risk`

## 4. 关键接口（建议）
1. `GET /api/pipeline/overview`
2. `GET /api/pipeline/layer/{layer_id}`
3. `GET /api/pipeline/layer/{layer_id}/rule-hit`
4. `GET /api/pipeline/issues`
5. `POST /api/pipeline/patch/verify`
6. `GET /api/pipeline/trace/{trace_key}`

## 5. 门禁触发（建议）
1. 行数守恒失败
2. 契约字段缺失
3. 指标表与事实表不一致
4. 异常标签闭环断裂
5. 无效LAC泄漏
