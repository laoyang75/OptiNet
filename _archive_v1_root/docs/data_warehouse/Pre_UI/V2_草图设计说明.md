# 本地治理链路工作台一期：V2 UI 草图设计说明

> 版本：v2.0 | 日期：2026-03-22
> 依据：《本地治理链路工作台_开发基础文档》+《本地治理链路工作台_UI开发意见》
> 定位：低保真草图级设计说明，需求对齐用

---

## 1. 设计定位

这不是"治理运营平台"的 UI，而是一个：

**治理链路调试与验证工作台**

核心价值：帮助人快速完成"看步骤 → 看规则/SQL/参数 → 改规则 → 重跑 → 看差异 → 看样本 → 判断是否合理"的闭环。

### 2.1 顶部上下文条

全局固定，所有页面共享。始终回答"我现在看的是哪次运行"。

| 字段 | 说明 |
|------|------|
| Run | 当前查看的 run_id |
| Compare Run | 对比的 run_id（差异基准） |
| 参数集 | parameter_set_id |
| 规则集 | rule_set_version |
| SQL 版本 | sql_bundle_version |
| 契约版本 | contract_version |
| 基线版本 | baseline_version（如适用） |
| 当前步骤 | 当前正在查看的步骤名 |

右侧操作：[版本抽屉] 按钮、[刷新] 按钮 + 最后刷新时间。

上下文条中 Run 和 Compare Run 可下拉切换，切换后当前页面数据刷新。

### 2.2 左侧链路导航

分两个区域：

**治理链路区**（主导航）：按链路顺序排列所有步骤节点。每个节点显示：
- 状态图标：● 已完成 / ◉ 当前 / ○ 未执行 / ✗ 失败
- 步骤业务名
- 输出行数或对象数
- 与 compare run 相比有显著变化时加 diff 标记

**辅助页面区**：字段治理、样本研究。

点击链路节点 → 进入该步骤的工作台页。
点击辅助页面 → 进入对应页面。

### 5.3 八个区块说明

| 区块 | 内容 | 不能缺的信息 |
|------|------|-------------|
| A. 步骤说明 | 名称、业务目的、上下游、状态、库映射 | 当前库映射（Y_codex_* 表名） |
| B. 输入/输出 | 库名、行数、主键、维度、与上次变化 | 必须有 diff 标记 |
| C. 规则区 | 规则列表，每条带目的/参数/命中数 | 每条可展开详情和命中样本 |
| D. 参数区 | 参数名/当前值/上次值/变化 | 必须展开具体参数值，不能只显示参数集名 |
| E. SQL 区 | SQL 列表，执行顺序，每条可展开 | 点击展开 → SQL 抽屉显示完整 SQL |
| F. 数据变化 | 本次 run 处理前后的指标 | 关键统计指标，不是只有行数 |
| G. 差异区 | current run vs compare run 的指标对比 | 新增/消失/变化对象数 |
| H. 样本区 | 分类样本表（正常/异常/边界） | 每行可打开详情抽屉 |

### 5.4 不同步骤的差异化内容

虽然共用模板，但各步骤的规则区/参数区/样本区内容不同。草图用 3 个代表性步骤展示：

| 步骤 | 特色内容 |
|------|---------|
| **可信 LAC** | 运营商过滤规则、LAC 编码黑名单、位置过滤参数、稳定性阈值 |
| **可信 BS** | Cell→BS 聚合规则、中心点计算、碰撞检测阈值、空间聚落分析样本 |
| **GPS 修正** | 距离阈值参数、修正率、漂移样本（修正前/修正后对比）、碰撞桶处理 |

## 9. 页面间跳转链路

```
治理链路总览
  │
  ├─ 点击链路节点 ──→ 步骤工作台（对应步骤）
  ├─ 点击差异行   ──→ 步骤工作台（对应步骤）
  ├─ 点击字段变化 ──→ 字段治理页
  └─ 点击问题提示 ──→ 样本研究页

步骤工作台
  │
  ├─ 左侧导航     ──→ 步骤工作台（其他步骤）
  ├─ SQL [展开]   ──→ SQL 抽屉
  ├─ 样本行       ──→ 样本/对象详情抽屉
  ├─ [跳转样本]   ──→ 样本研究页
  ├─ 规则展开     ──→ 同页折叠展开
  └─ [版本抽屉]   ──→ 版本与运行抽屉

字段治理页
  │
  ├─ 影响步骤     ──→ 步骤工作台（对应步骤）
  └─ 字段行展开   ──→ 同页展开治理内容

样本研究页
  │
  ├─ 样本行       ──→ 样本/对象详情抽屉
  ├─ [跳转步骤]   ──→ 步骤工作台（对应步骤）
  └─ [样本重跑]   ──→ 触发 run（结果在步骤工作台查看）
```

主要跳转特征：
- **步骤工作台是中枢**，几乎所有路径都经过它
- **减少深层跳转**，优先用同页展开和抽屉
- **字段治理 ↔ 步骤工作台** 双向可达
- **样本研究 ↔ 步骤工作台** 双向可达

## 12. 设计原则总结

1. **步骤是主线**：导航按链路节点排列，不按平台模块排列。人的注意力始终在"这一步怎么回事"。

2. **SQL 必须可见**：SQL 是工作台核心资产，不是后台细节。每一步都能看到实际执行的 SQL、参数替换结果、版本。

3. **差异是标配**：每个步骤都有 current vs compare 的 diff 区块，不是可选项。

4. **同页展开优于跳转**：字段治理内容在列表中展开，不跳新页面。样本详情用抽屉，不开新窗口。减少深层跳转，保持上下文。

5. **参数展开给人看**：不能只显示"参数集 P-003"，必须把 collision_threshold=1.5km 这些具体值摊开。

6. **工作台不是大屏**：优先用表格、指标卡片、diff 标记。不依赖复杂图表和动画。信息关系清楚比视觉炫酷重要。

7. **4 页 + 3 抽屉**：不做过多独立大页。版本/SQL/样本详情用抽屉承载，人的主注意力不离开治理链路。

8. **字段治理展开到位**：不只是字段列表。每个字段的映射规则、解析逻辑、健康趋势、影响步骤、变更历史都在展开区直接可见。
