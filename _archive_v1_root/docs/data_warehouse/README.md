## 背景

当前在本地 PostgreSQL 上已完成完整的数据处理流水线（`lac_enbid_project`，Layer_0 ~ Layer_5），需要将这套逻辑迁移到 MaxCompute 构建正式数仓，实现：

1. **可信基线构建**：用 14+ 天数据构建 LAC / BS / Cell 三级可信基线表
2. **日常增量比对**：每天新数据撞基线表，好数据沉淀、发现变化和漂移
3. **数据质量监控**：继承现有 Obs Mart 机制，在数仓层面实现质量门控
