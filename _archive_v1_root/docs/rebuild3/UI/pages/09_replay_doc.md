# 回放对比设计说明

## 页面目标
验证"3天初始化+4天增量"与"7天直接初始化"两种路径的结果收敛性，确保增量治理模式可靠

## 用户主要操作
1. 查看综合收敛度评分
2. 浏览各维度收敛指标
3. 下钻查看差异对象列表
4. 分析差异原因
5. 查看收敛趋势变化

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 核心字段 |
|---------|---------|---------|---------|
| 对比设置 | 8% | 显示对比的两个运行 | left_run, right_run, status |
| 综合评分 | 12% | 一眼看到收敛结果 | overall_score, 6维度得分 |
| 详细对比 | 35% | 各维度具体数据 | 对象/空间/信号/异常/决策收敛 |
| 差异对象 | 30% | 列出不一致的对象 | 对象, 两侧结果, 差异原因 |
| 趋势图 | 15% | 收敛随批次的变化 | 按批次的各指标曲线 |

## 筛选器 & 排序
- 差异对象: 按类型/差异原因筛选
- 趋势图: 选择展示的指标维度

## 状态表达规则
- 收敛达标(>95%): 绿色
- 收敛接近(90-95%): 黄色/amber
- 收敛不足(<90%): 红色
- 综合评分: 大号环形进度条
- 差异对象: 按差异类型着色

## 组件边界建议
- ReplayView.vue
  - ComparisonSetup (对比信息卡)
  - ConvergenceScore (环形进度 + 6维卡片)
  - DetailComparison
    - ObjectConvergence (表格)
    - SpatialConvergence (表格)
  - DivergentObjects (差异对象表)
  - ConvergenceTrend (ECharts 折线图)

## 读模型建议
- /api/v1/replay/:id/summary — 综合评分和6维度
- /api/v1/replay/:id/objects — 对象收敛详情
- /api/v1/replay/:id/spatial — 空间收敛
- /api/v1/replay/:id/divergent?page= — 差异对象
- /api/v1/replay/:id/trend — 按批次趋势
- 所有对比数据应后端预计算

## 空状态 / 错误状态
- 无对比任务: "暂无回放对比任务，请先完成初始化和增量运行"
- 对比进行中: 显示进度条和已完成步骤
- 部分指标不可用: 该指标卡显示"数据不足"

## 开发注意事项
- 收敛计算是耗时操作，使用异步任务模式
- 环形进度条建议 ECharts gauge 或 SVG
- 差异对象可能很多，必须分页
- 趋势图建议按批次 X 轴展示
