# 初始化数据页 — 设计文档

## 页面目标

展示系统冷启动（3天批量处理）的完整初始化流程与结果，让用户理解：

- 初始化经历了哪些步骤，每步产出了什么
- 初始化结束后系统的起始状态（对象数量、基线版本、四分流分布）
- 当前研究期采用了哪些特殊策略/口径限制

## 回答的核心问题

| 问题 | 对应区域 |
|------|----------|
| 初始化做了什么？ | 初始化流程（11步竖向流程） |
| 每一步的产出和通过率？ | 流程步骤的 step-detail 行 |
| 系统起始状态是什么？ | 初始化结果汇总（10张卡片） |
| 数据质量如何分布？ | 四分流分布（横向条形图） |
| 有哪些特殊策略需要注意？ | 研究期口径说明（提示框） |

## 用户操作

本页为**只读展示页**，不提供交互操作。用户可以：

- 查看初始化全流程及每步状态
- 查看汇总卡片了解起始规模
- 查看四分流比例判断数据质量
- 阅读口径说明了解当前限制条件
- 通过侧边栏导航至其他工作台

## 页面分区

### 1. 上下文条（context-bar）

显示本次初始化的关键标识：

- `run_id`: RUN-20260403-001
- `初始化窗口`: 2026-04-01 ~ 2026-04-03 (3天)
- `状态`: 已完成（绿色徽标）
- `完成时间`: 2026-04-03 00:15:32

### 2. 初始化流程（init-flow）

竖向步骤列表，共 11 步，每步包含：

- **步骤序号**：左侧数字
- **完成图标**：绿色勾号表示已完成
- **步骤名称**：粗体标题
- **步骤详情**：等宽字体显示具体数据（输入/输出/通过率）

步骤清单：

1. 冻结 run_id 与版本绑定
2. 全量标准化事件
3. 记录级合规检查
4. 研究期 LAC 预边界/区域缩边界
5. Cell 候选累计与晋升
6. 由 active Cell 派生 BS
7. 由 active BS 派生 LAC
8. 首轮事实治理
9. 异常检测与资格收敛
10. 完整回归一次
11. 生成 baseline v1

- 严格 GPS 过滤 = 当前北京样本验证阶段

## 组件复用

| 组件 | 来源 | 说明 |
|------|------|------|
| context-bar | 全局共用 | 上下文条，与流转总览页一致 |
| summary-card | 全局共用 | 汇总卡片，与流转总览页一致 |
| sidebar / topbar | 全局布局 | 侧边栏和顶栏，所有页面共享 |

本页新增组件：

| 组件 | 说明 |
|------|------|
| init-flow / init-step | 初始化步骤列表，竖向排列 |
| bar-chart / bar-row | 横向条形图，展示分布比例 |
| note-box | 提示说明框，靛蓝色背景 |
