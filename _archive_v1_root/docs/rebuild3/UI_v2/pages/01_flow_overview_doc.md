# 流转总览 设计说明

## 页面目标
作为系统首页，帮助用户在进入系统后第一时间看清当前流转状态，而不是只看到几张总览卡片。

## 这个页面主要回答什么问题
1. 当前 run / batch / baseline 是什么
2. 当前批次的数据流到了哪里（四分流去向）
3. 哪个节点的数量变化最大
4. waiting / issue / rejected 是否在堆积
5. 哪类对象失去了锚点或 baseline 资格
6. 最值得马上处理的问题是什么

## 用户主要操作
1. 扫描当前上下文（run/batch/baseline）
2. 观察四分流比例和 delta 变化
3. 识别关键变化指标中的异常
4. 点击问题入口下钻到对应页面

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 当前上下文条 | 5% | 显示运行上下文 | run_id/batch_id/window/baseline_version/rule_set_version | 否 | 是(→批次中心) |
| 批次流转结构 | 30% | 可视化四分流 | 输入量/四路分流计数和占比/晋升降级/baseline刷新 | 是 | 是(→事实列表) |
| 关键变化指标 | 40% | delta 扫描 | 8个核心指标的当前值+本批新增+较上批delta | 是 | 是(→对应页面) |
| 问题入口 | 25% | 引导下钻 | 问题类型/数量/严重度/跳转链接 | 是 | 是(→各工作台) |

## 筛选器与切换器
- 批次切换器（可切换查看历史批次的流转总览）
- 对比基准切换（较上批 / 较昨日 / 较上周同期）

## 状态表达规则
- 全局状态条：正常=隐藏，批次处理中=蓝色脉冲，重跑中=紫色，验证模式=灰蓝
- Delta: 改善=绿色，恶化=红色，中性=灰色
- 问题入口：按严重度排序，红>橙>黄>蓝

## 下钻路径
- 上下文条 run_id → 批次中心
- 四分流节点 → 该批次该层事实列表
- 变化最大指标 → 对应页面（等待池/异常/对象）
- delta 异常指标 → 批次中心对应批次
- 问题入口项 → 各工作台

## 组件边界建议
- FlowOverviewView.vue (页面)
- VersionContext.vue (上下文条)
- BatchFlowDiagram.vue (四分流可视化)
- MetricCardWithDelta.vue (delta 指标卡) x8
- PriorityIssueList.vue (问题入口列表)

## 读模型建议
- GET /api/flow-overview?batch_id=xxx 获取当前批次流转数据
- 预聚合：四分流计数、delta、问题入口列表在后端计算好
- 延迟加载：无（首页应全量立即加载）

## 空状态 / 错误状态 / 重跑中状态
- 空状态：首次启动未跑任何批次时，显示"尚未运行任何批次，请先在启动器中启动服务"
- 错误状态：API 连接失败时显示全屏错误 + 重试按钮
- 重跑中：顶部全局条显示紫色"重跑中"标记，受影响的指标卡显示"重跑中"覆盖层

## 开发注意事项
- 首页加载性能最关键，所有数据应在后端预聚合
- 四分流可视化使用 SVG 或 Canvas，不用 DOM 堆叠
- Delta 颜色语义需要后端标注（某些指标增加=恶化），不能前端自行判断
