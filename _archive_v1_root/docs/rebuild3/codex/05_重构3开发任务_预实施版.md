# rebuild3 开发任务（预实施版）

> 状态：预实施版  
> 适用对象：Agent、人类评审、后续 UI 设计协作者  
> 重要说明：这不是最终开发任务书。当前阶段先冻结任务边界与输入，审阅通过后先做 UI 设计，UI 确认后再生成最终实施任务书

## 1. 本文档的用途

这份文档不是让 agent 立刻开始大规模实现 rebuild3，而是让后续工作先满足三件事：

1. 把必须先确定的口径和结构冻结下来。
2. 给 UI 设计提供稳定的对象、状态、读模型和页面输入。
3. 为 UI 评审通过后的最终实施任务书准备清晰边界。

因此，当前阶段的任务目标应理解为：

- 冻结规则
- 冻结结构
- 冻结验证方式
- 准备 UI 设计输入

而不是：

- 直接做全量 SQL 实现
- 直接重写前后端
- 直接开始最终开发

## 2. 当前阶段的总边界

### 2.1 这阶段必须完成

1. 统一 rebuild3 的术语、主语和状态表达。
2. 冻结对象注册、事实分层、版本绑定、幂等键的最小方案。
3. 输出 UI 设计所需的页面对象、状态枚举、数据读模型。
4. 冻结本地验证路径和对比指标。
5. 明确哪些项是实现前阻塞，哪些项可以在实现过程中继续迭代。

### 2.2 这阶段明确不做

1. 不做最终页面开发。
2. 不做全量数据库迁移和历史回灌执行。
3. 不做最终 API 全量实现。
4. 不做云端化设计和复杂调度系统。
5. 不对所有异常类型做自动拆分或自动迁移处理。
6. 不在 UI 设计前直接进入 Vue 页面实现。

## 3. 当前阶段必须先拍板的 5 个阻塞项

这 5 项不确认，不应进入实现层任务拆解。

| 编号 | 阻塞项 | 必须确认的内容 |
| --- | --- | --- |
| D1 | `event_time` 口径 | 哪个时间用于批次归属、donor 查找、活跃判断、画像窗口 |
| D2 | 源适配边界 | 首轮到底支持单表回放、双表回放，还是两者都兼容 |
| D3 | 三层门槛 | Cell 的最低注册门槛、锚点门槛、画像成熟门槛 |
| D4 | 异常资格矩阵 | 哪些异常还能写 `fact_governed`、当锚点、参与 baseline |
| D5 | 热层保留策略 | 是否必须有 daily aggregate、N 是否分层、热库怎么淘汰 |

## 4. 预实施任务分期

当前建议把任务拆成 4 个阶段，其中前 3 个阶段属于“现在要做的准备”，第 4 阶段属于“UI 确认后再做”。

### 阶段 A：冻结业务与状态口径

目标：先把“说法统一”这件事做完，避免后面 UI 和实现各画各的。

| 任务编号 | 任务 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| A1 | 冻结对象主语 | Cell / BS / LAC 的职责说明 | 文档中不再出现互相冲突的对象主语 |
| A2 | 冻结二维状态模型 | `lifecycle_state` 与 `health_state` 枚举 | 不再用单一大状态枚举混写生命周期和健康度 |
| A3 | 冻结三类资格 | 存在资格、锚点资格、baseline 资格矩阵 | 每种异常都能回答“能否写事实、能否做锚点、能否进基线” |
| A4 | 冻结 LAC / BS 的“注册”和“激活”区别 | LAC/BS 的存在阈值与激活阈值说明 | 不再出现“≥1”与“≥N”冲突写法 |
| A5 | 冻结研究期口径与长期口径边界 | 北京严格过滤与长期轻量过滤的边界文档 | 研究规则不会再被误当成长期架构规则 |

### 阶段 B：冻结数据结构与版本口径

目标：把后续所有页面、API、SQL、回放都依赖的最小骨架确定下来。

| 任务编号 | 任务 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| B1 | 输出对象注册表草案 | `obj_cell_registry / obj_bs_registry / obj_lac_registry` 字段草案 | 主键、状态、当前关系、版本归属齐全 |
| B2 | 输出状态历史与关系历史草案 | `*_state_history`、`*_relation_history` 字段草案 | 能支持迁移、分裂、退役留痕 |
| B3 | 输出事实分层草案 | `fact_governed / fact_pending_observation / fact_pending_issue / fact_rejected` 职责说明 | 每条事件只能落到明确的一层 |
| B4 | 输出基线与画像草案 | baseline registry、三级 profile、热层/长层/归档层边界 | 解释清楚“上一版 baseline 判当前批” |
| B5 | 输出 run / batch / version / idempotency 口径 | 四类版本绑定与标准事件幂等键 | 2 小时回放可追溯、可重放、可比对 |

### 阶段 C：给 UI 设计准备稳定输入

目标：让 UI 设计基于真实对象与真实状态，在 `Vue 3 + TypeScript` 选型下重构前端结构，而不是继续围绕旧 Step 页面做视觉微调。

| 任务编号 | 任务 | 产出 | 验收标准 |
| --- | --- | --- | --- |
| C1 | 输出 UI 页面清单 | 初始化视图、对象注册视图、批次决策视图、等待/观察池视图、异常视图、画像视图 | 页面清单能覆盖初始化、增量、异常、基线四类场景 |
| C2 | 输出页面读模型草案 | 每个页面需要展示的字段、状态、标签、计数 | UI 设计不再需要直接阅读底层大表 |
| C3 | 输出关键交互状态 | 空状态、等待态、观察态、异常态、锚点禁用态、回放对比态 | UI 能完整表达系统状态，而不只展示统计值 |
| C4 | 输出样本场景集 | 正常对象、待晋升对象、碰撞对象、迁移怀疑对象、拒收事件样本 | UI 可用真实场景而不是抽象占位内容设计 |
| C5 | 输出页面与对象主语映射 | 每个页面到底围绕对象、决策、事实还是基线组织 | 避免再次退回 Step 导航思维 |
| C6 | 输出 Vue 组件边界草案 | 页面组件、共享组件、抽屉组件、筛选组件、状态存储边界 | 防止实现期再次出现超大单文件页面 |

### 阶段 D：UI 确认后再做的事

目标：在 UI 方案稳定后，把当前预实施任务文档升级成最终实施任务书。

| 任务编号 | 任务 | 产出 | 当前是否执行 |
| --- | --- | --- | --- |
| D1 | 基于 UI 方案重写最终实施任务书 | 最终开发任务、API 任务、SQL 任务、页面任务、验收计划 | 否 |
| D2 | 把读模型转换为正式 API 与查询任务 | API backlog、查询草案、缓存策略 | 否 |
| D3 | 把对象状态和页面状态对齐 | UI 状态机与后端状态机映射清单 | 否 |
| D4 | 进入真实编码与回放验证 | 实现排期和执行顺序 | 否 |

## 5. 当前建议 agent 先做的实际任务

如果后续让 agent 进入执行，这一阶段建议只允许它做下面这些任务：

1. 整理并冻结状态机、资格矩阵、异常分流矩阵。
2. 起草注册表、事实层、基线层、版本层的数据结构文档。
3. 输出 UI 设计需要的页面清单、页面读模型、关键状态说明。
4. 准备“3 天起跑，滚到 7 天，对比直接 7 天”的验证方案和指标说明。
5. 在出现口径冲突时，优先回到本轮 3 份最终文档，不再回到 `a/b` 候选稿各自取说法。

## 6. 当前阶段禁止 agent 直接做的事情

为了避免过早进入错误实现，当前明确禁止 agent 直接推进下面这些工作：

1. 直接实现全量数据库建表和全量回放。
2. 在没有 UI 方案和组件边界的情况下，直接开始写 Vue 页面。
3. 直接把静态可信库页面硬改成最终运营页面。
4. 未冻结 `event_time / batch / idempotency` 前开始写增量执行代码。
5. 未冻结异常资格矩阵前开始写 GPS 修正和信号补齐门控逻辑。
6. 未完成 UI 设计前开始写最终页面和最终 API。

## 7. 本阶段的验收标准

这份“预实施任务”是否完成，不看代码量，看下面 6 件事是否已经清楚：

1. 人和 agent 对对象主语、状态机、资格矩阵说法一致。
2. UI 设计已经拿到稳定输入，不需要再反复追问业务对象含义。
3. 数据结构层已经能回答“事件写去哪一层、对象状态写去哪一层、关系历史写去哪一层”。
4. 回放验证路径已经明确，不再停留在抽象口号。
5. 当前阶段的非目标已经明确，不会提前滑入最终开发。
6. 审阅通过后，能自然进入“先做 UI 设计”的下一阶段。

## 8. UI 设计完成后，最终实施任务书必须包含什么

这部分现在不展开实现，但必须先说清楚，避免后续遗漏。

UI 设计完成后，最终实施任务书至少要补齐：

1. 数据库层任务拆解。
2. API 与读模型任务拆解。
3. 页面与交互任务拆解。
4. 回放验证与指标任务拆解。
5. 开发顺序与依赖关系。
6. 每项任务的验收标准和回归标准。

## 9. 结论

当前这份开发任务文档的定位，是“把最终开发前必须先决定的事情决定掉”。

它服务于两个后续动作：

1. 让 UI 设计基于统一主语开展。
2. 让 UI 确认后生成最终实施任务书时，不再回头重新争论基础口径。
