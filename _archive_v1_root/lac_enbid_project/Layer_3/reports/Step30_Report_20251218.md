## 0) 字段说明（看表用）

- `PASS/WARN/FAIL`：通过/告警/阻断
- `OK（未触发阈值）`：未触发该项的默认阈值告警（等价于“正常”）
- `TopN/Top10`：前 N / 前 10（用于抽样展示）
- `tech_norm`：制式（归一化技术体制，取值示例：4G/5G）
- `bs_id`：基站 ID（同一制式下的基站编号）
- `wuli_fentong_bs_key`：物理分桶基站键（用于跨表 join/归并），规则为 `tech_norm|bs_id|lac_dec_final`（即 `concat_ws('|', tech_norm, bs_id::text, lac_dec_final::text)`）
- `shared_operator_list`：共享运营商列表（逗号分隔，例如 `46000,46015`）
- `gps_valid_level`：GPS 可用性分级（Usable=可用，Risk=风险，Unusable=不可用）
- `gps_valid_cell_cnt`：该基站桶内“有效小区”数量（用于判定桶质量/碰撞风险）
- `gps_valid_point_cnt`：该基站桶内“有效 GPS 点”数量
- `gps_p90_dist_m`：GPS 点到基站中心点距离的 P90（单位：米）
- `gps_max_dist_m`：GPS 点到基站中心点距离的最大值（单位：米）
- `anomaly_cell_cnt`：异常小区数量（来自 Layer_2 异常识别的汇总）
- `collision_reason`：碰撞疑似原因编码（例如 `GPS_SCATTER_P90_GT_THRESHOLD`=GPS 离散度(P90)超阈值）
