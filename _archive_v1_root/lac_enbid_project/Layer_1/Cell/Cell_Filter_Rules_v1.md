# L1-CELL：`cell_id` 筛选规则 v1（可直接按此操作）

> 目标：给出一份**简短、可直接执行**的 `cell_id` 筛选说明，用于从表  
> `public."网优cell项目_清洗补齐库_v1"` 中筛出“格式合规 + 剔除已知无效值”的 4G/5G Cell 明细。

本文件**独立可读**，不依赖其他文档。

---

## 1. 全局约束与直接过滤

**数据源：**

- 表：`public."网优cell项目_清洗补齐库_v1"`
- 必须满足：
  - `运营商id` ∈ {`46000`, `46001`, `46011`, `46015`, `46020`}
  - `原始lac` 非空且为纯数字：`"原始lac" ~ '^[0-9]+$'`

**全局直接过滤规则：**

- 过滤所有 `tech` 归一后为 2G/3G 或 OTHER 的记录：
  - 只保留 `tech_norm` ∈ {`'4g'`, `'5g'`}
- 过滤 `cell_id` 为以下情况：
  - `NULL` 或空字符串；
  - 明确无效占位值：`'0'`、`'-1'`；
  - 含非数字字符（字母、符号等）；
  - 数值 ≤ 0；
  - 溢出默认值：**2147483647**（十六进制 `0x7FFFFFFF`，在 5G 中大量出现且在同一运营商内对应多个 LAC）。

---

## 2. 运营商分组与制式划分

为方便说明与后续扩展，按以下方式划分：

- 运营商分组：
  - **第一组（移动系）**：`运营商id` ∈ {`46000`, `46015`, `46020`}
  - **第二组（联通 + 电信）**：`运营商id` ∈ {`46001`, `46011`}
- 制式归一：
  - 取 `tech_norm = lower(tech)`；
  - `tech_norm = '4g'` → 归为 4G
  - `tech_norm = '5g'` → 归为 5G
  - 其他（2g/3g/空/乱写）全部过滤，不在本轮 Cell 合规中使用。

---

## 3. 第一组（移动系）规则：有效规范与异常值

### 3.1 有效规范（保留条件）

对 `运营商id` ∈ {`46000`, `46015`, `46020`} 的记录：

- 制式必须是 4G 或 5G：
  - `lower(tech)` ∈ (`'4g'`, `'5g'`)
- LAC 必须数值合规：
  - `"原始lac" IS NOT NULL`
  - `btrim("原始lac") <> ''`
  - `btrim("原始lac") ~ '^[0-9]+$'`
- `cell_id` 必须满足：
  - 非空：`cell_id IS NOT NULL` 且 `btrim(cell_id::text) <> ''`
  - 纯数字：`btrim(cell_id::text) ~ '^[0-9]+$'`
  - 数值 > 0：`cell_id::bigint > 0`
  - 数值 ≠ 2147483647（溢出默认值）

在当前数据中，满足上述条件的移动系 4G/5G cell_id：

- 格式上几乎全部合规（极少有 0/-1 已被过滤）；  
- 十进制长度分布：
  - 4G：主要集中在 7–9 位；
  - 5G：主要集中在 9–11 位（以 10 位为主）。

本轮筛选**不对长度做硬过滤**，只记录上述分布特征，避免误伤正常编码。

### 3.2 异常值（本轮直接剔除）

对第一组（移动系），本轮明确认定为无效并过滤的 `cell_id` 类型：

- 所有制式下的：
  - 空值 / 空字符串；
  - `'0'` / `'-1'`；
  - 含非数字字符；
  - 数值 ≤ 0。
- 5G 中的：
  - `cell_id::bigint = 2147483647`（0x7FFFFFFF）：  
    - 在移动系 5G 中出现 5 万多次；  
    - 在同一运营商内对应大量不同 LAC → 典型溢出默认值。

其他 5G 大数（如 `5839671298`, `47261081712`, `5751177217` 等）在当前样本中均表现为“同运营商内绑定单个 LAC”，暂不判为无效，仅作为后续研究对象。

---

## 4. 第二组（联通 + 电信）规则：有效规范与异常值

### 4.1 有效规范（保留条件）

对 `运营商id` ∈ {`46001`, `46011`} 的记录：

- 制式必须是 4G 或 5G：
  - `lower(tech)` ∈ (`'4g'`, `'5g'`)
- LAC 必须数值合规（同上）；
- `cell_id` 必须满足（与第一组相同）：
  - 非空：`cell_id IS NOT NULL` 且 `btrim(cell_id::text) <> ''`
  - 纯数字：`btrim(cell_id::text) ~ '^[0-9]+$'`
  - 数值 > 0：`cell_id::bigint > 0`
  - 数值 ≠ 2147483647

当前数据中，满足上述条件的联通/电信 4G/5G cell_id：

- 格式几乎全部合规；  
- 十进制长度分布：
  - 4G：主要集中在 7–9 位；
  - 5G：主要集中在 9–11 位（9 位和 10 位均大量存在）。

同样，本轮不对长度做硬过滤。

### 4.2 异常值（本轮直接剔除）

对第二组（联通 + 电信），本轮明确剔除的 `cell_id` 类型与第一组一致：

- 所有制式下：
  - 空值 / 空字符串；
  - `'0'` / `'-1'`；
  - 含非数字字符；
  - 数值 ≤ 0。
- 5G 中：
  - `cell_id::bigint = 2147483647`：  
    - 在联通/电信 5G 中也为高频值，并且同一运营商内对应多个 LAC → 溢出默认值。

其他 5G 大数（如 `9548546066`, `9548546067`, `9562177557`, `13562122242` 等）当前表现为“同运营商内只绑定 1–2 个 LAC”，暂不直接判为无效。

---

## 5. 可直接使用的筛选 SQL（统一规则）

下面是一条可以直接用于生成“本轮 Cell 合规明细”的 SQL，已经包含上述所有过滤条件：

```sql
CREATE OR REPLACE VIEW public.v_cell_L1_stage1 AS
SELECT
  t.*
FROM public."网优cell项目_清洗补齐库_v1" t
WHERE
  -- 运营商范围：5 个核心 PLMN
  t."运营商id" IN ('46000','46001','46011','46015','46020')

  -- LAC 数值合规
  AND t."原始lac" IS NOT NULL
  AND btrim(t."原始lac") <> ''
  AND btrim(t."原始lac") ~ '^[0-9]+$'

  -- 只保留 4G / 5G
  AND lower(t.tech) IN ('4g','5g')

  -- cell_id 格式合规：非空、纯数字
  AND t.cell_id IS NOT NULL
  AND btrim(t.cell_id::text) <> ''
  AND btrim(t.cell_id::text) ~ '^[0-9]+$'

  -- 数值 > 0
  AND t.cell_id::bigint > 0

  -- 剔除已知溢出默认值 0x7FFFFFFF
  AND t.cell_id::bigint <> 2147483647;
```

> **使用建议：**
>
> - 后续所有关于 cell 的统计、TOP 研究、与 LAC/GPS 的组合分析，都可以直接以 `public.v_cell_L1_stage1` 作为输入；  
> - 如果未来识别出新的明确无效 `cell_id`（例如更多“同运营商内跨大量 LAC”的默认值），只需要在这一 SQL 的尾部追加相应的 `AND t.cell_id::bigint NOT IN (...)` 即可。
