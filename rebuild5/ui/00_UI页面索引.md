## 使用顺序建议

| 目标 | 建议先看 |
|------|----------|
| 搭整体框架 | `../docs/08_UI设计.md` |
| 设计导航和总壳 | `01_全局框架与导航.md` |
| 设计治理控制台页面 | `03_ETL数据接入页面.md` 到 `07_系统配置页面.md` |
| 设计业务页面 | `08_服务层页面.md` |

## 说明

- 控制侧链没有独立页面，统一在“运行历史”中表达初始化、局部重算、完整回归和对比验证。
- 后续如果要逐页深挖视觉稿，应在对应分册下面继续细分，而不是把全部内容继续堆回总纲。
