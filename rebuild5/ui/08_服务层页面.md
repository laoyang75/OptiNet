# 服务层页面

本文件定义 Step 6 的三页：基站查询、覆盖分析、统计报表。

## 页面组定位

| 项目 | 说明 |
|------|------|
| 页面组角色 | 面向业务用户提供最终查询、分析和报表能力 |
| 主要用户 | 业务用户 |
| 核心问题 | 基站在哪、覆盖如何、结果能否导出 |
| 语言风格 | 弱化治理术语，强调结果和可用性 |

## 页面一：基站查询

| 项目 | 说明 |
|------|------|
| 用户目的 | 查一个小区 / 基站的位置、质量和标签 |
| 核心模块 | 搜索框、结果列表、详情卡、地图预留区 |
| 关键功能 | 按 `cell_id`、运营商、LAC、坐标范围查询 |
| 设计重点 | 返回结果必须包含位置、精度、状态、异常提示 |

## 页面二：覆盖分析

| 项目 | 说明 |
|------|------|
| 用户目的 | 看某个区域的覆盖密度、质量结构和运营商差异 |
| 核心模块 | 区域选择、筛选器、密度图、质量分布图、运营商对比表 |
| 关键功能 | 按运营商、制式、区域范围切换分析 |
| 设计重点 | 页面重点是区域问题，不是单站点问题 |

## 页面三：统计报表

| 项目 | 说明 |
|------|------|
| 用户目的 | 拿到可复用的汇总结果 |
| 核心模块 | 汇总表格、趋势图、导出入口、版本说明 |
| 关键功能 | 按区域 / LAC / 运营商汇总，支持导出 |
| 设计重点 | 必须保留版本标识，保证报表可追溯 |

## 页面组成功标准

| 标准 | 表现 |
|------|------|
| 查得到 | 用户能快速找到目标站点 |
| 看得懂 | 区域覆盖和质量结构清晰可读 |
| 导得出 | 报表能作为业务结果复用 |
| 追得回 | 页面顶部始终显示数据集、运行批次和快照版本 |
