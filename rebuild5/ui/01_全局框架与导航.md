# 全局框架与导航

本文件定义整个 UI 的总壳，不描述单页细节，而描述所有页面共享的框架、导航和跨页跳转关系。

## 页面总壳

| 区域 | 作用 |
|------|------|
| 顶部版本条 | 固定显示当前数据集、运行批次、快照版本、发布状态、更新时间 |
| 左侧导航 | 按“全局管理 / 治理控制台 / 服务控制台”三段组织页面 |
| 主内容区 | 当前页面的概览、列表、图表、明细 |
| 详情抽屉 | 打开单个对象、单次运行、单条规则的解释 |
| 面包屑 | 告诉用户当前位于哪个分区、哪个页面 |

## 顶部版本条

页面顶部应先解决“现在看的是谁”的问题。

| 字段 | 必须性 | 说明 |
|------|--------|------|
| 数据集标识（`dataset_key`） | 必须 | 从 Step 2 开始所有页面都要显示 |
| 运行批次编号（`run_id`） | 必须 | 便于把页面和运行历史对上 |
| 快照版本号 / 引用版本 | 必须 | 说明页面看到的是哪个结果版本 |
| 页面状态 | 必须 | 运行中 / 已完成 / 验证中 / 已发布 / 已回退 |
| 最近更新时间 | 推荐 | 帮用户判断是否需要刷新 |
| 影响范围 | 按需 | 如当前是否是局部重算、某个 LAC 范围 |

- “运行历史”既是全局版本页，也是控制侧链入口页。

## 通用页面结构

| 结构层 | 说明 |
|--------|------|
| 概览层 | 卡片、核心指标、状态分布 |
| 解释层 | 规则命中、差异摘要、来源说明 |
| 明细层 | 表格、对象列表、运行列表 |
| 深入层 | 详情抽屉、对象关系、跳转到上下游页面 |

## 通用交互规则

| 交互 | 作用 |
|------|------|
| 卡片点进去看列表 | 从总量跳到可操作对象集合 |
| 列表展开看详情 | 从对象集合跳到单对象解释 |
| 跨页跳转 | 从一个问题跳到解决该问题的下一页 |
| 版本切换 | 切换当前数据集或运行版本后，所有页面同步更新 |
| 对比入口 | 在运行历史、流转快照、维护页中支持新旧版本对比 |

## 跨页跳转规则

| 当前页面 | 常见下一步 |
|----------|------------|
| 数据集选择 | 运行历史 |
| 运行历史 | ETL 页面、流转总览、服务页面 |
| 基础画像与分流 | 知识补数、流转总览 |
| 流转总览 | 流转快照、观察工作台、Cell/BS/LAC 评估 |
| Cell 评估 | 知识补数、Cell 维护 |
| 知识补数 | Cell 维护 |
| Cell / BS / LAC 维护 | 服务层页面 |

## 通用状态

| 状态 | 页面应如何表达 |
|------|----------------|
| 冷启动 | 明确说明“当前无上一轮正式库，因此没有 Path A / Step 4 补数” |
| 空数据 | 明确说明是“未运行”“过滤后为空”“当前版本无变化”中的哪一种 |
| 运行中 | 允许先展示当前已产出的统计，同时提示未完成 |
| 验证中 | 标注该版本还不能对外作为正式结果 |
| 已回退 | 保留历史可查，但明确不是当前版本 |

## 给后续设计工具的约束

| 约束 | 说明 |
|------|------|
| 不把治理页做成 BI 报表页 | 治理页要突出原因和规则，不只是图表堆叠 |
| 不把服务页做成运维页 | 服务页语言更接近业务查询，不强调内部算法术语 |
| 先做总壳，再做单页 | 先定导航和版本条，再逐页设计内容 |
