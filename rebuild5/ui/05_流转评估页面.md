# 流转评估页面

本文件定义 Step 3 的页面组：总流转页 + Cell/BS/LAC 三个独立流转页，共 4 页。

## 页面组定位

| 项目 | 说明 |
|------|------|
| 页面组角色 | 展示对象如何在 waiting / observing / qualified / excellent 中流动 |
| 主要用户 | 治理分析师 |
| 核心问题 | 整体收敛情况、晋升规则影响、不同批次间变化 |
| 页面数量 | 4 页 |

## 共享组件：批次选择器

4 个页面顶部统一包含批次选择器，允许查看不同数据切片：

| 组件 | 说明 |
|------|------|
| 批次下拉 | 从 `GET /api/evaluation/batches` 获取可用批次列表，默认选最新批次 |
| 显示信息 | `snapshot_version`、`run_at`、`dataset_key` |
| 切换行为 | 选择不同批次后，页面所有数据刷新为该批次的 snapshot 数据 |

```text
┌──────────────────────────────────────────┐
│  批次: [v3 - 2026-04-09 14:30 ▼]        │
│  数据集: sample_6lac                      │
└──────────────────────────────────────────┘
```

## 页面一：总流转页

| 项目 | 说明 |
|------|------|
| 路由 | `/evaluation/overview` |
| 用户目的 | 快速理解三层对象整体状态 |
| 设计重点 | 管理驾驶舱，先总览，不展开单对象 |

### 模块

| 模块 | 内容 |
|------|------|
| 三层状态卡片 | Cell/BS/LAC 各自的状态分布（数量 + 占比） |
| 关键指标 | Cell 总数、锚点 Cell 数、qualified+ BS 数、qualified LAC 数 |
| Diff 摘要 | 本批 vs 上批：新增/晋升/降级/资格变化/质心偏移/不变 |
| 跨批次趋势 | 折线图：各状态数量随批次变化 |
| 导航入口 | 跳转到 Cell/BS/LAC 流转页 |

### 消费的 API

| 端点 | 用途 |
|------|------|
| `GET /api/evaluation/batches` | 批次列表 |
| `GET /api/evaluation/stats?batch_id=` | 状态分布 + diff 摘要 |
| `GET /api/evaluation/trend` | 跨批次趋势数据 |

## 页面二：Cell 流转页

| 项目 | 说明 |
|------|------|
| 路由 | `/evaluation/cell` |
| 用户目的 | 理解 Cell 状态、规则影响、批次间变化 |
| 设计重点 | 三层结构：状态数据 → 规则影响 → 批次趋势 |

### 第一层：状态数据

| 模块 | 内容 |
|------|------|
| 状态分布 | waiting / observing / qualified / excellent 数量 + 占比（条形图或饼图） |
| Cell 列表 | 分页表格，支持按状态/运营商/LAC/anchor_eligible/baseline_eligible 筛选 |
| 列表字段 | cell_id, lac, operator, tech, lifecycle_state, position_grade, independent_obs, distinct_dev_id, p90_radius_m, observed_span_hours, anchor_eligible |
| 展开详情 | 单行展开查看完整画像（质心坐标、信号均值、gps_valid_count 等） |
| 晋升差距 | waiting/observing 状态的 Cell 显示"距 qualified 还差什么"（观测量/设备数/跨度/P90/碰撞阻断） |

### 第二层：规则影响

展示当前晋升规则的阈值及其影响面，帮助用户理解"调哪个阈值影响最大"。

| 模块 | 内容 |
|------|------|
| 规则展示 | 从 yaml 读取的当前阈值：lifecycle.active.*, anchorable.*, position_grade.excellent.* |
| 影响分析 | 每个阈值条件"卡住了多少 waiting/observing Cell" |

影响分析示例：

| 阈值条件 | 当前值 | 卡住数量 | 说明 |
|----------|--------|----------|------|
| `independent_obs >= 3` | `3` | `820` | 观测量不足 |
| `distinct_dev_id >= 2` | `2` | `456` | 设备数不足 |
| `p90_radius_m < 1500` | `1500m` | `89` | 空间质量不足 |
| `observed_span_hours >= 24` | `24h` | `112` | 观测跨度不足 |
| 碰撞阻断 | `is_collision_id=true` | `34` | 碰撞 cell_id |

数据来源：`GET /api/evaluation/cells/rule-impact?batch_id=`

### 第三层：批次趋势

| 模块 | 内容 |
|------|------|
| 趋势图 | Cell 各状态数量随批次变化的折线图 |
| 锚点趋势 | anchor_eligible Cell 数随批次增长曲线 |

### 消费的 API

| 端点 | 用途 |
|------|------|
| `GET /api/evaluation/batches` | 批次列表 |
| `GET /api/evaluation/cells?batch_id=&page=&lifecycle_state=` | Cell 列表 |
| `GET /api/evaluation/cells/:id?batch_id=` | 单 Cell 详情 |
| `GET /api/evaluation/cells/rule-impact?batch_id=` | 规则影响分析 |
| `GET /api/evaluation/trend` | 跨批次趋势 |

## 页面三：BS 流转页

| 项目 | 说明 |
|------|------|
| 路由 | `/evaluation/bs` |
| 用户目的 | 理解 BS 由 Cell 上卷的晋升逻辑 |
| 设计重点 | 强调"BS 完全由下属 Cell 上卷而来" |

### 第一层：状态数据

| 模块 | 内容 |
|------|------|
| 状态分布 | waiting / observing / qualified 数量 + 占比 |
| BS 列表 | 分页表格，列：bs_id, lac, operator, lifecycle_state, total_cells, qualified_cells, excellent_cells, classification |
| 展开详情 | 展开看下属 Cell 列表（cell_id, lifecycle_state, position_grade, independent_obs） |
| 晋升进度 | 如 "已有 2/3 个 qualified+ Cell" |

### 第二层：规则影响

| 阈值条件 | 当前值 | 卡住数量 | 说明 |
|----------|--------|----------|------|
| `下属 excellent Cell >= 1` | `1` | `N` | 缺少 excellent Cell |
| `下属 qualified+ Cell >= 3` | `3` | `M` | qualified+ Cell 不足 |

数据来源：`GET /api/evaluation/bs/rule-impact?batch_id=`

### 第三层：批次趋势

BS 各状态数量随批次变化的折线图。

### 消费的 API

| 端点 | 用途 |
|------|------|
| `GET /api/evaluation/batches` | 批次列表 |
| `GET /api/evaluation/bs?batch_id=&page=` | BS 列表 |
| `GET /api/evaluation/bs/:id?batch_id=` | 单 BS 详情（含下属 Cell） |
| `GET /api/evaluation/bs/rule-impact?batch_id=` | 规则影响分析 |
| `GET /api/evaluation/trend` | 跨批次趋势 |

## 页面四：LAC 流转页

| 项目 | 说明 |
|------|------|
| 路由 | `/evaluation/lac` |
| 用户目的 | 理解区域整体质量 |
| 设计重点 | 区域质量看板，不是单对象详情页 |

### 第一层：状态数据

| 模块 | 内容 |
|------|------|
| 状态分布 | waiting / observing / qualified 数量 + 占比 |
| LAC 列表 | 分页表格，列：lac, operator, lifecycle_state, total_bs, qualified_bs, qualified_bs_ratio, area_km2, anomaly_bs_ratio |
| 展开详情 | 展开看下属 BS 列表（bs_id, lifecycle_state, total_cells, qualified_cells） |
| 晋升进度 | 如 "还差 1 个 qualified BS" 或 "占比还差 3%" |

### 第二层：规则影响

| 阈值条件 | 当前值 | 卡住数量 | 说明 |
|----------|--------|----------|------|
| `qualified BS >= 3` | `3` | `N` | qualified BS 数量不足 |
| `qualified BS 占比 >= 10%` | `10%` | `M` | 占比不足 |

数据来源：`GET /api/evaluation/lac/rule-impact?batch_id=`

### 第三层：批次趋势

LAC 各状态数量随批次变化的折线图。

### 消费的 API

| 端点 | 用途 |
|------|------|
| `GET /api/evaluation/batches` | 批次列表 |
| `GET /api/evaluation/lac?batch_id=&page=` | LAC 列表 |
| `GET /api/evaluation/lac/:id?batch_id=` | 单 LAC 详情（含下属 BS） |
| `GET /api/evaluation/lac/rule-impact?batch_id=` | 规则影响分析 |
| `GET /api/evaluation/trend` | 跨批次趋势 |

## 页面间关系

```text
总流转页
  ├── Cell 流转页
  ├── BS 流转页
  └── LAC 流转页
```

总流转页是入口，用户从卡片区跳转到对应层级的流转页查看详情。

## 废弃页面

以下页面合并到上述 4 页中，不再作为独立页面：

| 废弃页面 | 合并去向 |
|----------|----------|
| 流转快照 (`/evaluation/snapshot`) | Diff 摘要合并到总流转页 |
| 观察工作台 (`/evaluation/watchlist`) | 晋升差距诊断合并到 Cell 流转页的状态筛选 |

## 页面成功标准

| 标准 | 表现 |
|------|------|
| 先总后分 | 总流转页看全局，分页看细节 |
| 规则可解释 | 每条晋升规则卡住了多少对象，一目了然 |
| 时间可追溯 | 切换批次看不同时间切片，趋势图看收敛 |
| 三层有边界 | Cell / BS / LAC 各自关注不同粒度和上卷逻辑 |
