# 系统配置页面

本文件定义三页配置页：晋级规则、防毒化规则、数据保留策略。

## 页面组定位

| 项目 | 说明 |
|------|------|
| 页面组角色 | 展示当前系统采用的核心阈值与策略 |
| 主要用户 | 系统管理员、治理分析师 |
| 当前阶段 | 以只读展示为主，后续可扩展为可编辑 |
| 设计重点 | 配置页回答“当前规则是什么”，不直接承载运行结果 |

## 页面一：晋级规则

| 项目 | 说明 |
|------|------|
| 用户目的 | 看 `waiting / observing / qualified / excellent` 的阈值定义 |
| 核心模块 | Cell 规则、BS 规则、LAC 规则、锚点资格、基线资格 |
| 关键功能 | 按层级查看规则，跳转到相关评估页 |
| 设计重点 | 把规则写成业务可理解语言，不直接堆配置键 |

## 页面二：防毒化规则

| 项目 | 说明 |
|------|------|
| 用户目的 | 看哪些异常会阻断基线刷新 |
| 核心模块 | 异常类型说明、阻断条件、典型影响对象、相关维护页入口 |
| 关键功能 | 查看规则命中说明，跳转到 Cell / BS / LAC 维护页 |
| 设计重点 | 强调“为什么 baseline_eligible 会被关掉” |

## 页面三：数据保留策略

| 项目 | 说明 |
|------|------|
| 用户目的 | 看 waiting 清理、dormant、retired 的策略 |
| 核心模块 | 等待态清理、已入库对象静默处理、退出链路说明 |
| 关键功能 | 查看策略值、查看影响范围说明 |
| 设计重点 | 强调这是生命周期管理规则，不是数据源删除规则 |

## 页面成功标准

| 标准 | 表现 |
|------|------|
| 规则看得懂 | 用户不用翻配置文件也能知道当前阈值 |
| 能关联运行结果 | 从规则页能跳到被该规则影响的页面 |
| 只读边界清晰 | 当前版本先展示，不暗示已开放在线编辑 |
