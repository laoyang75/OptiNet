# 目录清理归档

## 目标

整理 `/Users/yangcongan/cursor/WangYou_Data/` 根目录，将历史产物归档，保留核心资产，为 rebuild5 开发建立干净的工作环境。

## 原则

- **不删除任何文件**，只移动到归档目录
- rebuild2 和 rebuild4 是核心资产，保留在原位
- rebuild5 是当前活跃开发目录，保留在原位
- 根目录下不应有散落的 .png/.md/.sql/.py 文件

## 操作步骤

### 1. 创建归档目录

```bash
mkdir -p /Users/yangcongan/cursor/WangYou_Data/_archive/loose_files
mkdir -p /Users/yangcongan/cursor/WangYou_Data/_archive/early_versions
mkdir -p /Users/yangcongan/cursor/WangYou_Data/_archive/standalone_tools
mkdir -p /Users/yangcongan/cursor/WangYou_Data/_archive/playwright_reports
```

### 2. 移动根目录散落文件（约 60 个）

```bash
cd /Users/yangcongan/cursor/WangYou_Data

# 截图文件
mv *.png _archive/loose_files/

# playwright 报告
mv playwright-*.md _archive/playwright_reports/

# 散落的 SQL 脚本
mv create_indexes.sql generate_l0_*.sql _archive/loose_files/

# 散落的 Python 脚本
mv execute_l0.py execute_l0_nohup.py _archive/loose_files/

# chrome 日志
mv chrome.log _archive/loose_files/
```

### 3. 移动早期版本目录

这些是 rebuild2 之前的早期尝试，已无活跃开发：

```bash
# 早期版本
mv rebuild/ _archive/early_versions/rebuild1/
mv rebuild3/ _archive/early_versions/

# 根目录下的独立 backend/frontend（不属于任何 rebuild）
mv backend/ _archive/standalone_tools/root_backend/

```

### 4. 移动独立工具和文档

```bash
# 独立工具
```

### 5. 移动配置文件

```bash
# .gemini 和 .playwright-mcp 配置
mv .gemini/ _archive/standalone_tools/
mv .playwright-mcp/ _archive/standalone_tools/
```

### 6. 保留 launcher.py 在根目录

`launcher.py` 是启动器，检查一下它启动的是哪个 rebuild 版本。如果是 rebuild4 专用的，移到 rebuild4 下；如果是通用的，保留在根目录。

```bash
# 先检查内容
head -20 launcher.py
# 根据内容决定：
# 如果是 rebuild4 专用: mv launcher.py rebuild4/
# 如果是通用启动器: 保留
```

清理后根目录应该只剩：


```bash
# 验证

```

- **不要删除 _archive 中的任何内容** — 只是移走，不丢弃
