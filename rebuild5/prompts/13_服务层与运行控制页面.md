# 服务层与运行控制页面 Prompt

## 背景

审计和文档重构后，Doc 06（结果库浏览与查询）和 Doc 09（管道运行与重跑）已更新。现有页面需要对齐新文档。

两组页面的分工：
- **服务层页面**（Doc 06）：面向业务用户的结果库查询，回答"基站在哪、精度如何、区域覆盖怎样"
- **运行控制页面**（Doc 09）：面向管道管理员的运行触发和历史管理，回答"跑哪个数据集、从哪步开始、状态如何"

**与治理页面（prompt 12）的区别**：治理页面聚焦变化和异常（碰撞/漂移/防毒化/退出），是操作导向；服务层聚焦存量全貌，是报告导向；运行控制聚焦触发和追溯。

## 当前约束（覆盖旧指令）

- 数据集页面当前是“单活数据集模式”说明页，不支持在线切换。
- `DatasetSelect.vue` 只展示元数据、当前上下文和当前数据集准备入口；不要实现“每个卡片切换/运行”的交互。
- 如文档中旧描述仍出现“切换数据集”或“重建 sample_6lac”，以本节为准。

### 后端 API（已就绪）

#### 服务层 API
| 端点 | 说明 |
|------|------|
| `GET /api/service/search?q=&level=&operator_code=&page=&page_size=` | 按 Cell/BS/LAC 搜索 |
| `GET /api/service/cell/{cell_id}` | Cell 画像详情（SELECT *） |
| `GET /api/service/bs/{bs_id}` | BS 画像 + 子 Cell 列表 |
| `GET /api/service/lac/{lac}` | LAC 画像 + 子 BS 列表 |
| `GET /api/service/coverage` | 覆盖统计（总量 + 运营商对比） |
| `GET /api/service/report` | LAC 粒度报表 |

#### 运行控制 API
| 端点 | 说明 |
|------|------|
| `GET /api/system/config` | 当前数据集配置、版本、参数 |
| `GET /api/system/run-log` | 运行历史列表 |
| `POST /api/system/prepare-current-dataset` | 数据准备（按当前 dataset.yaml 合并去重） |
| `POST /api/pipeline/run?from_step=N` | **运行管道**，from_step: 1=全量, 2=Step2-5, 5=仅Step5 |

#### 前端 API 客户端

文件：`rebuild5/frontend/design/src/api/service.ts`（服务层，已完整）
文件：`rebuild5/frontend/design/src/api/system.ts`（运行控制，已完整）

`api/system.ts` 新增的函数：
```typescript
export function runPipeline(fromStep: number = 1): Promise<PipelineRunPayload>
```

**重要**：所有 API 返回的 `meta.dataset_key` 现在由后端 `envelope.py` 从 `config/dataset.yaml` 动态读取，不再硬编码。前端页面中 hardcoded 的 `dataset_key: 'sample_6lac'` 默认值需要相应清理。

## 改造任务

### 1. StationQuery.vue 改造

- 搜索框 + Cell/BS/LAC 层级切换
- 搜索结果表格（按 P90 升序）
- Cell/BS/LAC 详情卡（点击展开）

#### 改造点

**a. Cell 详情卡增强**

当前只展示 6 个字段，需补全为完整画像：

| 字段组 | 字段 | 说明 |
|--------|------|------|
| 标识 | operator_code / operator_cn / lac / bs_id / cell_id / tech_norm | 已有 |
| 位置 | center_lon / center_lat | 已有 |
| 精度 | p50_radius_m / p90_radius_m | 已有 p90，补 p50 |
| 质量 | position_grade / lifecycle_state | 需补 |
| 资格 | anchor_eligible / baseline_eligible | 需补 |
| 标签 | drift_pattern / is_collision / is_dynamic / is_multi_centroid | 需补 |
| 信号 | rsrp_avg / rsrq_avg / sinr_avg / pressure_avg | **新增** |
| 活跃度 | independent_obs / unique_devices / active_days_30d | **新增** |

后端 `GET /api/service/cell/{cell_id}` 返回 `SELECT *`，所有字段已可用。

布局：用 `detail-grid` 分 4 列或 3 列展示，标签字段用 `StatusTag` 组件，碰撞/动态标签用红色警告文字。
