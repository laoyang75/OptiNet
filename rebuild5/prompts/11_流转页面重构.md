# 流转评估页面重构 Prompt

## 背景

Step 3 流转评估页面需要从 6 页合并重构为 4 页，并增加批次切片能力。

## 设计文档

完整设计见 `rebuild5/ui/05_流转评估页面.md`。

## 当前状态

```
rebuild5/frontend/design/src/views/evaluation/
├── FlowOverview.vue   → 保留，改造为总流转页
├── CellEval.vue       → 保留，改造为 Cell 流转页
├── BSEval.vue         → 保留，改造为 BS 流转页
├── LACEval.vue        → 保留，改造为 LAC 流转页
├── Snapshot.vue       → 删除，功能合并到 FlowOverview
└── Watchlist.vue      → 删除，功能合并到 CellEval
```

### 路由调整

文件：`rebuild5/frontend/design/src/router/index.ts`

- 删除 `/evaluation/snapshot` 和 `/evaluation/watchlist` 路由
- 保留 `/evaluation/overview`、`/evaluation/cell`、`/evaluation/bs`、`/evaluation/lac`

### API 调整（后端已完成）

文件：`rebuild5/frontend/design/src/api/evaluation.ts`

新增端点：
- `GET /api/evaluation/batches` — 返回 `{ batches: [{ batch_id, snapshot_version, run_at, dataset_key }] }`
- `GET /api/evaluation/cells/rule-impact?batch_id=` — 返回 `{ rules: {...}, impact: [{ rule, threshold, blocked, desc }] }`
- `GET /api/evaluation/bs/rule-impact?batch_id=` — 同上格式
- `GET /api/evaluation/lac/rule-impact?batch_id=` — 同上格式

所有现有列表/详情端点已支持 `?batch_id=` 可选参数（默认最新批次）。

## 重构任务

### 1. 共享批次选择器组件

创建 `components/common/BatchSelector.vue`：
- 调用 `GET /api/evaluation/batches` 获取可用批次列表
- 下拉选择，默认选最新批次
- 显示 `snapshot_version` 和 `run_at`
- 选中后 emit `batch-change` 事件，携带 `batch_id`
- 所有 4 个页面顶部引入此组件

### 2. FlowOverview.vue（总流转页）

改造要点：
- 顶部加 BatchSelector
- 三个卡片区（Cell / BS / LAC），每个显示状态分布数量 + 占比
- 关键指标：Cell 总数、锚点 Cell 数、qualified+ BS 数、qualified LAC 数
- Diff 摘要面板（从现有 Snapshot.vue 移入）：新增/晋升/降级/资格变化/质心偏移
- 跨批次趋势折线图（调用 `/api/evaluation/trend`）
- 导航按钮跳转到 Cell/BS/LAC 页面
- 所有数据请求带 `?batch_id=` 参数

### 3. CellEval.vue（Cell 流转页）

改造要点：

**第一层 - 状态数据**（改造现有内容）：
- 顶部加 BatchSelector
- 状态分布条形图/饼图
- Cell 列表保留现有表格，增加筛选：lifecycle_state, operator_code, lac, anchor_eligible
- 从 Watchlist.vue 移入晋升差距诊断：waiting/observing 的 Cell 显示 gap 文案

**第二层 - 规则影响**（新增）：
- 调用 `/api/evaluation/cells/rule-impact?batch_id=`
- 展示规则阈值表格：规则名、当前阈值、卡住数量、说明
- 阈值只读展示

**第三层 - 批次趋势**（新增）：
- 从 trend 数据中提取 Cell 层折线图
- X 轴 = 批次，Y 轴 = 各状态数量

### 4. BSEval.vue（BS 流转页）

改造要点：
- 顶部加 BatchSelector
- 第一层：现有 BS 列表 + 展开下属 Cell（保留）
- 第二层：调用 `/api/evaluation/bs/rule-impact?batch_id=`，展示 BS 晋升规则影响
- 第三层：BS 趋势折线图

### 5. LACEval.vue（LAC 流转页）

改造要点：
- 顶部加 BatchSelector
- 第一层：现有 LAC 列表 + 展开下属 BS（保留）
- 第二层：调用 `/api/evaluation/lac/rule-impact?batch_id=`，展示 LAC 晋升规则影响
- 第三层：LAC 趋势折线图

### 6. 删除废弃文件

- 删除 `Snapshot.vue`
- 删除 `Watchlist.vue`
- 从 `router/index.ts` 删除对应路由
- 从侧边栏 `AppLayout.vue` 删除对应菜单项

### 7. API 客户端更新

文件：`rebuild5/frontend/design/src/api/evaluation.ts`

新增函数：
```typescript
export function fetchBatches(): Promise<...>
export function fetchCellRuleImpact(batchId?: number): Promise<...>
export function fetchBSRuleImpact(batchId?: number): Promise<...>
export function fetchLACRuleImpact(batchId?: number): Promise<...>
```

所有现有 fetch 函数增加可选 `batchId` 参数，拼接到 URL query string。

## 技术约束

- Vue 3 + TypeScript
- 不引入新 UI 库，使用现有自定义样式
- 趋势图可用简单 SVG 或 CSS bar chart，不要求引入图表库
- 保持现有分页组件和筛选模式

## 设计与验证要求

### 设计一致性

使用 `/frontend-design` skill 进行页面开发，确保与现有页面风格一致。现有设计系统参考：
- CSS 变量：`--c-primary` / `--c-border` / `--c-surface` / `--c-bg` / `--c-text-muted` 等（见 `style.css`）
- 状态颜色：excellent=绿 / qualified=蓝 / observing=黄 / waiting=灰 / dormant=橙 / retired=红
- 通用组件：`SummaryCard` / `StatusTag` / `PageHeader` / `Pagination` / `PercentBar`（见 `components/common/`）
- 布局类：`.card` / `.data-table` / `.grid` / `.grid-3` / `.grid-4` / `.gap-lg` / `.flex`
- 字体：`.font-mono` / `.font-semibold` / `.text-xs` / `.text-muted` / `.text-secondary`

**严格要求**：所有改造在现有 Vue 文件上直接修改，不要新建独立页面再整合。合并功能（如 Snapshot → FlowOverview）通过把代码移入目标文件实现，不是新建文件再替换。

### Playwright 验证

每个页面改造完成后，使用 Playwright MCP 工具进行截图验证：
1. 启动前端开发服务器（如未启动）
2. 导航到对应页面路由
3. 截图确认：布局正确、组件渲染正常、风格与其他页面一致
4. 如发现风格偏差，立即修正

## 执行方式

本 prompt 可在新会话中直接执行。也可以通过 Agent 工具作为子 agent 执行以加速并行处理。

如需并行执行多个 prompt（11/12/13），注意：
- 三个 prompt 改造的页面目录不同（evaluation / governance / service+global），无文件冲突
- 共享文件（router/index.ts、AppLayout.vue）的修改需要合并，建议最后统一处理
- 每个 prompt 独立完成页面改造 + Playwright 验证后，再统一更新路由和侧边栏
