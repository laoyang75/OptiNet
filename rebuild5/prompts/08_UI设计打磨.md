# rebuild5 UI 设计打磨 Prompt

**使用方式**：新开会话，将本文件内容作为首条消息发送。

---

## 启动前检查

在开始任何设计工作之前，你必须先完成以下检查：

1. **检查 Playwright MCP 是否可用**：尝试调用 `mcp__playwright__browser_navigate` 或 `mcp__playwright__browser_snapshot`，确认浏览器可以正常启动和截图。如果不可用，提示用户修复后再继续。
2. **阅读项目文档**：依次阅读以下文件，理解系统全貌：
   - `rebuild5/docs/00_全局约定.md` — 实体模型、状态定义、颜色映射、技术栈
   - `rebuild5/docs/08_UI.md` — UI 页面定义（如果存在）
   - `rebuild5/docs/02_基础画像.md` ~ `rebuild5/docs/05_画像维护.md` — 各 Step 的功能、输入输出、页面要求

## 项目上下文

- **项目名称**：rebuild5 — 基站定位数据治理系统
- **技术栈**：后端 Python FastAPI + PostgreSQL 17，前端 Vue 3 + TypeScript
- **系统结构**：6 个主步骤（Step 1-6），每个步骤有独立 UI 页面
- **UI 定位**：数据治理工作台，不是 BI 报表。用户是数据工程师和网络优化工程师，关注规则命中、异常检测、数据流转状态
- **语言**：中文为主界面，字段名和枚举值保留英文

## 设计方向

- **风格**：简洁、克制、浅色主题
- **信息密度**：高但不拥挤，治理数据需要表格 + 摘要卡片 + 图表结合
- **状态颜色**（已冻结，不可更改）：
  - `excellent` = 绿（Green）
  - `qualified` = 蓝（Blue）
  - `observing` = 黄（Yellow）
  - `waiting` = 灰（Gray）
  - `dormant` = 橙（Orange）
  - `retired` = 红（Red）
- **交互风格**：规则区只读展示（后续可扩展为可编辑）；表格支持筛选和排序；层级关系支持展开折叠（BS → Cell，LAC → BS）

## 设计输入

- **主要依据**：`rebuild5/docs/08_UI.md`
- **补充依据**：各 Step 文档中的"UI 页面要求"章节（02 ~ 05 文档末尾均有定义）
- **如果 `08_UI.md` 的设计不够好**：你可以根据上述开发文档中已经写明的页面要求自行设计，文档对每个页面的模块、字段、展示逻辑描述得很清楚，足以支撑独立设计

## 输出要求

### 输出目录

所有设计产物输出到 `rebuild5/frontend/design/`

### 输出内容

对每个页面输出：

1. **功能描述**（该页面解决什么问题、用户在这个页面做什么）
2. **组件拆解**（页面由哪些组件构成、每个组件的职责）
3. **数据流说明**（页面消费哪些 API 端点、展示哪些字段）
4. **Vue 组件代码**（可运行的 Vue 3 + TypeScript 组件）

### 质量检查

每个页面完成后，你必须：

1. 使用 Playwright 在浏览器中打开页面
2. 截图检查布局、颜色、间距、文字是否符合设计方向
3. 如果发现问题，自行修复后再次截图确认
4. 确认无误后再进入下一个页面

## 设计工作流

建议按以下顺序执行：

1. 先运行 `/teach-impeccable` 建立设计上下文（使用上述项目信息）
2. 设计全局布局框架（导航、侧边栏、头部、内容区）
3. 设计通用组件（状态标签、摘要卡片、数据表格、筛选器）
4. 逐页面设计，每页完成后用 Playwright 验证
5. 全部完成后做一次整体 `/audit` 检查
