## 核心概念

### 这个步骤的定位

Step 5 是可信赖库的**深度维护引擎**。Step 3 负责"准入"，Step 5 负责"持续治理"。

Step 5 的四大职责：
1. **维护进入**：全局碰撞检测、多质心分析、面积异常处理
2. **异常处理**：GPS 异常时序判定（迁移 vs 漂移 vs 碰撞）、异常标签
3. **特征标签**：漂移分类、动态 Cell、分类标签
4. **维护退出**：滑动窗口数据保鲜、静默检测、退出确认

### 三层各自维护

Cell/BS/LAC 各自有独立的维护逻辑和数据窗口：

| 层级 | 维护重点 | 数据窗口 |
|------|---------|---------|
| Cell | 质心精度、碰撞、迁移、GPS 异常、漂移分类 | 保留最近 N 天或最少 M 条原始观测 |
| BS | 质心准确性、多质心检测、面积异常、下属 Cell 健康度 | 需要满足一定上报量（来自下属 Cell 的聚合） |
| LAC | 区域完整性、异常 BS 比例、边界稳定性 | 主要是 BS 级的单条聚合记录 |

### 定时执行

Step 5 建议每天执行一次（可配置），对可信赖库做全量维护。不是每批数据都触发（那是 Step 3 的事），而是定时的深度分析。

### 多数据源考虑

考虑未来接入多种数据源，Step 5 的维护结果需要展示：
- 不同数据源的单独结果
- 综合结果（多源合并后的画像）
- 补数前 vs 补数后的质量对比

## 文档结构要求

```
# Step 5: 画像维护

## 模块定位
  - 可信赖库的深度维护引擎
  - 四大职责：维护进入、异常处理、特征标签、维护退出
  - 定时执行（建议每天一次），不是每批触发
  - 三层（Cell/BS/LAC）各自独立维护

## 5.1 全局碰撞检测（collision_id_list 产出）
  ### 检测范围
  - 扫描可信赖库中所有 cell_id
  - 找出同一 cell_id 出现在不同 (operator_code, lac) 组合下的情况
  
  ### 检测逻辑
  - 按 cell_id 分组，检查是否有多个 (operator_code, lac) 组合
  - 有 → 标记 is_collision_id = true
  - 没有 → is_collision_id = false
  
  ### 产出
  - collision_id_list 表：cell_id + is_collision_id + 涉及的 (operator, lac) 组合列表
  - 每个碰撞 cell_id 记录各组合的质心位置（用于 Step 2 碰撞防护的 GPS 比对）
  - 这个表供 Step 2 读取，做碰撞局部防护
  
  ### 碰撞统计
  - 碰撞 cell_id 总数
  - 碰撞涉及的运营商/LAC 分布

## 5.2 Cell 维护
  ### 5.2.1 面积异常 Cell 的多质心分析
  - 只对 P90 过大或面积异常的 Cell 做（不全量计算）
  - 阈值：P90 > 某值（可配置）→ 触发多质心分析
  - 分析方法：对观测点做聚类，判断是否存在多个物理位置
  - 结果：标记 is_multi_centroid，记录各质心位置和所属观测比例
  - 如果是多质心 → 可能是碰撞（同 cell_id 多个物理位置）
  
  ### 5.2.2 GPS 异常时序判定
  - 输入：Step 4 产出的 gps_anomaly_log（GPS 与可信质心偏差过大的记录）
  - 按 Cell 累积时间序列分析：
    · 连续多批出现 GPS 异常 → gps_anomaly_type = 'migration_suspect'（迁移嫌疑）
    · 偶尔出现 → gps_anomaly_type = 'drift'（GPS 漂移）
    · 集中在特定时段 → gps_anomaly_type = 'time_cluster'（时段性异常）
  - 漂移处理：GPS 漂移时，信号参数仍然可用，可以信赖
    把 Cell 的可信质心赋予给这条报文的 GPS，过滤掉异常 GPS
  - 连续异常处理：可能是碰撞、迁移或移动 Cell
    · 如果多质心分析确认多位置 → 碰撞确认
    · 如果质心持续单向移动 → 迁移确认
    · 如果设备数极少（1-2台）→ 可能是移动基站

  ### 5.2.3 防毒化（独立子步骤）
  防毒化不只检测 GPS 异常，而是检测任何指标的异常突变：
  
  机制：
  - 新数据加入 Cell 的滑动窗口后，先试算新画像
  - 与旧画像（上一次维护结果）对比
  - 变化在容忍范围内 → 正常更新
  - 变化超出阈值 → 标记待审核，画像暂不更新
  
  检测维度（每个都有独立可配置阈值，来自 antitoxin_params.yaml）：
  - 质心漂移：新旧质心距离 > max_shift_m → 异常
  - P90 膨胀：新 P90 / 旧 P90 > max_ratio → 异常（防止错误数据把半径撑大）
  - 设备突增：新设备数 / 旧设备数 > max_ratio → 异常（可能是 cell_id 复用）
  - 观测模式异常：某时段观测量占比 > max_single_window_ratio → 异常
  
  处理方式：
  - 触发防毒化 → baseline_eligible 暂时设为 false
  - 不影响 is_registered 和 lifecycle_state
  - 待人工或后续批次数据验证后恢复

  ### 5.2.4 漂移分类（完整标签）
  - 基于多日质心轨迹的 max_spread 和 net_drift/ratio
  - 分类：stable / collision / migration / large_coverage / moderate_drift / insufficient
  - 阈值从 profile_params.yaml 读取（全局约定中已定义）
  
  ### 5.2.6 滑动窗口数据保鲜
  - 每个 Cell 维护一个滑动窗口的原始观测数据
  - 窗口大小：最近 N 天（可配置，默认 7 天）或最少 M 条记录（可配置，默认 50 条）
  - 取二者的较大范围
  - 每次维护时，基于窗口内的数据重新计算画像
  - 超出窗口的数据归档（保留聚合统计，删除明细）
  - 这确保画像反映的是"最近"的真实状态，不是历史累积的旧数据

  ### 5.2.7 退出管理
  - 静默检测：Cell 在窗口期内观测量持续下降或完全无新数据
    · 一个设备平均每天上报一次，连续 7 天不上报 → 可能无效
    · 具体阈值可配置
  - dormant 标记：满足静默条件 → lifecycle_state = 'dormant'
  - retired 确认：dormant 持续 M 天仍无新数据 → lifecycle_state = 'retired'
  - retired 处理：从可信赖库移除，画像归档
  - 重新激活：retired Cell 再次出现数据 → 回到 Step 2 重新积累

## 5.3 BS 维护
  ### 5.3.1 BS 质心和面积精确化
  - 利用已过滤的可信 Cell 数据重新计算 BS 质心
  - 排除碰撞 Cell、动态 Cell 后的质心更准确
  - 对面积过大的 BS 做多质心检测（只处理异常子集）
  
  ### 5.3.2 BS 分类标签
  - collision_bs：下属有碰撞确认的 Cell
  - dynamic_bs：下属有动态 Cell
  - large_spread：Cell→BS 最大距离 > 2500m
  - normal_spread：正常
  - multi_centroid：多质心 BS
  
  ### 5.3.3 BS 数据窗口
  - BS 的窗口 = 下属 Cell 窗口的聚合
  - BS 需要满足一定上报量：下属 Cell 在窗口内有足够活跃的观测
  - 如果所有下属 Cell 都进入 dormant → BS 也进入 dormant
  
  ### 5.3.4 BS 退出管理
  - 与 Cell 退出联动：所有下属 Cell 退出 → BS 退出
  - 独立超时：BS 自身无新的下属 Cell 活动 → dormant → retired

## 5.4 LAC 维护
  ### 5.4.1 LAC 区域完整性
  - LAC 的数据窗口主要是 BS 级的单条聚合记录
  - LAC 变动应该很小（区域级实体，物理上不常变化）
  - 关注指标：异常 BS 比例、边界稳定性、区域面积变化
  
  ### 5.4.2 LAC 退出管理
  - 与 BS 退出联动：足够多的 BS 退出 → LAC 进入 dormant
  - LAC 退出是极端情况（整个位置区被废弃）

## 5.5 进阶标签（只处理异常子集）
  - 不全量计算，只针对被标记为异常的实体
  - 全局 cell_id 碰撞扫描（5.1 已描述）
  - 多质心检测（仅 P90 过大的 BS/Cell）
  - 碰撞确认（结合多质心 + GPS 异常时序）
  - 迁移确认（结合质心持续单向移动 + GPS 异常时序）
  - 动态 Cell 识别（移动基站/车载/临时站点）

## 5.6 维护结果产出
  ### 更新可信赖库
  - 更新 trusted_cell_library：加入漂移分类、碰撞标记、动态标记、多质心标记、置信度分级、规模分级
  - 更新 trusted_bs_library：加入 BS 分类、多质心标记、精确化质心
  - 更新 trusted_lac_library：更新区域统计
  - 更新 collision_id_list：供 Step 2 读取
  
  ### 完整画像字段
  Step 5 之后的 trusted_cell_library 才是完整画像，包括 Step 3 没有的：
  - drift_pattern, drift_max_spread_m, drift_net_m, drift_days
  - is_collision, is_dynamic, is_multi_centroid
  - position_grade, gps_confidence, signal_confidence, cell_scale
  - gps_anomaly_type（来自 GPS 异常时序判定）
  
  ### 维护统计
  - 碰撞检测：碰撞 cell_id 数量、新发现 vs 已知
  - 多质心：触发分析的 Cell/BS 数量、确认多质心的数量
  - GPS 异常：迁移嫌疑、漂移、时段异常的分布
  - 退出：新 dormant 数量、新 retired 数量、重新激活数量
  - 数据窗口：窗口内平均观测量、归档数据量

## 5.7 多数据源展示
  ### 未来多数据源场景
  当接入多种数据源时，维护结果需要区分展示：
  - 各数据源的单独画像结果
  - 多源合并后的综合画像
  - 补数前（原始数据）vs 补数后（Step 4 补齐后）的质量对比
  
  ### 展示维度
  - 按数据源分组的覆盖率和质量统计
  - 补数贡献度：有多少画像改善来自知识补数
  - 数据源交叉：多个数据源对同一 Cell 的观测是否一致
```
