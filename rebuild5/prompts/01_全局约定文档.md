# 生成 rebuild5 全局约定文档

## 任务

基于 `rebuild5/README.md` 中"全局约定"部分的内容，生成一份完整的 `rebuild5/docs/00_全局约定.md` 文档。

## 输入材料

必须阅读：
1. `rebuild5/README.md` — 全局约定段落（实体层级、运营商编码、生命周期状态、三层资格、冻结快照原则、坐标常量、信号范围）
2. `rebuild4/backend/app/etl/profile_params.yaml` — 当前已验证的全部阈值参数（lifecycle、anchorable、position_grade、gps_confidence、signal_confidence、cell_scale、drift、bs_lifecycle、lac_lifecycle）
3. `rebuild4/docs/02_profile/05_pipeline.md` — 画像管道实现中的阈值清单和分级规则

## 文档要求

### 结构

```
# 全局约定

## 系统核心原则
  - 冻结快照原则（最重要，从 README 展开，补充具体场景示例）

  - 修正优于丢弃原则

  - 三层实体（LAC/BS/Cell）的完整定义、唯一键、层级关系
  - 实体间的包含关系和数量级（1 LAC ≈ 100-300 BS ≈ 500-3000 Cell）

## 生命周期状态
  - 状态定义（waiting/observing/qualified/excellent/dormant/retired）
  - 状态流转图（正向晋升 + 退出路径 + 重新激活）
  - Cell/BS/LAC 各自的晋升条件（全部从 profile_params.yaml 提取具体阈值）
  - BS 晋升如何依赖 Cell 质量
  - LAC 晋升如何依赖 BS 质量

## 三层资格
  - is_registered / anchor_eligible / baseline_eligible

  - gps_confidence（high/medium/low/none）+ 具体阈值
  - signal_confidence（high/medium/low/none）+ 具体阈值
  - cell_scale（major/large/medium/small/micro）+ 具体阈值

## 漂移分类
  - stable/collision/migration/large_coverage/moderate_drift/insufficient
  - 各分类的阈值和判定逻辑

## 物理常量
  - 坐标有效范围（中国边界）
  - 经纬度→米转换系数
  - 信号有效范围（RSRP/RSRQ/SINR）

## 运营商编码
  - 三大运营商的 operator_code 和中文名
  - tech_norm 制式分类

  - 配置文件命名和位置
  - 前端状态颜色映射

  - 后端：Python 3 + FastAPI + psycopg (PostgreSQL)
  - 前端：Vue 3 + TypeScript
  - 数据库：PostgreSQL 17
  - 配置：YAML 文件
  - 启动：launcher.py（前后端统一启动）
  - 文档语言：中文（技术术语保留英文）

## 配置文件结构
  - 完整列出 profile_params.yaml 的所有参数及其含义
  - 标注哪些是"算法常量"（不应修改），哪些是"业务阈值"（可调）
  - 未来扩展的 antitoxin_params.yaml 和 retention_params.yaml 的预期结构
```

- 状态流转用 ASCII 图
- 不写空泛的描述，每个约定都要有具体数值或具体规则
