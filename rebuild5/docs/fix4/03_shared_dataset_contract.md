# Fix4 共享样例数据集约束

## 目标

让不同研究线都在同一份原始样例数据上工作，保证结果可以横向比较。

## 核心要求

### 1. 原始样例输入必须一致

允许不同：

- 修复策略
- SQL 实现
- 参数设置
- 优化方法
- 结果解释

不允许不同：

- 原始样例数据成员
- 原始样例时间范围
- 原始样例输入表

### 2. 数据集确认权在主 agent

共享样例数据集由主 agent 最终确认。

所有研究线：

- 不能私自增加或减少原始样例 Cell
- 不能私自更换时间范围
- 不能私自换用另一套原始输入

### 3. 可以提补样建议，但不能直接改样例集

如果某位 agent 认为当前共享样例集不够：

- 可以输出补样建议
- 可以说明应新增哪些 Cell 和为什么

但不能：

- 直接把这些补样混入自己的实验
- 让两条研究线使用不同的样例版本

### 4. 共享基底固定，但允许派生工作集自由构建

共享原始样例表固定，不代表研究过程中不能再构建自己的派生数据集。

允许：

- 基于共享样例表创建派生表
- 基于共享重点 Cell 清单切分子集
- 针对性能 / 逻辑验证分别构建不同工作集

前提是：

1. 派生工作集必须来自共享原始样例表
2. 必须可复现
3. 必须在文档里明确说明构造逻辑

## 当前共享数据状态

共享样例数据已经**建好并写入**，不需要研究线自行建表或导入。

当前固定使用以下三张表：

1. `rebuild5_fix4.raw_gps_shared_sample`
2. `rebuild5_fix4.etl_cleaned_shared_sample`
3. `rebuild5_fix4.focus_cells_shared`

详见：

- `rebuild5/docs/fix4/04_shared_dataset_spec.md`

## 比对口径

后续比较 `Codex` 和 `Claude` 的输出时，只承认以下对比：

1. 同一份原始样例数据
2. 同一时间范围
3. 同样要求跑完整 `7` 次 ETL / 日批次
4. 同一版共享数据说明
