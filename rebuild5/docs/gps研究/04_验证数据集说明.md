# GPS 标签研究验证数据集

> 创建日期：2026-04-14
> 数据源：beijing_7d 数据集 batch 7（最新发布版本 v7）
> 用途：研究 6 类 GPS 异常标签的判定逻辑和修复策略

## 1. 数据集组成

### 表名

| 表 | 说明 | 行数 |
|----|------|------|
| `rebuild5.research_gps_cells` | cell 清单表，字段对齐 `trusted_cell_library` | **790** |
| `rebuild5.research_gps_records` | 原始观测记录表，字段对齐 `etl_cleaned` + 污染标记 | **299,049** |

### 标签分布

| 标签类别 | 选中数（行） | 独立 cell 数 | 采样规则 |
|---------|------------|-------------|---------|
| multi_centroid | 300 | 300 | 6 组合 × 50（10 top + 40 random） |
| standard | 300 | 300 | 6 组合 × 50（10 top + 40 random） |
| migration | 60 | 60 | 6 组合 × 10（3 top + 7 random） |
| dynamic | 60 | 60 | 6 组合 × 10（3 top + 7 random） |
| large_coverage | 60 | 60 | 6 组合 × 10（3 top + 7 random） |
| collision_substitute | 10 | 10 | 全局 p90 最大 × window_obs_count 降序 top10 |
| **合计** | **790** | **720 (唯一)** | |

注：允许同一 cell 出现在多个标签（如 multi_centroid 同时也是 dynamic），因此行数 790 > 独立 cell 数 720。

### 运营商 × 制式组合

每个主标签均覆盖：
- 移动 4G (46000, 4G)
- 移动 5G (46000, 5G)
- 联通 4G (46001, 4G)
- 联通 5G (46001, 5G)
- 电信 4G (46011, 4G)
- 电信 5G (46011, 5G)

## 2. 规模门槛（选择前置条件）

所有候选 cell 必须满足：
```
independent_obs >= 30
AND distinct_dev_id >= 10
AND active_days >= 3
```

## 3. 标签口径

| 标签 | 判定条件 |
|------|---------|
| multi_centroid | `is_multi_centroid = true` |
| migration | `drift_pattern = 'migration'` OR `centroid_pattern = 'migration'` |
| dynamic | `is_dynamic = true` |
| large_coverage | `drift_pattern = 'large_coverage'` |
| standard | `drift_pattern='stable'` AND `is_multi_centroid=false` AND `is_dynamic=false` AND `is_collision=false` AND `centroid_pattern IS NULL` AND `gps_anomaly_type IS NULL` |
| collision_substitute | batch 7 中 `is_collision = true` 为 0，用 `p90_radius_m DESC, window_obs_count DESC` top 10 替代 |

**重要发现**：

1. batch 7 中 `is_collision = true` 的 cell 为 0（碰撞标签在当前版本未产出数据）
2. **所有 `is_dynamic=true` 的 cell 都同时是 `is_multi_centroid=true`**，不存在"纯 dynamic"样本。这说明 dynamic 在当前实现上是 multi_centroid 的子集（触发条件重叠：max_spread > 1500m 且 drift_pattern 非 stable）

## 4. 关键字段

### 表 1：research_gps_cells

核心新增字段（相对于 trusted_cell_library）：
- `label_category`：所属标签类别
- `selection_method`：`top`（按数据量 top 选出）或 `random`（从剩余池随机）
- `vol_rank`：该 cell 在 (运营商, 制式, 标签) 组内按 window_obs_count 的排名

其余字段与 `trusted_cell_library` 完全一致。

### 表 2：research_gps_records

继承 `etl_cleaned` 全部字段（70 列），并新增：
- `gps_valid_original`：原始 gps_valid 值（未修复前）
- `gps_valid_fixed`：修复 ss1 污染后的 gps_valid
- `ss1_contaminated`：布尔值，标记是否为 ss1 污染记录

## 5. ss1 污染检查与修复

### 污染定义

`gps_valid_original = true` 但 `gps_info_type NOT IN ('1', 'gps')`（参见 `fix_ss1_gps.md`）

### 污染情况

| cell_origin | gps_info_type | 污染记录数 |
|-------------|---------------|-----------|
| ss1 | 5（网络定位） | 4,354 |
| ss1 | null | 1,861 |
| ss1 | 4（基站定位） | 1,747 |
| ss1 | 2（WiFi 定位） | 787 |
| **合计** | | **8,749** |

cell_infos 来源在 Step 1 ETL 解析时已正确过滤 gps_info_type，无污染。

### 修复操作

已在 `research_gps_records` 表中执行：
```sql
UPDATE rebuild5.research_gps_records
SET gps_valid_fixed = false,
    ss1_contaminated = true
WHERE gps_valid_original = true
  AND (gps_info_type IS NULL OR gps_info_type NOT IN ('1', 'gps'));
```

- 修复前 `gps_valid = true` 记录：260,228
- 修复后 `gps_valid = true` 记录：251,479
- 翻转数量：**8,749**（2.92% 的总记录）

### 污染率按标签分布（重要发现）

| 标签 | cell 数 | 污染点数 | 总点数 | 污染率 | 受影响 cell 数 |
|------|---------|----------|--------|--------|----------------|
| dynamic | 60 | 902 | 18,622 | **4.84%** | 54/60 |
| large_coverage | 60 | 1,165 | 24,638 | **4.73%** | 58/60 |
| collision_substitute | 10 | 84 | 1,912 | **4.39%** | 10/10 |
| multi_centroid | 300 | 3,382 | 85,112 | **3.97%** | 257/300 |
| standard | 300 | 3,713 | 171,179 | 2.17% | 221/300 |
| migration | 60 | 343 | 21,038 | 1.63% | 50/60 |

**关键观察**：**异常标签（dynamic / large_coverage / collision / multi_centroid）的污染率普遍高于标准 cell（约 2 倍）**，支持了"ss1 污染可能贡献了部分异常分类"的假设。后续可基于修复后的 `gps_valid_fixed` 重算质心，对比这些异常 cell 的标签是否会变回 standard。

## 6. 使用方式

### 查询某个标签的所有 cell
```sql
SELECT * FROM rebuild5.research_gps_cells
WHERE label_category = 'multi_centroid';
```

### 查询某个 cell 的所有原始记录（修复后）
```sql
SELECT * FROM rebuild5.research_gps_records
WHERE cell_id = 5731057665
  AND gps_valid_fixed = true;  -- 使用修复后的 gps_valid
```

### 对比修复前后的差异
```sql
SELECT cell_id,
       count(*) FILTER (WHERE gps_valid_original) as valid_before,
       count(*) FILTER (WHERE gps_valid_fixed) as valid_after,
       count(*) FILTER (WHERE ss1_contaminated) as contaminated
FROM rebuild5.research_gps_records
GROUP BY cell_id
HAVING count(*) FILTER (WHERE ss1_contaminated) > 0
ORDER BY contaminated DESC
LIMIT 20;
```

### 识别污染严重的 cell
```sql
SELECT r.operator_code, r.lac, r.cell_id, r.tech_norm,
       c.label_category, c.p90_radius_m,
       count(*) FILTER (WHERE r.ss1_contaminated) as bad_pts,
       count(*) as total_pts,
       round(count(*) FILTER (WHERE r.ss1_contaminated)::numeric / count(*) * 100, 2) as contam_pct
FROM rebuild5.research_gps_records r
JOIN rebuild5.research_gps_cells c
  ON c.cell_id = r.cell_id AND c.lac = r.lac 
  AND c.operator_code = r.operator_code AND c.tech_norm IS NOT DISTINCT FROM r.tech_norm
GROUP BY r.operator_code, r.lac, r.cell_id, r.tech_norm, c.label_category, c.p90_radius_m
HAVING count(*) FILTER (WHERE r.ss1_contaminated) >= 20
ORDER BY contam_pct DESC
LIMIT 30;
```

## 7. 索引

为支持研究查询，已建立以下索引：

```
research_gps_cells:
  idx_research_gps_cells_cell (operator_code, lac, cell_id, tech_norm)
  idx_research_gps_cells_label (label_category)

research_gps_records:
  idx_research_records_cell (operator_code, lac, cell_id, tech_norm)
  idx_research_records_dev (dev_id)
  idx_research_records_gps_type (gps_info_type, cell_origin)
```

## 8. 后续研究方向

1. **ss1 污染影响验证**：用 `gps_valid_fixed` 重算每个异常 cell 的质心和 p50/p90，看标签是否改变
2. **dynamic 标签独立性**：当前 dynamic 完全是 multi_centroid 的子集，需研究是否应该作为独立标签或并入 multi_centroid
3. **碰撞检测定义**：batch 7 中 is_collision=0，需重新设计碰撞检测逻辑
4. **大覆盖 vs 多质心分界**：large_coverage 和 multi_centroid 的判定边界是否合理
5. **标准 cell 的异常污染**：300 个 standard cell 中有 221 个也有 ss1 污染点，修复后是否还保持 standard？
