# 第一部分：研究结论

## 2. 问题确认

### 2.1 基础质心算法对 p90 普遍偏大

当前 library 中 standard cell 的 p90 中位数为 **179m**，对于"正常单簇"cell 显著偏大。根因是 `core_position_filter` 的参数：

```yaml
core_position_filter:
  snap_grid_m: 200
  keep_quantile: 0.8
  keep_min_radius_m: 800    # 即使实际点都在 100m 内，也强制保留 800m
  keep_max_radius_m: 3000   # 多质心/噪声 cell 始终保留 3km 内的点
```

固定绝对阈值不随数据离散度调整，是普遍偏大的根本原因。

### 2.2 这个问题是"多质心识别前必修"的

- 现 p90 常高达 400-800m，导致**大量 standard cell 被误纳入多质心深度分析候选**
- 多质心算法在失真的"正常规模"基线上工作，难以得到可信的阈值与分类
- 先把基础算法修准，再谈多质心识别的阈值校准

## 3. MAD 过滤路线的研究结论

### 3.1 算法定义

对每个 cell 的所有 `gps_valid=true` 点：
```
1. center₀ = 边际中位数（median(lon), median(lat)）
2. d_i    = distance(point_i, center₀)
3. med_d  = median(d_i)
4. mad    = median(|d_i - med_d|)
5. is_core = (d_i ≤ med_d + k_mad × mad)
6. center = is_core 点的中位数
7. p50   = P50 of dist(is_core points → center)    // 核心半径
8. p90   = P90 of dist(is_core points → center)    // 核心半径
9. raw_p90 = P90 of dist(all points → center)      // 全量半径（大覆盖触发信号）
10. core_outlier_ratio = 1 - kept_count / total_count
```

### 3.2 主要 p90 中位数对比（全量，720 cell）

| 标签 | 唯一 cell 数 | 当前 library | k_mad=2.5 | **k_mad=3.0** | k_mad=4.0 |
|------|-------------|-------------|-----------|---------------|-----------|
| **standard** | 300 | 179m | 91m (-49%) | **100m (-44%)** | 110m (-39%) |
| **multi_centroid** | 300 | 499m | 346m (-31%) | **379m (-24%)** | 408m (-18%) |
| large_coverage* | 49 | 443m | 263m (-41%) | 330m (-26%) | 344m (-22%) |
| migration* | 56 | 192m | 128m (-33%) | 147m (-23%) | 161m (-16%) |
| dynamic† | 5 | 561m | 325m | 357m | 355m |
| collision_sub† | 10 | 2356m | 343m | 354m | 357m |

\* 中等样本，支持性观察
† 小样本，趋势信号（不宜作为最终定论）

### 3.3 判别比（multi_centroid p90 / standard p90）

| 算法 | 判别比 | 评价 |
|------|-------|------|
| 当前 library | 2.79× | 区分度弱 |
| k_mad = 2.5 | 3.80× | 最佳 |
| **k_mad = 3.0** | **3.79×** | 与 2.5 几乎相同，更稳健 |
| k_mad = 4.0 | 3.71× | 仍优于当前 |

### 3.4 raw_p90 信号保留

新算法下 standard 和 multi_centroid 的 raw_p90（全量 P90）中位数对比：

| 标签 | k=3.0 raw_p90 中位数 |
|------|---------------------|
| standard | 206m |
| multi_centroid | 685m |
| large_coverage | 990m |
| migration | 6663m |

**raw_p90 成功保留了"大覆盖"触发信号**：standard cell 仍保持 ~200m 量级，而异常标签的 raw_p90 显著偏大。

基于以上研究（包含 2026-04-17 第二轮扩展测试），本阶段**固定**以下事项：

| 决策项 | 本阶段取值 | 是否可调 |
|--------|-----------|---------|
| 基础质心算法 | MAD 过滤（替代 seed + keep_quantile） | ✅ 路线已定 |
| **默认 k_mad 策略** | **方案 B：按数据量自适应** | 参数可调，策略路线已定 |
| **观察点质量提升** | **按 (cell, dev, 日期) 去重**（见 §10.1） | ✅ 保留（第四轮新增）|
| raw_p90 保留 | 是（作为大覆盖触发信号） | ✅ 保留 |
| core_outlier_ratio 保留 | 是（作为多簇触发辅助信号） | ✅ 保留 |
| 多质心深度分析算法 | 本阶段不动 | 待基础算法修正后再评估 |
| Step 5 候选触发阈值联动 | 第一版建议已给出（§9），不是最终值 | 实施后校正 |

### 6.1 方案 B：按数据量自适应 k_mad（推荐）

**动机**：单值 k=3.0 对小样本（50-200 点）的过滤强度不足，对 standard 50-99 档只能压到 141m、100-199 档只能到 121m。扩展测试 k=1.5 发现：小样本下更激进过滤能显著提升，但大样本下会过度；因此用数据量分档。

**规则**：
```
if  total_pts < 50:        k_mad = 2.5    （所有 k 值都收敛，不敏感）
elif total_pts < 200:      k_mad = 1.5    （小样本激进，压紧虚高）
else (total_pts ≥ 200):    k_mad = 2.5    （大样本稳定）
```

**效果（全样例对比）**：

| 标签 | 当前 | k=3.0 单值 | **方案 B** |
|------|------|-----------|-----------|
| standard | 179m | 100m | **76m** (-58%) |
| multi_centroid | 499m | 379m | **303m** (-39%) |
| large_coverage | 443m | 330m | **240m** (-46%) |
| migration | 192m | 147m | **122m** (-36%) |

**判别比（multi/standard）**：当前 2.79× → 方案 B **3.99×**（k=3.0 单值为 3.79×）

### 6.2 方案 B 变化分布（720 cell）

| 变化类型 | 数量 | 占比 |
|---------|------|------|
| 变小（>30%） | **412** | **57.2%** |
| 基本不变 | **299** | **41.5%** |
| 变大（>50%） | **9** | **1.3%** |

**standard cell**：66% 变小，**0 个变大**（算法不会误伤正常 cell）。

**9 个变大 cell 全部属于 multi_centroid 或 migration 标签**，其中 4 个是"假精度恢复"（当前算法给出 7-62m 的极小 p90，新算法恢复真实跨度 1-10km 的大覆盖/可疑碰撞）。详见 §8 变大 cell 分析。

### 6.3 为什么用方案 B 而不是单值 k

| 维度 | 单值 k=3.0 | 方案 B |
|------|-----------|-------|
| standard 100-199 档 p90 | 121m | **82m** |
| standard 50-99 档 p90 | 141m | 118m |
| 大样本收敛（500+）| 15m | 15m（与 k=3 一致）|
| 实现复杂度 | 简单 | 略增加一个 if 分支 |
| 可解释性 | 数学清晰 | 数据驱动但仍清晰 |

**方案 B 付出很小的实现复杂度，换来对小样本 cell 明显更好的精度**。

### 6.4 备选方案（不作为默认）

- **单值 k=3.0**：如果觉得自适应太复杂，可以退而求其次。standard p90 = 100m，比当前好但比方案 B 差。
- **单值 k=2.5**：standard p90 = 91m，介于 k=3.0 和方案 B 之间。
- **单值 k=1.5**：standard p90 = 73m，对大样本可能过度过滤（虽然数据未显示明显问题）。

### 6.5 业界对标

- **Ichnaea**（Mozilla）：权重衰减采用类似的鲁棒统计，截断在 3σ 级别
- **OpenCelliD**：未公开具体参数，但采用类似的"离群过滤 + 主簇定位"思路
- **方案 B 本质上**是把"固定 3σ"改成"数据量敏感的 1.5-2.5 MAD"，更贴合真实 GPS 数据的分布

> **性质说明**：以下是基于当前样本提出的第一版联动建议，目的是保证基础算法修正后，多质心/大覆盖候选触发仍可用。**最终阈值将在实现完成后结合真实结果校正。**

### 9.1 建议调整的阈值

**文件**：`rebuild5/config/antitoxin_params.yaml`（`postgis_centroid` 段）

| 参数 | 当前值 | 第一版建议 | 理由 |
|------|-------|-----------|------|
| `candidate_min_p90_m` | 800 | **300** | p90 已从 179→100m 普遍收紧，触发阈值同步降低 |
| `candidate_min_max_spread_m` | 2200 | 2200（不动） | max_spread 是独立指标（基于日质心漂移），与 p90 解耦 |

### 9.2 触发组合（第一版建议）

触发深度多质心分析的 OR 逻辑：
```sql
(p90_radius_m >= 300)
OR (raw_p90_radius_m >= 800 AND core_outlier_ratio >= 0.15)
OR (core_outlier_ratio >= 0.20)
OR (max_spread_m >= 2200)
```

第一、二项对应"核心紧凑但全局散"和"核心收紧但离群点多"两种多质心嫌疑形态；max_spread 保留作为独立信号。

### 9.3 注意

上述阈值不是最终值。**实现完成后请按 §10 的验证标准重跑，根据候选触发量、假阳率等指标重新校准**。

## 9.5 关于"合理精度范围"的原则

在评估算法效果时，需要明确一个关键原则：

**p90 的"好"有上下边界**，不是越小越好。

- **p90 ≤ 800m**：满足定位服务的实用要求，属于可接受精度
- **p90 过小（如 < 10m）**：可能是"假精度"的信号，反而是另一种污染

### 为什么过小的 p90 也是问题

当前 library 中出现 p90 = 7m、30m、53m 的 cell，看起来超精确，但实际原因可能是：
- 单个设备高频在同一地点上报（比如 IoT 设备、静态 POS 机）
- 算法 seed 锁定这个高频点簇，过滤掉了其他所有合法观测
- 结果：给出的 center 不代表 cell 的真实位置，只代表"某个设备经常停靠的地方"

对应的 cell（如 25275343: cur_p90=7m）在新算法下暴露为真实跨度 ~2-5km，这是算法改进的预期行为，而非"变差"。

### 评估算法的正确尺度

| p90 范围 | 评价 |
|---------|------|
| ≤ 100m | 优秀（大样本 standard cell 可达）|
| 100-400m | **良好**（大部分正常 cell 的合理范围）|
| 400-800m | 可接受（多质心或大覆盖 cell 的正常值）|
| 800-2000m | 偏大，应进入多质心深度分析 |
| > 2000m | 异常或真实大覆盖，需要分流处理 |

本报告中讨论的"变小"比例，应该在这个尺度下理解：把 p90 从 500m 压到 400m 没什么意义，但把 500m 压到 150m 或把 5m 的假精度恢复到 300m 都是有意义的改进。

### 两种类型

**Type 1：真大覆盖（均匀散布，低 outlier_ratio）**— 5 cell
- 27132465, 17411075, 3107595, 107641910, 20148490
- 点在 2-3km 内均匀分布，outlier_ratio 低（0-7%）
- MAD 过滤保留几乎所有点，p90 反映**真实覆盖跨度**
- 这些 cell 可能是 5G 宏站大覆盖 OR 碰撞 cell（待碰撞检测落地后区分）

**Type 2：假精度被揭示（散布极广，高 outlier_ratio）**— 4 cell
- 25275343, 107410995, 1709252866, 5875249154
- 当前算法给出 7-62m 的**极不合理小 p90**，而 raw_p90 实际是 3-12km
- outlier_ratio 40-49%（近半点被 MAD 过滤，但余下的仍然很分散）
- 新算法恢复的 p90（158-1987m）才接近真实状况

### 关键观察

1. **0 个 standard cell 变大** — 方案 B 不会误伤正常 cell
2. **所有变大 cell 都已被标签为 multi_centroid/migration** — 已知的异常 cell
3. **全部有多设备（12-92 个）、多天（7-8 天）** — 不是单设备漂移、不是瞬时抖动
4. **"假精度"cell 的数据特征**：点散布达 4-12km，但 10 个 cell 中有 4 个（40%）当前 p90 被压到 < 100m，这是当前算法的严重缺陷
5. **方案 B 揭示问题的机制**：`k_mad=1.5` 对 50-200 点的小样本激进收紧，但对**分布本身散的 cell**（outlier_ratio 高）收紧有限，p90 仍会保留较大值

### 结论

- 方案 B 的"变大"行为**是算法改进的表现，不是回退**
- 9 个变大 cell 中有 4 个是被当前算法隐瞒的严重问题（cur_p90 < 100m，实际覆盖 3-12km）
- 这些 cell 的去向应该是：
  - 通过 p90 ≥ 300m 触发条件进入多质心深度分析
  - 由 DBSCAN 判定是"真大覆盖"还是"碰撞"（碰撞检测当前缺失，是下一阶段任务）

## 10.1 观察点质量提升：设备-天去重

### 背景

cell 25275343 在方案 B 下 p90 仍为 1987m（偏大）。深度分析发现根源：
- 设备 A6BLG426007BK70WB 贡献了 58 个点中的 23 个（40%），全部集中在同一位置
- 单设备高频同位置上报把 MAD 的"中位距离"拉向该位置，导致过滤不起作用

在数据规模大到无法靠算法一一修正每个异常设备的情况下，**按 `(cell, dev_id, DATE(event_time))` 去重**是一个朴素但有效的数据质量提升手段：

- **核心假设**：一个 cell 的有效观察单位是"每个设备每天 1 次接入"，而不是"每次上报"
- **物理合理性**：cell 的覆盖范围应由多个独立设备在多天的独立观测来验证，而不是单设备的高频重复
- **防御异常设备**：单设备 GPS 漂移或锁定造成的数据偏移，经过去重自动被稀释到"1 个投票"

### 实验：设备-天去重 + 方案 B

按 `(cell, dev_id, DATE(event_time))` 聚合，每组随机保留 1 个点，然后跑方案 B。

**对关键 cell 的效果**：

| cell | 特征 | 当前 | 方案 B | **B+去重** |
|------|------|------|--------|-----------|
| 25275343 | 单设备 23 点污染 | 7 | 1987 | **335m** ✅ |
| 27132465 | 真多簇 | 305 | 3495 | 1558m |

cell 25275343 从 1987m 落到 335m，完全落入 §9.5 定义的"合理精度范围"。

### 全样本效果（按 §9.5 合理精度尺度评估）

| 标签 | 方案 B | B+去重 | 评估 |
|------|--------|--------|------|
| standard | 76m | **138m** | B+去重更物理合理（见下文）|
| multi_centroid | 303m | 375m | 都在可接受范围 |
| large_coverage | 240m | 252m | 基本持平 |
| migration | 122m | 127m | 基本持平 |
| collision_sub | 343m | **272m** | B+去重更好 |
| dynamic | 325m | **206m** | B+去重更好 |

### 重要修正：方案 B 单独用于大样本 cell 时存在"假精度"

**standard cell 按数据量档看**：

| 原始点数档 | 方案 B p90 | B+去重 p90 | 物理意义 |
|-----------|-----------|-----------|---------|
| <50 | 139m | 147m | 两者都合理 |
| 50-99 | 118m | 143m | 两者都合理 |
| 100-199 | 82m | 136m | 两者都合理 |
| 200-499 | **71m** | 133m | 71m 略乐观 |
| **500+** | **15m** ⚠️ | **139m** ✅ | **15m 是假精度** |

**关键认知修正**：15m p90 对一个真实物理 cell 来说**不符合物理常识**：
- 4G/5G cell 的天线物理覆盖半径至少 50-300m
- 15m 半径的"cell"只能是某个设备长期静止在同一点、多次上报同位置（设备信号，不是 cell 信号）
- 这就是 §9.5 所说的"**过小 p90 也是一种污染**"

**方案 B 单独对 500+ 点 cell 给出 15m**：是因为多设备同位置重复上报的"伪确认"主导了 MAD 估计。MAD 算法本身无法区分"100 个独立观测"和"同一个设备 100 次重复上报"。

**B+去重后的 139m 才是物理真实值**：每个 (设备, 天) 贡献 1 票，多设备多天投票汇总得到的覆盖范围。

### 全样本变化分布对比（按 §9.5 尺度重新解读）

| 变化类型 | 方案 B | B+去重 |
|---------|--------|--------|
| 变小（>30%） | 57.2% | 37.1% |
| 基本不变 | 41.5% | 56.1% |
| 变大（>50%） | 1.3% | 6.8% |

B+去重下"变大"的 6.8% 多数是从方案 B 的假精度（如 500+ 档的 15m）恢复到物理真实值（139m），不是算法变差。按 §9.5 尺度看：所有标签的 p90 中位数都落在 <500m 范围，没有一个超出可接受精度。

### 为什么"按天去重"是合适的

1. **颗粒度合理**：按天而不是按小时，保留了"一个设备在多天的独立观测"（即真实的独立确认），同时消除了"单日高频重复"的伪权重
2. **物理对应"独立接入"**：一个正常用户一天到达某 cell 一次是常态，按天取 1 点相当于取一个独立的"接入事件"
3. **对数据规模友好**：251,427 点 → 40,843 点（-84%），算法性能大幅提升
4. **防御异常无需算法检测**：不需要复杂的"异常设备识别"逻辑，暴力去重自然把单设备权重限制到合理区间

### 本阶段决策修正

**推荐采用 方案 B + 设备-天去重 作为最终基线**（替代原单独方案 B）：

| 维度 | 方案 B | **方案 B + 去重** |
|------|--------|------------------|
| standard 整体 p90 中位 | 76m（含假精度）| **138m（物理真实）** |
| 500+ 大样本 standard p90 | 15m（假精度）| **139m（真实）** |
| cell 25275343 | 1987m（偏大）| **335m（合理）**|
| 对单设备高频污染免疫 | ❌ | ✅ |
| 对真多簇的识别 | ✅（p90 保持大）| ✅（p90 仍大，如 1558m）|
| 数据量 | 251,427 | 40,843（-84%）|
| 实现复杂度 | MAD 过滤 | **去重 + MAD 过滤**（多一步 DISTINCT ON）|
