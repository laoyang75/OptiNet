# 第一阶段修改意见：Step 4 UI 小修复

## 目标

本阶段只对 Step 4 知识补数页做轻量 UI 收口，不扩展复杂筛选或治理联动。

目标是：

- 让页面与当前后端能力一致
- 明确异常样本只展示最新批次
- 把未实现筛选能力标为待开发

## 本阶段不做

- 运营商 / LAC / donor 质量 / 异常状态多维筛选
- Step 4 异常列表的复杂联动
- 与 Cell 维护页的深度双向跳转

## 建议修改项

### 1. 异常样本说明改成“最新批次”

当前问题：

- 页面只展示少量异常样本，但没有明确样本范围
- 用户容易误以为是跨批次全量异常池

建议：

- 在异常样本区域明确写：
- `当前仅展示最新批次异常样本`

建议触达文件：

- `rebuild5/frontend/design/src/views/governance/KnowledgeFill.vue`

### 2. 筛选能力改为待开发说明

当前问题：

- 文档原先写了按运营商 / LAC / donor 质量 / 是否异常筛选
- 当前页面还是轻量概览版，没有这些筛选器

建议：

- 页面与 UI 文档统一说明：
- 当前已支持概览卡片、字段级补数表、donor 质量分布、最新批次异常样本
- 筛选能力待开发

建议触达文件：

- `rebuild5/frontend/design/src/views/governance/KnowledgeFill.vue`
- `rebuild5/ui/06_知识补数与治理页面.md`

### 3. GPS 异常口径说明补齐

当前问题：

- 用户容易把 Step 4 GPS 异常理解为“任意 GPS 都会参与比对”

建议：

- 页面或文档中补一句：
- `仅对原始且有效的 GPS 证据做 donor 质心比对`

建议触达文件：

- `rebuild5/docs/04_知识补数.md`
- `rebuild5/docs/human_guide/04_Step4_知识补数.md`

## 验收标准

- Step 4 页面不再暗示已支持复杂筛选
- 异常样本范围明确为最新批次
- GPS 异常检测口径说明与当前实现一致
