# Step 6: 结果库浏览与查询

> 更新声明(2026-04-28)
> - 已按当前生产环境对齐到 PG 18.3 / Citus 14.0-1 / PostGIS 3.6.3 / `5488` / `yangca`

> - `rebuild4_meta.lac_location_snapshot` 是否长期保留为地址映射源表,当前文档未擅自改写,待 user 二审

Step 1-5 是数据加工管道，产出可信赖的运营商基站数据库。Step 6 是这个结果库的**消费层** — 提供查询、浏览和统计功能。它没有下游处理，就是终点。

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块角色 | 结果库的只读查询与可视化浏览 |
| 数据来源 | `trusted_cell_library`、`trusted_bs_library`、`trusted_lac_library`、`enriched_records` |
| 读写边界 | **只读**，不回写任何管道表 |
| 用户问题 | "库里有什么、某个基站在哪、这个区域质量如何" |

### 结果库的本质

管道跑完后积累下来的数据就是"运营商基站数据库"：

- **逐条粒度**：`enriched_records` — Step 4 产出的补齐后观测记录，是最细粒度的事实数据
- **聚合粒度**：`trusted_cell/bs/lac_library` — Step 5 窗口计算 + 标签后发布的画像

```
enriched_records（逐条观测，补齐后）
    ↓ Step 5 窗口聚合
trusted_cell_library → trusted_bs_library → trusted_lac_library
    ↓
Step 6: 浏览、查询、统计
```

页面顶部统一展示当前数据集标识（`dataset_key`）、运行批次编号（`run_id`）和快照版本号（`snapshot_version`）。

## 数据来源

| 表名 | 粒度 | 查询用途 |
|------|------|----------|
| `trusted_cell_library` | Cell | 按 cell_id 查位置、精度、质量、标签 |
| `trusted_bs_library` | BS | 按 bs_id 查站级聚合、子 Cell 列表 |
| `trusted_lac_library` | LAC | 按 LAC 查区域概览、子 BS 列表 |
| `enriched_records` | 单条观测 | 浏览某 Cell 的逐条补齐数据（未实现，待扩展） |

所有查询只读 `MAX(batch_id)` 的最新版本。

## 6.1 站点查询

按 Cell/BS/LAC 三个层级搜索和查看画像详情。

### 搜索

| 参数 | 说明 |
|------|------|
| 关键词（`q`） | 模糊匹配 cell_id / bs_id / lac / 运营商名称 |
| 层级（`level`） | cell / bs / lac |
| 运营商筛选（`operator_code`） | 接口支持，当前页面待开发 |

搜索结果分页展示，默认按精度（`p90_radius_m`）升序排列。当前页面已支持关键词 + 层级切换；运营商筛选和坐标范围查询保留为待开发。

### Cell 详情

返回单个 Cell 的完整画像：

| 字段组 | 字段 |
|--------|------|
| 标识 | operator_code / operator_cn / lac / bs_id / cell_id / tech_norm |
| 位置 | center_lon / center_lat |
| 精度 | p50_radius_m / p90_radius_m |
| 质量 | position_grade / lifecycle_state |
| 资格 | anchor_eligible / baseline_eligible |
| 标签 | drift_pattern / is_collision / is_dynamic / is_multi_centroid |
| 信号 | rsrp_avg / rsrq_avg / sinr_avg / pressure_avg |
| 活跃度 | active_days_30d / consecutive_inactive_days / window_obs_count |

碰撞或动态标签命中时，页面显式提示异常特征。

### BS 详情

返回 BS 画像 + 其下所有 Cell 列表（lifecycle_state / position_grade / p90_radius_m / drift_pattern / is_collision）。

### LAC 详情

返回 LAC 画像 + 其下所有 BS 列表（lifecycle_state / classification / total_cells / qualified_cells / anomaly_cell_ratio）。

## 6.2 覆盖统计

全局维度的汇总统计，回答"库里有什么"。

### 总览卡片

| 指标 | 说明 |
|------|------|
| 可信 Cell 总数 | trusted_cell_library 最新批次行数 |
| LAC 覆盖数 | Cell 库中 DISTINCT lac 数 |
| 平均 P90 精度 | 所有 Cell 的 AVG(p90_radius_m) |

### 运营商对比

| 字段 | 说明 |
|------|------|
| 运营商 | operator_code / operator_cn |
| Cell 数 | 该运营商的 Cell 数量 |
| 合格占比 | qualified + excellent 占比 |
| 优秀占比 | excellent 占比 |
| 平均 P90 | 该运营商的平均精度 |

## 6.3 LAC 统计报表

按 LAC 粒度的汇总报表，支持 CSV 导出。

| 字段 | 说明 |
|------|------|
| 运营商 / LAC | 标识 |
| Cell 总数 / 合格数 / 优秀数 | Cell 层级统计 |
| BS 总数 / 合格数 | BS 层级统计 |
| 平均 P90 | 该 LAC 下 Cell 的平均精度 |

## UI 页面

| 页面 | 路由 | 核心功能 |
|------|------|----------|
| 站点查询 | `/service/query` | 搜索 + Cell/BS/LAC 详情卡 |
| 覆盖统计 | `/service/coverage` | 总览卡片 + 运营商对比表 |
| 统计报表 | `/service/report` | LAC 粒度报表 + CSV 导出 |

页面风格比管道层更简洁，弱化治理术语。当前详情页仍保留部分治理字段作为过渡展示，后续继续收口为更业务友好的风险提示和质量表达。距离用米、比例用百分比、坐标保留 6 位小数。

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/service/search` | 搜索（q / level / operator_code / page / page_size） |
| GET | `/api/service/cell/{cell_id}` | Cell 详情；支持附带 `operator_code`、`lac`、`tech_norm` 作为上下文约束 |
| GET | `/api/service/bs/{bs_id}` | BS 详情 + 子 Cell；支持附带 `operator_code`、`lac` 作为上下文约束 |
| GET | `/api/service/lac/{lac}` | LAC 详情 + 子 BS；支持附带 `operator_code` 作为上下文约束 |
| GET | `/api/service/coverage` | 覆盖统计 |
| GET | `/api/service/report` | LAC 报表 |

## 成功标准

| 能力 | 判定标准 |
|------|----------|
| 查得到 | 按 cell_id 或关键词能快速定位基站画像 |
| 看得清 | 总览能了解库的规模、质量分布和运营商结构 |
| 能追溯 | 页面展示当前数据集、运行批次和快照版本 |
| 可导出 | LAC 报表支持 CSV 导出 |
