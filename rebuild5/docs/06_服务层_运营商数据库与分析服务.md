# Step 6: 服务层 - 运营商数据库与分析服务
本文件定义 `rebuild5` 的服务层。Step 1-5 负责生产和维护数据，Step 6 负责把这些结果组织成面向业务用户的查询、分析和报表服务。

来源：
- `rebuild5/README.md` 的“服务层”“Step 6: 服务层 - 运营商数据库与分析服务”“UI 设计”段落
- `rebuild5/docs/00_全局约定.md` 的实体模型、生命周期、三层资格、命名规范

## 模块定位
| 项目 | 说明 |
|------|------|
| 模块角色 | 面向业务用户提供查询、分析、报表和服务接口 |
| 上游输入 | 可信小区正式库（`trusted_cell_library`）、可信基站正式库（`trusted_bs_library`）、可信位置区正式库（`trusted_lac_library`） |
| 数据语义 | 只消费 Step 5 发布后的正式库，不参与 Step 2-5 的判级和治理 |
| 读写边界 | 只读消费正式库，不回写管道中间结果 |
| 用户视角 | 关注“基站在哪、覆盖如何、区域质量怎样”，不是“规则是否命中” |
服务层读取的是“上一轮已经发布、可供对外使用”的正式结果，而不是本批运行中的中间态。页面顶部统一展示当前数据集标识（`dataset_key`）、运行批次编号（`run_id`）和快照版本号（`snapshot_version`）。

## 数据来源
| 表名 | 粒度 | 服务侧用途 |
|------|------|------------|
| `trusted_cell_library` | Cell | 基站位置查询、附近站点检索、覆盖质量分析 |
| `trusted_bs_library` | BS | 站点聚合、区域密度、站级质量统计 |
| `trusted_lac_library` | LAC | 区域概览、LAC 级报表、运营商区域对比 |
服务侧消费原则：只读消费、展示完整画像、版本可追溯、不暴露管道中间态。

## 6.1 基站位置查询
| 场景 | 输入 | 返回 |
|------|------|------|
| 按小区编号查询 | 小区编号（`cell_id`），可选运营商编码（`operator_code`）、位置区（`lac`） | 位置、`p90_radius_m`、`position_grade`、生命周期状态、画像标签 |
| 按坐标查询 | 中心点 + 半径 / 地图框选 | 范围内基站列表、距离排序、运营商和制式筛选 |
| 按运营商 / LAC 筛选 | 运营商编码（`operator_code`）、位置区（`lac`）、制式（`tech_norm`） | 指定区域内的 Cell / BS 列表 |
建议返回：运营商编码 / 中文名（`operator_code` / `operator_cn`）、位置区 / 基站 / 小区编号（`lac` / `bs_id` / `cell_id`）、中心坐标（`center_lon` / `center_lat`）、位置精度（`p90_radius_m`）、定位质量等级（`position_grade`）、生命周期状态（`lifecycle_state`）、可作补数锚点（`anchor_eligible`）、可参与基线刷新（`baseline_eligible`）、漂移 / 碰撞 / 动态标签（`drift_pattern` / `is_collision` / `is_dynamic`）。命中碰撞或动态标签时，要显式提示“该对象存在异常特征，建议谨慎使用”。

## 6.2 信号覆盖分析
| 主题 | 说明 |
|------|------|
| 基站密度 | 指定区域内 Cell / BS 数量、单位面积密度 |
| 信号质量分布 | `RSRP`、`RSRQ`、`SINR` 的均值或分位分布 |
| 运营商覆盖对比 | 同一区域不同运营商的覆盖数量、质量等级和制式结构 |
| 制式分布 | `4G` / `5G` 占比、优秀站点占比 |
常见视图包括区域密度热区、质量分布图、运营商对比表、制式占比图。

## 6.3 区域统计报表
| 报表 | 统计口径 |
|------|----------|
| 省 / 市 / 区汇总 | 基站数量、Cell 数量、合格及以上占比、优秀占比 |
| LAC 汇总 | 每个 LAC 的 BS 数、Cell 数、质量分布、异常标记数 |
| 趋势报表 | 多个数据集版本或运行版本之间的数量和质量变化 |
建议字段包括：区域名 / 位置区（`region_name` / `lac`）、基站数 / 小区数（`bs_count` / `cell_count`）、合格占比 / 优秀占比（`qualified_ratio` / `excellent_ratio`）、碰撞数 / 动态数（`collision_count` / `dynamic_count`）、快照版本号（`snapshot_version`）。

## 6.4 接口服务（未来扩展）
| 接口 | 作用 | 典型输入 | 典型输出 |
|------|------|----------|----------|
| 基站位置查询接口 | 单站点检索 | `cell_id` / `operator_code` / `lac` | 单条画像详情 |
| 批量匹配接口 | 批量回查位置 | `cell_id` 列表 | 每个 Cell 的位置与质量摘要 |
| 覆盖评估接口 | 区域分析 | 区域边界、运营商、制式 | 密度、质量分布、覆盖对比 |
接口返回的是服务层产品视图，不暴露管道内部步骤字段；所有返回值都必须带版本标识，便于离线报表和线上查询对齐。

## 6.5 多数据源展示
| 展示主题 | 说明 |
|----------|------|
| 各源覆盖率 | 不同来源对同一地区的站点覆盖程度 |
| 各源质量差异 | 不同来源的 `position_grade`、精度、信号质量对比 |
| 补数前后对比 | 原始画像与补数后画像的质量变化 |
| 综合画像 vs 单源画像 | 展示融合结果是否优于单一来源 |

## UI 页面
| 页面 | 核心组件 | 面向用户的问题 |
|------|----------|----------------|
| 基站查询页 | 搜索框、结果列表、详情卡、地图预留区 | “这个基站在哪，质量怎么样？” |
| 覆盖分析页 | 区域选择、筛选器、图表区 | “这个区域覆盖够不够、哪家更好？” |
| 统计报表页 | 汇总表格、趋势图、导出入口 | “我要拿一份可复用的汇总结果” |
页面风格应比管道层更简洁，弱化治理术语；顶部固定显示当前数据集标识、运行批次编号、快照版本号；距离用米、比例用百分比、坐标保留 6 位小数；报表页支持导出。

## 成功标准
| 能力 | 判定标准 |
|------|----------|
| 查得准 | 能按 `cell_id` 或坐标快速定位基站，并给出精度与质量说明 |
| 看得清 | 能按区域理解覆盖密度、运营商差异和质量结构 |
| 能追溯 | 所有查询和报表都能明确对应的数据集、运行批次和快照版本 |
| 可扩展 | 后续可平滑接接口服务、多数据源和地图能力 |
