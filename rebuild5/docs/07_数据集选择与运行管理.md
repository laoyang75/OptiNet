# 数据集选择与运行管理

> 更新声明(2026-04-28)
> - 已按当前生产环境对齐到 PG 18.3 / Citus 14.0-1 / PostGIS 3.6.3 / `5488` / `yangca`
> - 本轮只更新环境信息、schema/SQL 示例、量化基线和文档路径;业务规则、状态机和术语定义保持不变

本文件定义 `rebuild5` 的数据管理模块。当前版本以“单活数据集模式”为准：系统只运行 `config/dataset.yaml` 中指定的当前数据集，并记录该数据集的运行结果版本；多数据集在线切换保留为后续开发项。

来源：
- `rebuild5/README.md` 的“数据管理”“数据版本”段落
- `rebuild5/docs/00_全局约定.md` 的命名规范、技术栈、版本化约束

## 模块定位
| 项目 | 说明 |
|------|------|
| 模块角色 | 管理当前数据集元数据、运行记录和未来切换规划 |
| 管理对象 | 数据集元数据、当前单活数据集、同数据集下的多次运行结果 |
| 作用范围 | 贯穿 Step 2-5；Step 1 是通用 ETL，不受当前数据版本约束 |
| 页面视角 | 全局配置与版本管理，不属于某个 Step 的内部逻辑 |
| 当前状态 | 不支持在线切换；列表页主要用于展示元数据和当前上下文 |
数据版本语义：数据集标识（`dataset_key`）表示当前处理哪批数据，运行批次编号（`run_id`）表示同一数据集的某次执行，快照版本号（`snapshot_version`）表示该次执行发布的冻结快照。Step 2-5 的结果都要绑定当前数据集标识；当前实现尚未支持多个数据集结果并存切换。

## 当前实现状态

```text
config/dataset.yaml 指定唯一当前数据集
    -> Step 0-5 共享结果表围绕该数据集构建
    -> 页面展示当前单活数据集的结果
    -> 其他数据集仅保留元数据/历史，不作为在线可切换结果源
```

约束：

| 项目 | 当前行为 |
|------|----------|
| 数据集页面 | 展示元数据与当前上下文，不提供切换按钮 |
| 切换数据集 | 需修改 `config/dataset.yaml` 后清理共享产物并重跑 |
| 结果表 | 当前按“单活数据集”使用，不保证多数据集结果并存可切换 |
| 后续规划 | 多数据集切换方案见 `rebuild5/docs/dev/05_多数据集切换方案.md` |

## 数据集定义
| 组成 | 说明 |
|------|------|
| 原始数据 | 一批已导入的源数据，可供 Step 1 清洗与结构化 |
| 元数据 | 来源、导入时间、记录数、LAC 范围、时间范围 |
| 状态 | 是否已导入、是否可运行、是否为当前选中版本 |

## 未来功能：数据集切换（未开发）
```text
选择新的数据集标识（dataset_key）
    -> 确认数据集已具备运行条件
    -> 重新执行 Step 2-5
    -> 发布该数据集对应的新结果版本
```
| 规则 | 说明 |
|------|------|
| 当前实现 | 暂不支持；页面不提供在线切换 |
| 过渡操作 | 如需更换数据集，修改 `config/dataset.yaml` 后重跑 |
| 规划目标 | Step 2-5 结果按 `dataset_key` 隔离，支持查看/切换/重跑 |
| 方案文档 | `rebuild5/docs/dev/05_多数据集切换方案.md` |
`etl_cleaned` 的长期目标是成为可复用的原始准备层；但在切换能力开发前，仍按单活模式维护共享产物。

## 运行记录
同一数据集可以因为参数调整、规则变更或验证目的被重复运行。数据管理必须回答“这次结果是怎么来的”。
| 字段 / 能力 | 说明 |
|-------------|------|
| 数据集标识 / 运行批次编号（`dataset_key` / `run_id`） | 本次运行属于哪批数据、是哪一次执行 |
| 运行类型（`run_type`） | 日常运行 / 初始化 / 局部重算 / 完整回归 / 对比验证 |
| 参数文件（`params_file`） | 使用的参数文件版本 |
| 开始 / 结束时间（`started_at` / `finished_at`） | 运行时间 |
| 结果摘要（`result_summary`） | 覆盖率、对象数量、异常统计等 |
| 快照版本号（`snapshot_version`） | 本次发布的冻结快照版本 |
| 历史查看 | 查看同一数据集的多次运行记录 |
| 结果对比 | 比较不同运行批次编号的结果摘要和关键指标 |
| 版本回溯 | 知道当前线上版本来自哪次运行 |

## 前期 vs 正式运行
| 阶段 | 重点 | 页面策略 |
|------|------|----------|
| 开发 / 验证阶段 | 固定单个数据集跑通主链路 | 以只读元数据 + 运行历史为主 |
| 正式运行阶段 | 固定主数据集持续处理 | 页面可保持只读展示 |
后续若开发切换能力，再把数据管理页面升级成“主动控制台”。

## UI 页面
| 页面 | 核心内容 |
|------|----------|
| 当前数据集页 | 数据集列表、状态、元数据、当前选中标识、单活模式提示 |
| 运行历史页 | 每次运行的时间、参数、结果摘要、审计追溯 |
页面要求：强展示元数据，强版本感知，明确提示“当前不支持在线切换”，整体风格保持简洁实用。

## 数据表
| 表名 | 核心字段 | 用途 |
|------|----------|------|
| `dataset_registry` | `dataset_key`、`source_desc`、`imported_at`、`record_count`、`lac_scope`、`time_range`、`status`、`is_current` | 存放数据集元数据和当前选中状态 |
| `dataset_run_log` | `dataset_key`、`run_id`、`run_type`、`params_file`、`result_summary`、`started_at`、`finished_at`、`snapshot_version` | 存放运行历史、参数和结果摘要 |

## 成功标准
| 能力 | 判定标准 |
|------|----------|
| 选得清 | 管理员能明确知道当前系统跑的是哪一批数据 |
| 说得清 | 用户能明确知道当前是单活模式，页面不支持在线切换 |
| 查得到 | 任意一次运行都能追溯参数、时间和结果摘要 |
| 对得上 | 页面展示的版本信息与当前单活数据集的 Step 2-5 产物一致 |
