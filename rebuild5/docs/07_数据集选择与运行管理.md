# 数据集选择与运行管理
本文件定义 `rebuild5` 的数据管理模块。它负责选择“当前系统正在处理哪一个数据集”，并记录同一数据集在不同参数下的运行结果版本。

来源：
- `rebuild5/README.md` 的“数据管理”“数据版本”段落
- `rebuild5/docs/00_全局约定.md` 的命名规范、技术栈、版本化约束

## 模块定位
| 项目 | 说明 |
|------|------|
| 模块角色 | 管理数据集选择、切换、运行记录和结果对比 |
| 管理对象 | 数据集元数据、当前选中数据集、同数据集下的多次运行结果 |
| 作用范围 | 贯穿 Step 2-5；Step 1 是通用 ETL，不受当前数据版本约束 |
| 页面视角 | 全局配置与版本管理，不属于某个 Step 的内部逻辑 |
数据版本语义：数据集标识（`dataset_key`）表示当前处理哪批数据，运行批次编号（`run_id`）表示同一数据集的某次执行，快照版本号（`snapshot_version`）表示该次执行发布的冻结快照。Step 2-5 的结果都要绑定当前数据集标识；切换数据集不是轻量配置切换，而是切换整套管道处理语境。

## 数据集定义
| 组成 | 说明 |
|------|------|
| 原始数据 | 一批已导入的源数据，可供 Step 1 清洗与结构化 |
| 元数据 | 来源、导入时间、记录数、LAC 范围、时间范围 |
| 状态 | 是否已导入、是否可运行、是否为当前选中版本 |

## 数据集切换
```text
选择新的数据集标识（dataset_key）
    -> 确认数据集已具备运行条件
    -> 重新执行 Step 2-5
    -> 发布该数据集对应的新结果版本
```
| 规则 | 说明 |
|------|------|
| 管理入口 | 管理员在“数据集选择与运行管理”页面选择数据集 |
| 重跑范围 | 切换后重新执行 Step 2-5；Step 1 产出可复用 |
| 结果替换 | 新数据集运行成功后，当前正式库和快照切换到新版本 |
| 可追溯性 | 切换前后的运行批次编号、参数文件和结果摘要都要保留 |
`etl_cleaned` 可以属于多个数据集的原始准备层；数据版本约束从 Step 2 开始正式生效。切换失败时，不应覆盖当前已发布版本。

## 运行记录
同一数据集可以因为参数调整、规则变更或验证目的被重复运行。数据管理必须回答“这次结果是怎么来的”。
| 字段 / 能力 | 说明 |
|-------------|------|
| 数据集标识 / 运行批次编号（`dataset_key` / `run_id`） | 本次运行属于哪批数据、是哪一次执行 |
| 运行类型（`run_type`） | 日常运行 / 初始化 / 局部重算 / 完整回归 / 对比验证 |
| 参数文件（`params_file`） | 使用的参数文件版本 |
| 开始 / 结束时间（`started_at` / `finished_at`） | 运行时间 |
| 结果摘要（`result_summary`） | 覆盖率、对象数量、异常统计等 |
| 快照版本号（`snapshot_version`） | 本次发布的冻结快照版本 |
| 历史查看 | 查看同一数据集的多次运行记录 |
| 结果对比 | 比较不同运行批次编号的结果摘要和关键指标 |
| 版本回溯 | 知道当前线上版本来自哪次运行 |

## 前期 vs 正式运行
| 阶段 | 重点 | 页面策略 |
|------|------|----------|
| 开发 / 验证阶段 | 频繁切换数据集，验证规则与收敛效果 | 强调切换、对比、快速查看历史 |
| 正式运行阶段 | 固定主数据集持续处理 | 切换入口降级，页面可简化为只读展示 |
前期数据管理更像“主动控制台”；正式运行后更像“版本看板”，重点是展示当前版本和历史追溯。

## UI 页面
| 页面 | 核心内容 |
|------|----------|
| 数据集选择页 | 数据集列表、状态、元数据、当前选中标识、切换操作 |
| 运行历史页 | 每次运行的时间、参数、结果摘要、对比入口 |
页面要求：强展示元数据，强版本感知，支持同数据集不同运行结果对比，整体风格保持简洁实用。

## 数据表
| 表名 | 核心字段 | 用途 |
|------|----------|------|
| `dataset_registry` | `dataset_key`、`source_desc`、`imported_at`、`record_count`、`lac_scope`、`time_range`、`status`、`is_current` | 存放数据集元数据和当前选中状态 |
| `dataset_run_log` | `dataset_key`、`run_id`、`run_type`、`params_file`、`result_summary`、`started_at`、`finished_at`、`snapshot_version` | 存放运行历史、参数和结果摘要 |

## 成功标准
| 能力 | 判定标准 |
|------|----------|
| 选得清 | 管理员能明确知道当前系统跑的是哪一批数据 |
| 切得稳 | 切换数据集时不会混淆旧结果和新结果 |
| 查得到 | 任意一次运行都能追溯参数、时间和结果摘要 |
| 对得上 | 页面展示的版本信息与 Step 2-5 产物一致 |
