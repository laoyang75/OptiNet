# 08 标签定义总览（rebuild5 cell 画像）

> **创建日期**：2026-04-18
> **基于**：Phase B 在 999 样本 (p90≥800m) 上的实证研究结果
> **配套文档**：`标准质心修复.md`（本次修复总纲 + 完整重跑指南）
> **状态**：规则已验证部分 + 待完整数据验证部分，明确标注

本文档定义 rebuild5 中 cell 质心形态标签的**精确判定规则**、**判定流程**、**阈值清单**、**SQL 伪代码**，以及 **生产代码改造范围**。独立 agent 拿此文档可直接实施标签逻辑并重跑。

## 1. 标签体系一览

8 个标签 + 1 个技术兜底标签（`oversize_single`），对应 cell 在 GPS 时空维度上的不同物理模式：

| 标签 | 物理含义 | k_eff | 核心特征 |
|------|---------|-------|---------|
| **insufficient** | 证据不足 | 0 | 无有效质心（补数为主 / 观测稀疏）|
| **stable** | 单质心紧凑 | 1 | p90 < 1200m，典型普通固定基站 |
| **large_coverage** | 单质心大范围 | 1 | p90 1200m - 10km，体育场/学校/商场 |
| **dual_cluster** | 双质心同管辖 | 2 | 距离 < 5km + 时间高重叠，cell/基站分离 |
| **migration** | 迁移 | 2 | 距离 < 5km + 时间不重叠 + 跳后稳 + 不回头 |
| **collision** | 碰撞 | 2 | 距离 > 100km，同 cell_id 跨城误配 |
| **multi_centroid** | 多质心（含 uncertain + dynamic）| ≥ 3 | 见 §2.7 进一步细分 |
| **dynamic** | 动态（高铁/车载）| ≥ 3 | 轨迹化 + 短驻留 + 大跨度 |
| **uncertain** | 多质心但模式不明 | ≥ 3 | 既非 dynamic 也非已定义子类 |
| **oversize_single**（技术兜底）| 单质心但 p90>10km | 1 | 可疑：副簇未达门槛 / 残留飞跃污染 |

**设计原则**：
- `collision` 是双质心的一种，只在 `k_eff=2` 且两簇距离 > 100km 时判定（物理上同 cell_id 不可能跨城）
- `k_eff ≥ 3` 的跨城 case 归 multi_centroid 族（dynamic / uncertain），不归 collision
- `collision` 判定优先级高于 dual_cluster / migration（距离是硬信号，不受时间重叠影响）

## 2. 判定流程（分层排查）

```
         Step 5 完成（标准质心算法修复后）
                       │
                       ▼
       ┌──────────────────────────┐
       │ 过滤补数：只取 raw_gps   │  — gps_fill_source_final = 'raw_gps'
       │ 设备-天去重              │  — DISTINCT ON (cell, dev, DATE)
       └────────────┬─────────────┘
                    │
                    ▼
       ┌──────────────────────────┐
       │ DBSCAN 聚类              │  — eps=250m, minPts=4
       │ （坐标 lon×85300, lat×111000）
       └────────────┬─────────────┘
                    │
                    ▼
       ┌──────────────────────────┐
       │ 准入门槛：每簇 dev-day 去重点数 ≥ 10
       │ 不达标的簇直接丢弃，算入 noise
       └────────────┬─────────────┘
                    │
                    ▼
           k_eff = 有效簇数
                    │
      ┌─────────────┼──────────────────────┐
      ▼             ▼              ▼       ▼
   k_eff=0       k_eff=1         k_eff=2   k_eff≥3
      │             │              │         │
insufficient     p90<1200         双簇分析   多质心族
              → stable        ┌──┴───┐   ┌──┴──┐
              1200-10km       时间    时间  轨迹  其他
              → large_cov     重叠    不重叠 化+短 → uncertain
              >10km           +近距   +稳+不回头  驻留+大跨度
              → oversize      → dual   → migration  → dynamic
              _single         _cluster
```

## 3. 每个标签的精确判定

### 3.1 insufficient（证据不足）

**判定**：`k_eff = 0`（无任何有效质心）

**触发场景**（任一）：
- 总 raw_gps 点数 < 10（dev-day 去重后）
- 点数够但 DBSCAN 分不出足够密的簇（最大簇 dev-day 也 < 10）
- 历史上 qualified（lifecycle_state sticky）但当前窗口数据衰减

**样本命中**：718 / 878 = 82.3%（绝大多数）

---

### 3.2 stable（单质心紧凑）

**判定**：
- `k_eff = 1` 且
- `p90_radius_m < 1200m`（MAD 滤后半径）

**阈值来源**：Phase B 直接定 1200m（宽于 publish_cell.py 现有的 500m stable_max_spread_m）。`max_spread_m` 是日质心跨度，跟 p90 半径是两个概念，不冲突。

**样本命中**：19 / 878 = 2.2%

**配置项**：`antitoxin_params.yaml` 新增 `label_rules.stable_max_p90_m: 1200`

---

### 3.3 large_coverage（单质心大范围）

**判定**：
- `k_eff = 1` 且
- `1200 ≤ p90_radius_m ≤ 10000m`

**物理含义**：体育场、大学校园、商场等单点大范围覆盖。

**样本命中**：82 / 878 = 9.4%

**配置项**：`label_rules.large_coverage_max_p90_m: 10000`

---

### 3.4 oversize_single（技术兜底）

**判定**：
- `k_eff = 1` 且
- `p90_radius_m > 10000m`

**物理含义**：单质心不应该 >10km。可能原因：
1. 有真实副簇但 dev-day < 10 被门槛过滤（升级数据后会变成 k_eff≥2）
2. MAD 没滤干净的飞跃污染（Step 5 质心算法边界 case）
3. 极罕见的真·机场超大覆盖

**样本命中**：10 / 878 = 1.1%

**建议**：UI 上归入"证据不足"或单独标红，提醒运维人工复查。完整重跑后数据增加，预期此类会减少。

---

### 3.5 dual_cluster（双质心同管辖）

**判定**：
- `k_eff = 2` 且
- 两簇距离 `dist_m < 5000m` 且
- 时间重叠度 `overlap_ratio ≥ 0.5`

其中：
```
dist_m = 两簇质心的地球距离（米）
overlap_days = MAX(0, LEAST(la, lb) - GREATEST(fa, fb) + 1)
            （la/fa = 簇 A 的 last/first active day，同理簇 B）
overlap_ratio = overlap_days / MIN(da, db)
            （da/db = 簇 A/B 的 active day 数）
```

**物理含义**：cell 和基站物理分离部署，但在同一管辖区域内。

**样本命中**：37 / 878 = 4.2%（距离全 <4km，重叠全 ≥50%）

**配置项**：
- `label_rules.dual_cluster_max_dist_m: 5000`
- `label_rules.dual_cluster_min_overlap_ratio: 0.5`

---

### 3.6 migration（迁移）

**判定**：
- `k_eff = 2` 且
- 两簇距离 `dist_m < 5000m` 且
- 时间重叠度 `overlap_ratio = 0`（完全不重叠）且
- **跳后稳**：后段（first_day 较晚的簇）的 `day_cnt ≥ 2`
- **不回头**：后段的 `last_day` 是整个 cell 活跃范围的最后一天（后段出现后前段不再出现）

**阈值来源**：
- 严格 overlap_ratio=0 —— 参照 cell 1353583 北京→重庆案例（完全不重叠）
- "跳后稳" / "不回头" 基于实证观察，避免跟周期性切换（multi_centroid）混淆

**样本命中**：0 / 878（本样本 k_eff=2 全部 overlap ≥ 0.5）

**未验证说明**：样本里严格迁移会退化成 `k_eff=1`（起点段 dev-day < 10，只剩终点段被识别为唯一有效质心）。**需要完整数据 + 更长时间窗口验证**。

**配置项**：
- `label_rules.migration_max_overlap_ratio: 0.0`
- `label_rules.migration_min_post_days: 2`

---

### 3.7 multi_centroid 族（k_eff ≥ 3）

k_eff ≥ 3 的 cell 先按 dynamic 特征判断，不满足的归 uncertain。

#### 3.7.1 dynamic（动态/高铁类）

**判定**（全部满足）：
- `k_eff ≥ 3`
- 簇间最远距离 `D = max pairwise dist > 5000m`
- `line_ratio = D / L > 0.3`（L = 按时间序 相邻簇累积距离）
- `cv(相邻距离) < 0.5`（距离均匀）
- `avg(dwell_days) ≤ 2`（平均每簇驻留 ≤ 2 天）

**物理含义**：高铁列车上的移动基站、车载移动 cell。

**样本命中**：0 / 878（样本 k_eff≥3 全 D < 2.2km，dwell=6，都不达标）

**未验证说明**：样本中 k_eff≥3 的 6 cell 都是**小范围稳定多质心**（校园/商圈/建筑群内多点），不是移动场景。**需要包含高铁沿线 cell 的数据验证**。

**配置项**：
- `label_rules.dynamic_min_span_m: 5000`
- `label_rules.dynamic_min_line_ratio: 0.3`
- `label_rules.dynamic_max_distance_cv: 0.5`
- `label_rules.dynamic_max_avg_dwell_days: 2`

#### 3.7.2 uncertain（多质心但未分类）

**判定**：`k_eff ≥ 3` 但不满足 dynamic 条件。

**物理含义**（推测）：
- 校园 / 商圈 / 建筑群内的多个 GPS 聚集点
- 非移动基站但非单质心
- 定期移动的移动基站（未来可能细化）

**样本命中**：6 / 878 = 0.7%

**操作建议**：UI 单独设 tab，方便运维人工标注 + 数据积累。

## 5. 完整 SQL 管道（生产实施时的参考伪代码）

所有中间表均 `UNLOGGED`，建索引后 `ANALYZE`。按 `batch_id` 隔离。

### 5.1 构建标签输入 `label_input_points`

```sql
DROP TABLE IF EXISTS rebuild5.label_input_points;
CREATE UNLOGGED TABLE rebuild5.label_input_points AS
SELECT DISTINCT ON (e.cell_id, e.lac, e.tech_norm, e.dev_id, DATE(e.event_time_std))
  :batch_id AS batch_id,
  e.cell_id, e.lac, e.bs_id, e.tech_norm, e.operator_code,
  e.dev_id,
  DATE(e.event_time_std) AS obs_date,
  e.event_time_std,
  e.lon_final, e.lat_final
FROM rebuild5.enriched_records e
WHERE e.batch_id = :batch_id
  AND e.gps_valid
  AND e.lon_final IS NOT NULL AND e.lat_final IS NOT NULL
  AND e.gps_fill_source_final = 'raw_gps'  -- 严格排除补数
ORDER BY e.cell_id, e.lac, e.tech_norm, e.dev_id,
         DATE(e.event_time_std), e.event_time_std, e.record_id;

CREATE INDEX idx_label_input_key ON rebuild5.label_input_points(cell_id, lac, tech_norm);
ANALYZE rebuild5.label_input_points;
```

### 5.2 DBSCAN 聚类 `label_clustered_points`

```sql
DROP TABLE IF EXISTS rebuild5.label_clustered_points;
CREATE UNLOGGED TABLE rebuild5.label_clustered_points AS
SELECT
  cell_id, lac, bs_id, tech_norm, operator_code,
  dev_id, obs_date, event_time_std, lon_final, lat_final,
  ST_ClusterDBSCAN(
    ST_MakePoint(lon_final * 85300, lat_final * 111000),
    eps := 250,
    minpoints := 4
  ) OVER (PARTITION BY cell_id, lac, tech_norm) AS cluster_id
FROM rebuild5.label_input_points;

CREATE INDEX idx_label_clustered_key ON rebuild5.label_clustered_points(cell_id, lac, tech_norm, cluster_id);
ANALYZE rebuild5.label_clustered_points;
```

**依赖**：PostGIS `ST_ClusterDBSCAN`（rebuild5 数据库已验证可用，版本 3.6.2）。

### 5.3 每簇聚合 `label_clusters`

```sql
DROP TABLE IF EXISTS rebuild5.label_clusters;
CREATE UNLOGGED TABLE rebuild5.label_clusters AS
SELECT
  cell_id, lac, bs_id, tech_norm, operator_code, cluster_id,
  COUNT(*) AS dev_day_pts,
  COUNT(DISTINCT dev_id) AS dev_cnt,
  COUNT(DISTINCT obs_date) AS day_cnt,
  MIN(obs_date) AS first_day,
  MAX(obs_date) AS last_day,
  (MAX(obs_date) - MIN(obs_date) + 1) AS dwell_days,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY lon_final) AS center_lon,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY lat_final) AS center_lat,
  PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY EXTRACT(EPOCH FROM event_time_std)) AS mid_time_epoch
FROM rebuild5.label_clustered_points
WHERE cluster_id IS NOT NULL  -- 排除 noise
GROUP BY cell_id, lac, bs_id, tech_norm, operator_code, cluster_id;

CREATE INDEX idx_label_clusters_key ON rebuild5.label_clusters(cell_id, lac, tech_norm);
ANALYZE rebuild5.label_clusters;
```

### 5.4 每 cell 的 k_eff 统计 `label_cell_kstats`

```sql
DROP TABLE IF EXISTS rebuild5.label_cell_kstats;
CREATE UNLOGGED TABLE rebuild5.label_cell_kstats AS
SELECT
  cell_id, lac, bs_id, tech_norm, operator_code,
  COUNT(*) AS k_raw,
  COUNT(*) FILTER (WHERE dev_day_pts >= 10) AS k_eff,
  SUM(dev_day_pts) FILTER (WHERE dev_day_pts >= 10) AS total_valid_pts
FROM rebuild5.label_clusters
GROUP BY cell_id, lac, bs_id, tech_norm, operator_code;

CREATE INDEX idx_label_kstats_key ON rebuild5.label_cell_kstats(cell_id, lac, tech_norm);
ANALYZE rebuild5.label_cell_kstats;
```

### 5.5 k_eff=2 双簇特征 `label_k2_features`

```sql
DROP TABLE IF EXISTS rebuild5.label_k2_features;
CREATE UNLOGGED TABLE rebuild5.label_k2_features AS
WITH vc AS (
  SELECT c.* FROM rebuild5.label_clusters c
  JOIN rebuild5.label_cell_kstats k USING (cell_id, lac, tech_norm)
  WHERE c.dev_day_pts >= 10 AND k.k_eff = 2
)
SELECT
  a.cell_id, a.lac, a.tech_norm,
  a.center_lon AS lon_a, a.center_lat AS lat_a,
  b.center_lon AS lon_b, b.center_lat AS lat_b,
  a.first_day AS fa, a.last_day AS la, a.day_cnt AS da,
  b.first_day AS fb, b.last_day AS lb, b.day_cnt AS db,
  SQRT(POWER((a.center_lon - b.center_lon) * 85300, 2)
     + POWER((a.center_lat - b.center_lat) * 111000, 2)) AS dist_m,
  GREATEST(0, LEAST(a.last_day, b.last_day) - GREATEST(a.first_day, b.first_day) + 1) AS overlap_days,
  LEAST(a.day_cnt, b.day_cnt) AS min_day_cnt,
  -- migration "无回头"：后段 last_day 是整个 cell 的最大 last_day
  CASE
    WHEN a.first_day < b.first_day THEN b.last_day = GREATEST(a.last_day, b.last_day)
    ELSE a.last_day = GREATEST(a.last_day, b.last_day)
  END AS no_comeback
FROM vc a
JOIN vc b
  ON a.cell_id = b.cell_id AND a.lac = b.lac
 AND a.tech_norm IS NOT DISTINCT FROM b.tech_norm
 AND a.cluster_id < b.cluster_id;
```

### 5.6 k_eff≥3 多质心特征 `label_kmany_features`

```sql
DROP TABLE IF EXISTS rebuild5.label_kmany_features;
CREATE UNLOGGED TABLE rebuild5.label_kmany_features AS
WITH vc AS (
  SELECT c.*,
    ROW_NUMBER() OVER (PARTITION BY c.cell_id ORDER BY c.mid_time_epoch) AS time_rank
  FROM rebuild5.label_clusters c
  JOIN rebuild5.label_cell_kstats k USING (cell_id, lac, tech_norm)
  WHERE c.dev_day_pts >= 10 AND k.k_eff >= 3
),
pairs AS (
  SELECT cell_id, lac, tech_norm,
    a.center_lon AS lon_a, a.center_lat AS lat_a,
    b.center_lon AS lon_b, b.center_lat AS lat_b,
    SQRT(POWER((a.center_lon - b.center_lon) * 85300, 2)
       + POWER((a.center_lat - b.center_lat) * 111000, 2)) AS d_pair
  FROM vc a JOIN vc b USING (cell_id, lac, tech_norm)
  WHERE a.time_rank < b.time_rank
),
adj AS (
  SELECT a.cell_id, a.lac, a.tech_norm, a.time_rank,
    SQRT(POWER((a.center_lon - b.center_lon) * 85300, 2)
       + POWER((a.center_lat - b.center_lat) * 111000, 2)) AS d_step
  FROM vc a
  JOIN vc b ON a.cell_id = b.cell_id AND a.lac = b.lac
   AND a.tech_norm IS NOT DISTINCT FROM b.tech_norm
   AND b.time_rank = a.time_rank + 1
)
SELECT
  vc.cell_id, vc.lac, vc.tech_norm,
  (SELECT MAX(d_pair) FROM pairs p WHERE p.cell_id = vc.cell_id AND p.lac = vc.lac) AS max_span_m,
  (SELECT SUM(d_step) FROM adj ad WHERE ad.cell_id = vc.cell_id AND ad.lac = vc.lac) AS total_path_m,
  (SELECT STDDEV(d_step) / NULLIF(AVG(d_step), 0) FROM adj ad
    WHERE ad.cell_id = vc.cell_id AND ad.lac = vc.lac) AS distance_cv,
  AVG(vc.dwell_days)::double precision AS avg_dwell_days
FROM vc
GROUP BY vc.cell_id, vc.lac, vc.tech_norm;
```

### 5.7 最终标签结果 `label_results`

```sql
DROP TABLE IF EXISTS rebuild5.label_results;
CREATE UNLOGGED TABLE rebuild5.label_results AS
SELECT
  :batch_id AS batch_id,
  k.cell_id, k.lac, k.bs_id, k.tech_norm, k.operator_code,
  k.k_eff, k.k_raw, k.total_valid_pts,
  t.p90_radius_m,
  -- 双簇特征（仅 k_eff=2）
  f2.dist_m AS pair_dist_m,
  CASE WHEN f2.min_day_cnt > 0
       THEN f2.overlap_days::float / f2.min_day_cnt END AS pair_overlap_ratio,
  f2.no_comeback AS pair_no_comeback,
  -- 多质心特征（仅 k_eff≥3）
  fm.max_span_m, fm.total_path_m,
  CASE WHEN fm.total_path_m > 0 THEN fm.max_span_m / fm.total_path_m END AS line_ratio,
  fm.distance_cv, fm.avg_dwell_days,
  -- 标签
  CASE
    WHEN k.k_eff = 0 THEN 'insufficient'
    WHEN k.k_eff = 1 AND t.p90_radius_m < 1200 THEN 'stable'
    WHEN k.k_eff = 1 AND t.p90_radius_m <= 10000 THEN 'large_coverage'
    WHEN k.k_eff = 1 THEN 'oversize_single'
    WHEN k.k_eff = 2 AND f2.dist_m < 5000
         AND f2.overlap_days::float / NULLIF(f2.min_day_cnt, 0) >= 0.5
         THEN 'dual_cluster'
    WHEN k.k_eff = 2 AND f2.dist_m < 5000
         AND f2.overlap_days = 0
         AND f2.min_day_cnt >= 2  -- 跳后稳的兜底
         AND f2.no_comeback
         THEN 'migration'
    WHEN k.k_eff = 2 THEN 'uncertain'  -- 双簇但不满足 dual/migration 条件
    WHEN k.k_eff >= 3
         AND fm.max_span_m > 5000
         AND fm.max_span_m / NULLIF(fm.total_path_m, 0) > 0.3
         AND fm.distance_cv < 0.5
         AND fm.avg_dwell_days <= 2
         THEN 'dynamic'
    WHEN k.k_eff >= 3 THEN 'uncertain'
    ELSE 'unknown'
  END AS label
FROM rebuild5.label_cell_kstats k
LEFT JOIN rebuild5.trusted_cell_library t
  ON t.cell_id = k.cell_id AND t.lac = k.lac
 AND t.tech_norm IS NOT DISTINCT FROM k.tech_norm
 AND t.batch_id = :batch_id
LEFT JOIN rebuild5.label_k2_features f2
  ON f2.cell_id = k.cell_id AND f2.lac = k.lac
 AND f2.tech_norm IS NOT DISTINCT FROM k.tech_norm
LEFT JOIN rebuild5.label_kmany_features fm
  ON fm.cell_id = k.cell_id AND fm.lac = k.lac
 AND fm.tech_norm IS NOT DISTINCT FROM k.tech_norm;

CREATE INDEX idx_label_results_key ON rebuild5.label_results(batch_id, cell_id);
ANALYZE rebuild5.label_results;
```

### 5.8 回灌到 trusted_cell_library

在 Step 5 pipeline 的 `publish_cell_library` 之后追加一步：

```sql
UPDATE rebuild5.trusted_cell_library t
SET drift_pattern = l.label,
    centroid_pattern = CASE
        WHEN l.label IN ('dual_cluster', 'migration') THEN l.label
        WHEN l.label IN ('dynamic', 'uncertain') THEN 'multi_cluster'
        ELSE NULL END,
    is_multi_centroid = (l.k_eff >= 2),
    is_dynamic = (l.label = 'dynamic')
FROM rebuild5.label_results l
WHERE t.batch_id = :batch_id
  AND l.batch_id = :batch_id
  AND t.cell_id = l.cell_id AND t.lac = l.lac
  AND t.tech_norm IS NOT DISTINCT FROM l.tech_norm;
```

**注意**：新规则下 `drift_pattern` 语义扩展到 8 个标签（含 stable/large_coverage/dual_cluster/migration/dynamic/uncertain/insufficient/oversize_single），前端枚举类型 `DriftPattern` 需同步扩展。

## 10. 设计原则回顾（源于 Phase B 讨论）

1. **质心有效性门槛**（每簇 dev-day ≥ 10）—— 避免"假多簇"误判，这是整个规则的基石
2. **只用 raw_gps**—— 补数（ss1_own/trusted_cell）是"旧质心传染的衍生点"，会污染多质心判定
3. **先单质心再多质心**—— 分层排查，减少边界 case 混淆
4. **严格 migration 定义**（overlap=0 + 跳后稳 + 不回头）—— 宁漏毋错，避免周期切换被误判为迁移
5. **dynamic 规则现在定而不等样本**—— 为未来放大数据做好准备
6. **uncertain 是合法的未分类状态**—— 留给运维人工标注 + 后续规则迭代
