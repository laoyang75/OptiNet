# 第一阶段修改意见：Step 1 UI 小修复

## 目标

本阶段只做 Step 1 页面的小修复，目标是：

- 让页面文案、统计口径、边界说明与当前实现一致
- 保证 ETL 主流程有效的前提下，减少 UI 对未开发能力的误导
- 不在本阶段扩展多数据源管理、复杂交互或新的后台统计能力

## 范围边界

以下内容在本阶段明确不开发：

- 多数据源注册/切换能力
- 数据源新增 / 编辑 / 停用交互
- 字段审计页的实时覆盖率、样本值、排序筛选能力
- 清洗页的规则条件展开器
- 重新设计 Step 1 统计表或补做细粒度解析统计

本阶段允许做的只有：

- 页头文案修正
- 空状态文案修正
- 说明卡、提示文案、边界说明补齐
- 明显错误的数字口径修正
- 对当前统计口径做降级表达，避免把混合口径说成单阶段结果

## 页面级修改建议

### 1. 数据源注册页

当前问题：

- 页面标题叫“数据源注册”，但实际只有只读展示
- 容易让用户误以为当前支持新增 / 编辑 / 停用
- 空状态文案仍写死 `sample_6lac`

建议修改：

- 页头补一句：`当前为单活只读模式，暂不支持在线新增 / 编辑 / 停用数据源`
- 保留现有表格，不新增按钮
- 空状态文案改成与当前数据集无关的通用文案
- 如果后端已有 `dataset_key / source_desc / time_range`，优先展示“当前处理数据上下文”

建议触达文件：

- `rebuild5/frontend/design/src/views/etl/DataSource.vue`

### 2. 字段审计页

当前问题：

- 页面名称和描述容易让人理解为“实时字段审计”
- 实际返回的是冻结字段定义与目标结构，不是实时覆盖率报告

建议修改：

- 页头描述改为：`当前页展示 Step 1 冻结字段定义与目标结构，不代表实时源表覆盖率审计`
- 保留分组表格，不承诺样本值、覆盖率排序、字段筛选
- 页面内部补一条静态说明：`实时字段覆盖率审计暂未纳入本阶段`

建议触达文件：

- `rebuild5/frontend/design/src/views/etl/FieldAudit.vue`

### 3. 解析页

当前问题：

- 页面中的“字段覆盖率变化”并非纯解析阶段结果，而是复用了当前 Step 1 汇总口径
- 容易把“解析 + 清洗 + 补齐后的覆盖变化”误读成“解析阶段新增覆盖”

建议修改：

- 页头或表头改成更保守的表达，例如：`当前 Step 1 产出覆盖变化`
- 若不改后端统计，就不要再使用“解析前后字段覆盖率”这种强口径表述
- 给覆盖率表补一条注释：`当前展示为 Step 1 当前汇总口径，非纯解析阶段独立审计`

建议触达文件：

- `rebuild5/frontend/design/src/views/etl/Parse.vue`

### 4. 清洗页

当前问题：

- 文档口径曾写成 `16 条规则`，与当前实现的 `ODS-001 ~ ODS-018` 共 19 条不一致
- 页面已能看命中数，但还应更明确“最终删行规则”和“置空规则”的区别

建议修改：

- 页面静态说明统一改成 `19 条规则`
- 规则说明区补充一句：`最终删行条件为 cell_id IS NULL 或 event_time_std IS NULL`
- 保留当前表格，不新增复杂展开

建议触达文件：

- `rebuild5/frontend/design/src/views/etl/Clean.vue`
- 相关说明文档

### 5. 补齐页

当前问题：

- 页面已经强调“不是 Step 4”，但还没有把 `cell_infos`、`ss1 <= 60s`、`ss1 > 60s` 三条边界说清楚

建议修改：

- 在边界说明区补成三条明确规则：
- `cell_infos` 行之间可全字段互补
- `ss1` 在同 `record_id + cell_id` 且时间差 `<= 60s` 时允许全补
- `ss1` 在时间差 `> 60s` 时只补 `operator / lac`
- 保留现有来源分布与覆盖率表，不改补齐逻辑

建议触达文件：

- `rebuild5/frontend/design/src/views/etl/Fill.vue`

## 实施顺序

### P0

- 数据源页：只读模式文案与空状态修正
- 字段审计页：改成“冻结定义展示”口径
- 解析页：弱化“纯解析覆盖率”表述
- 清洗页：统一 19 条规则口径
- 补齐页：补齐三条边界说明

### P1

- 统一 Step 1 五页页头用词，避免同一模块内同时出现“注册 / 管理 / 审计 / 实时分析”等过强承诺

## 验收标准

- 页面不再暗示当前支持未开发的多数据源管理能力
- 页面不再把静态定义页说成实时审计页
- 页面不再把混合统计口径误表述为纯解析阶段结果
- Step 1 五页的数字口径和说明文字不再互相冲突
- 不修改 ETL 主流程，不新增新的后台依赖
