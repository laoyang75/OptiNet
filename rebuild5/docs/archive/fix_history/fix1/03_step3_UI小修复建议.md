# 第一阶段修改意见：Step 3 UI 小修复

## 目标

本阶段只对 Step 3 页面做轻量 UI 收口，不扩展复杂交互。

目标是：

- 把候选池清理结果展示出来
- 把“分钟级”表达澄清为“批内观察点去重”
- 对当前还做不到的统计或钻取能力明确标注待开发

## 本阶段不做

- 详情展开器重做
- 更复杂的 diff 钻取
- dormant / retired 的完整治理面板
- 更细粒度历史证据回放

## 建议修改项

### 1. 总流转页增加候选池清理摘要

当前问题：

- `waiting_pruned_cell_count` 没有在页面上展示
- 用户无法快速看到本批候选池是否发生清理

建议：

- 总流转页增加“候选池清理”摘要
- 展示 `waiting_pruned_cell_count`
- 对 `dormant_marked_count` 若当前未启用，标注“待开发”

建议触达文件：

- `rebuild5/frontend/design/src/views/evaluation/FlowOverview.vue`

### 2. “分钟级”表达做语义澄清

当前问题：

- 容易把“分钟级观察点去重”误解成“分钟级运行或分钟级评估”

建议：

- 页面和文档统一表述为：
- `批次仍按天运行`
- `分钟级仅用于批内独立观察点去重`

建议触达文件：

- `rebuild5/docs/03_流式质量评估.md`
- `rebuild5/docs/human_guide/03_Step3_流式质量评估.md`
- `rebuild5/ui/05_流转评估页面.md`

### 3. 不可实现统计先标待开发

当前问题：

- 当前页面并不能完整覆盖 dormant/retired 治理或所有对象详情钻取

建议：

- 当前能展示的就直接展示：
- 状态分布
- diff 摘要
- 规则影响
- 趋势
- 候选池等待超时清理
- 当前做不到的：
- dormant/retired 治理统计
- 更复杂的对象详情钻取
- 一律在页面或文档中标为“待开发”

## 验收标准

- 总流转页能展示候选池等待超时清理
- 页面不再把“分钟级”误表达成调度频率
- 当前做不到的统计在页面或文档中明确标为待开发
