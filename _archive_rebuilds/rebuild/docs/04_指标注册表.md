## 1. 指标体系概述

### 1.1 指标层次

| 层次 | 说明 | 存储位置 |
|------|------|---------|
| **步骤级指标** | 每个步骤执行后产出的汇总指标 | workbench.wb_step_metric |
| **层级快照** | 每层（Layer）的行数和通过标志 | workbench.wb_layer_snapshot |
| **质量门控** | 关键验收条件的通过/失败判定 | workbench.wb_gate_result |
| **异常统计** | BS/Cell级异常聚合计数 | workbench.wb_anomaly_stats |
| **对账校验** | 上下游数据一致性验证 | workbench.wb_reconciliation |

### 1.2 指标命名规范

```
{object}_{measure}_{qualifier}
```

- object: 对象层级（lac / bs / cell / record / gps / signal）
- measure: 度量（count / ratio / pct / avg / p50 / p90 / max）
- qualifier: 限定（valid / missing / filled / drift / collision 等）

## 4. 质量门控指标

存储在 `workbench.wb_gate_result`，每次 run 执行后自动计算：

| 门控代码 | 门控名称 | 验收规则 | 严重级别 |
|---------|---------|---------|---------|
| G01_ROW_CONSERVATION | 行数守恒 | step40_rows = fact_final_rows | CRITICAL |
| G02_LAC_COVERAGE | LAC覆盖率 | trusted_lac_cnt / total_lac_cnt ≥ 阈值 | HIGH |
| G03_GPS_FILL_RATE | GPS回填率 | filled / (missing+drift) ≥ 阈值 | HIGH |
| G04_SIGNAL_FILL_RATE | 信号补齐率 | missing_after ≤ missing_before | MEDIUM |
| G05_NO_INVALID_LAC | 无无效LAC泄漏 | Layer5中LAC_dec NOT IN 占位值 | CRITICAL |
| G06_FIELD_CONTRACT | 字段契约完整 | Layer5画像包含所有契约字段 | CRITICAL |
| G07_METRIC_RECONCILE | 指标对账一致 | fact表聚合 = metrics表数值 | HIGH |
| G08_COLLISION_RATIO | 碰撞占比合理 | collision_suspect / total_bs ≤ 阈值 | MEDIUM |
| G09_FILTER_RATE | 过滤率合理 | filter_rate ∈ 期望区间 | MEDIUM |
| G10_PROFILE_COVERAGE | 画像覆盖率 | profile_cnt / upstream_distinct_cnt ≥ 阈值 | LOW |

## 5. 指标维度

所有步骤级指标支持以下维度切分：

| 维度 | 字段 | 说明 |
|------|------|------|
| 运营商 | operator_id_raw | 五大PLMN |
| 制式 | tech_norm | 4G / 5G |
| 日期 | report_date | 按上报日期 |
| 运行实例 | run_id | 按运行批次 |

维度组合存储格式：

```
-- 全量汇总行
dimension_key = 'ALL'

-- 按运营商
dimension_key = 'operator:46000'

-- 按制式
dimension_key = 'tech:4G'

-- 按运营商+制式
dimension_key = 'operator:46000|tech:4G'
```

## 6. 指标计算时机

| 时机 | 触发条件 | 计算内容 |
|------|---------|---------|
| 步骤完成后 | 每个步骤 SQL 执行完毕 | 该步骤的所有指标 |
| 层级完成后 | 一个 Layer 的所有步骤完成 | 层级快照 + 层间对账 |
| 全链路完成后 | Step52(画像)完成 | 门控验收 + 异常统计 |
| 手动触发 | 用户在工作台点击"重算指标" | 指定步骤的指标重算 |
