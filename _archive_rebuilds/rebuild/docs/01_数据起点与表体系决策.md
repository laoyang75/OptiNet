## 1. 数据起点决策

**决策：从 Layer0 完整起点开始。**

理由：重构的目的是确保流程每一步都准确清晰且文档化，不跳过任何环节。

### 起点数据

| 表名 | 行数 | 大小 | 说明 |
|------|------|------|------|
| Y_codex_Layer0_Lac | 1.19亿 | 47GB | LAC 解析的原始记录 |
| Y_codex_Layer0_Gps_base | 1.33亿 | 52GB | GPS 解析的原始记录 |

### 完整链路

```
Layer0（原始数据）
  → 标准化解析（中文字段→英文、运营商识别、制式识别）
  → 合规过滤（运营商/LAC编码/位置/稳定性）
  → 可信 LAC
  → Cell 统计
  → 可信 BS
  → GPS 修正
  → 信号补齐
  → 反向纯化（BS→Cell）
  → 完整回归（从 Layer0 重放全量）
  → 画像/基线
```

### 完整回归的含义

完整回归必须回到 Layer0 全量数据重新处理。这是"有效 cell_id = 有效上报"原则的要求——通过 LAC 合规检查的全量数据都应进入后续处理。

## 2. 表命名决策

**决策：完全新命名，放在独立 schema 中。**

### 2.1 Schema 规划

| Schema | 用途 | 说明 |
|--------|------|------|
| `legacy` | 旧表参考 | 将现有 Y_codex 表移入，仅用于开发期间对比验证 |
| `pipeline` | 治理链路产物 | 新命名的链路步骤产出表 |
| `workbench` | 工作台元数据 | wb_* 表 |
| `meta` | 字段治理 | meta_* 表 |

### 2.2 治理链路产物表命名

| 步骤 | 新表名 | 说明 | 旧表参考 |
|------|--------|------|---------|
| 数据起点 | `pipeline.raw_records` | 标准化后的全量记录 | Layer0_Lac / Layer0_Gps_base |
| 可信LAC | `pipeline.dim_lac_trusted` | 可信LAC维表 | Layer2_Step04_Master_Lac_Lib |
| Cell统计 | `pipeline.dim_cell_stats` | Cell统计维表 | Layer2_Step05_CellId_Stats_DB |
| 合规过滤 | `pipeline.fact_filtered` | LAC合规过滤后的明细 | Layer2_Step06_L0_Lac_Filtered |
| 可信BS | `pipeline.dim_bs_trusted` | BS主库 | Layer3_Step30_Master_BS_Library |
| GPS修正 | `pipeline.fact_gps_corrected` | GPS修正后记录 | Layer3_Step31_Cell_Gps_Fixed |
| 信号补齐 | `pipeline.fact_signal_filled` | 信号补齐后记录 | Layer3_Step33_Signal_Fill_Simple |
| 完整回归 | `pipeline.fact_final` | 最终明细 | Layer4_Final_Cell_Library |
| LAC画像 | `pipeline.profile_lac` | LAC级画像 | Layer5_Lac_Profile |
| BS画像 | `pipeline.profile_bs` | BS级画像 | Layer5_BS_Profile |
| Cell画像 | `pipeline.profile_cell` | Cell级画像 | Layer5_Cell_Profile |
| Cell-BS映射 | `pipeline.map_cell_bs` | Cell到BS的映射关系 | Layer3_Cell_BS_Map |

辅助中间表（需要时才创建）：

| 新表名 | 说明 | 旧表参考 |
|--------|------|---------|
| `pipeline.stats_lac` | LAC统计中间结果 | Layer2_Step03_Lac_Stats_DB |
| `pipeline.stats_base_raw` | 原始基础统计 | Layer2_Step01_BaseStats_Raw |
| `pipeline.detect_collision` | 碰撞检测结果 | Layer3_Step37_Collision_Data_Insufficient_BS |
| `pipeline.detect_anomaly_bs` | BS异常标记 | Layer3_Step36_BS_Id_Anomaly_Marked |
| `pipeline.compare_gps` | GPS修正对比 | Layer3_Step32_Compare |
| `pipeline.compare_signal` | 信号补齐对比 | Layer3_Step34_Signal_Compare |

### 2.3 命名规则

- `dim_` 前缀：维度表（对象级，如 LAC/BS/Cell）
- `fact_` 前缀：事实表（记录级明细）
- `profile_` 前缀：画像/基线表
- `map_` 前缀：映射关系表
- `stats_` 前缀：统计中间表
- `detect_` 前缀：检测/标记结果
- `compare_` 前缀：对比分析表

所有字段使用英文命名，不使用中文列名。
