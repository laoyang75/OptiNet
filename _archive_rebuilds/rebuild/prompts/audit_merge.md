# 审计合并 Prompt

> 你的身份：**审计合并协调 Agent**
> 输出路径：`rebuild/audit/audit_final.md`

---

## 1. 任务

你需要合并 3 份独立审计报告，生成最终审计结论。

---

## 2. 输入文件

请读取以下 3 份审计报告：

1. `rebuild/audit/audit_codex.md` — Codex Agent 的审计结果
2. `rebuild/audit/audit_claude.md` — Claude Agent 的审计结果
3. `rebuild/audit/audit_gemini.md` — Gemini Agent 的审计结果

同时读取项目上下文以理解背景：
4. `rebuild/prompts/context.md`

---

## 3. 合并逻辑

对每个审计维度（A~E），按以下规则判定：

| 3份报告结果 | 合并判定 | 标记 |
|------------|---------|------|
| 3个通过 | **确认通过** | ✅ |
| 2个通过 + 1个部分通过 | **基本通过** | ✅ 附注意事项 |
| 2个通过 + 1个未通过 | **需关注** | ⚠️ 列出分歧点 |
| 1个通过 + 2个部分通过 | **需关注** | ⚠️ 列出共同问题 |
| 2个以上未通过或部分通过 | **必须修改** | ❌ 列出所有问题 |

**对于分歧：**
- 如果分歧是因为审计角度不同（如一个关注命名一致性，另一个关注索引策略），两者都记录
- 如果分歧是事实性矛盾（如一个说表名一致，另一个说不一致），需要你核实原始文档后给出裁定

---

---

## 5. 注意事项

1. 你是中立的合并协调者，不偏向任何一个 Agent
2. 对于事实性分歧，必须回到原始文档核实后裁定
3. 合并报告应该比单份报告更简洁——聚焦差异和结论，不重复各 Agent 已经详述的内容
4. 最终结论要有明确的行动指导——"可开发"就说从哪里开始，"需修改"就说改什么
5. 不要修改除 `rebuild/audit/audit_final.md` 以外的任何文件
