# 异常工作台 设计说明

## 页面目标
理解两类异常（对象级 + 记录级）的影响路径，回答"这个异常影响了什么"和"怎么处理"。对象级异常改变对象状态，记录级异常影响单条事实去向。

## 这个页面主要回答什么问题
1. 对象级异常数量和类型分布
2. 哪些对象禁锚、禁基线
3. 每个对象异常的下游影响范围
4. 证据趋势（恶化/稳定/改善）
5. 异常新增/解除趋势
6. 记录级异常的总量和分类
7. 各类记录级异常的事实去向（fact_governed / fact_rejected）
8. 记录级异常是否影响锚点/基线判断

## 页面结构：双视角 Tab

### 一级视角
| Tab 名称 | 说明 |
|---------|------|
| 对象级异常 | 改变对象状态的异常：collision_suspect, collision_confirmed, dynamic, migration_suspect, gps_bias |
| 记录级异常 / 结构不合规 | 不改变对象状态，但影响事实去向：normal_spread, single_large, 结构不合规, GPS缺失回填, donor补齐 |

### 顶部汇总条
同时展示两类异常的统计：
- 对象级异常总数 + delta + 按类型分布 + 禁锚/禁基线数
- 记录级异常总数 + 本批新增 + 按类型分布
- 下游影响：对象异常影响的 BS/LAC 数、记录进入 fact_rejected 数、记录进入 fact_governed（条件性）数

## 用户主要操作
1. 通过一级 Tab 切换对象级/记录级视角
2. 在对象级视角中按类型 sub-tab 分组查看
3. 按严重度/趋势筛选对象异常
4. 展开行查看影响路径
5. 点击对象跳转详情
6. 在记录级视角查看各类异常的事实去向和处理说明
7. 查看汇总趋势

## 页面区块说明

### 对象级异常 Tab
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 汇总条 | 5% | 总量+分类+delta（两类异常合并展示） | 异常数/按类型/禁锚禁基线/下游影响 | 是 | 否 |
| 视角Tab | 3% | 切换对象级/记录级 | 两个tab+计数 | 否 | 否 |
| Sub-Tab+筛选 | 5% | 分组过滤 | 类型/严重度/趋势 | 否 | 否 |
| 统计卡片行 | 15% | 类型分布+趋势+新增解除 | 占比图/近N批趋势 | 是 | 否 |
| 异常表格 | 50% | 逐对象审查 | 主键/类型/严重度/发现批次/事实去向/禁锚/禁基线/趋势/影响/动作 | 是 | 是(展开行/跳转详情) |
| 行展开 | (内嵌) | 影响路径 | 碰撞组/受影响Cell/baseline候选/时间线/事实去向说明 | 否 | 是 |

### 对象级表格列定义
| 列名 | 说明 |
|------|------|
| 对象主键 | BS/Cell 唯一标识，可点击跳转详情 |
| 异常类型 | collision_suspect / collision_confirmed / dynamic / migration_suspect / gps_bias |
| 严重度 | 高/中/低 |
| 发现批次 | 首次发现异常的批次号 |
| 事实去向 | fact_pending_issue（对象级异常统一去向） |
| 是否禁锚 | 该对象是否被禁止参与锚点计算 |
| 是否禁基线 | 该对象是否被禁止参与基线候选 |
| 证据趋势 | 恶化/稳定/改善/新发现 |
| 下游影响 | 影响的 Cell 数量或 BS 关系 |
| 建议动作 | 系统推荐的处置动作 |

### 记录级异常 Tab
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 说明文字 | 3% | 解释记录级异常的特征 | 描述文本 | 否 | 否 |
| 统计卡片行 | 15% | 类型分布+去向分布+本批新增 | 类型/数量/去向 | 是 | 否 |
| 异常表格 | 45% | 按类型汇总展示 | 类型/数量/新增/去向/锚点影响/基线影响/说明 | 是 | 否 |
| 规则说明卡 | 15% | 处理规则详解 | fact_governed条件/fact_rejected条件/区别说明 | 否 | 否 |

### 记录级表格列定义
| 列名 | 说明 |
|------|------|
| 异常类型 | normal_spread / single_large / 结构不合规 / GPS缺失回填 / donor信号补齐 |
| 数量 | 当前累计条数 |
| 本批新增 | 本批次新发现的条数 |
| 事实去向 | fact_governed（条件性/标注来源）或 fact_rejected |
| 是否影响锚点 | 不影响 / 有条件影响 / 不参与 / 不影响锚点判断 |
| 是否影响基线 | 不影响 / 有条件影响 / 不参与 / 由标签决定 |
| 处理说明 | 具体的处理方式描述 |

## 状态表达规则
- collision_suspect: 橙色 | collision_confirmed: 红色 | dynamic: 蓝色 | migration_suspect: 紫色 | gps_bias: 橙色
- normal_spread: 黄色 | single_large: 橙色 | 结构不合规: 灰色 | GPS缺失回填: 蓝色 | donor补齐: 紫色
- 严重度: 高=红底, 中=橙底, 低=黄底
- 趋势: 恶化=红, 稳定=灰, 改善=绿, 新发现=蓝
- 事实去向: fact_governed=绿色, fact_rejected=红色, fact_pending_issue=灰色

## 下钻路径
- 对象主键 → 对象详情
- 展开行受影响Cell → 对象详情
- 发现批次 → 批次中心

## 组件边界建议
- AnomalyWorkspaceView.vue（主视图，管理双视角切换）
- ObjectAnomalyPanel.vue（对象级异常面板）
- RecordAnomalyPanel.vue（记录级异常面板）
- AnomalyTable.vue（对象级表格）
- RecordAnomalyTable.vue（记录级表格）
- AnomalyExpandedRow.vue（展开行详情）
- ImpactPathVisualization.vue（影响路径可视化）
- AnomalyTypeSummary.vue（类型分布统计卡）

## 读模型建议
- GET /api/anomaly-workspace?view=object&type=all（对象级异常列表）
- GET /api/anomaly-workspace?view=record（记录级异常汇总）
- GET /api/anomaly-workspace/:key/impact（对象影响路径详情）
- GET /api/anomaly-workspace/summary（顶部汇总，含两类异常统计）

## 空状态 / 错误状态 / 重跑中状态
- 对象级空: "当前没有对象级异常，系统运行健康"
- 记录级空: "当前没有记录级异常"

## 开发注意事项
- 影响路径按需加载（展开行时请求）
- 碰撞对象组需后端预计算
- 一级视角 Tab 切换为前端路由或状态管理，不重新请求
- 记录级异常表格为汇总视图（按类型聚合），非逐条展示
- 顶部汇总条始终展示两类异常数据，不随视角 Tab 切换而隐藏
