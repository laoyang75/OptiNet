# 对象详情 设计说明

## 页面目标
对单个对象进行深度诊断——为什么处于当前状态、证据链是什么、资格原因、下游影响。

## 这个页面主要回答什么问题
1. 该对象当前状态和资格
2. 状态是何时/因何变化的（时间线）
3. 最近批次的事实分布（governed/pending/rejected各多少）
4. 资格禁用的具体原因
5. 关联的异常类型和严重度
6. 对下游 BS/LAC/baseline 的影响

## 用户主要操作
1. 查看当前快照和画像
2. 追溯状态历史时间线
3. 检查资格禁用原因
4. 导航到关联的 BS/LAC 详情
5. 导航到关联的异常

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 头部状态区 | 10% | 主键+状态+资格一览 | 主键/lifecycle/health/watch/anchorable/baseline_eligible | 否 | 否 |
| 基本信息 | 15% | 对象属性 | 运营商/制式/LAC/CellID/BS/时间 | 否 | 是(→BS详情) |
| 画像摘要 | 15% | 空间和信号质量 | 质心/半径/样本数/设备数/天数/GPS率/补齐率 | 否 | 否 |
| 事实分布 | 15% | 近N批事实去向 | 批次/各层计数 | 是(批次间变化) | 是(→批次中心) |
| 状态历史 | 15% | 状态变更时间线 | 时间/批次/变化内容 | 否 | 是(→批次) |
| 资格原因 | 10% | 禁用原因说明 | 锚点原因/基线原因 | 否 | 否 |
| 关联异常 | 10% | 异常信息 | 类型/批次/严重度 | 否 | 是(→异常工作台) |
| 下游影响 | 10% | 影响范围 | BS/LAC/对象数 | 否 | 是(→BS/LAC详情) |

## 筛选器与切换器
- 事实分布: 显示近5批 / 近10批
- 状态历史: 显示全部 / 仅lifecycle / 仅health / 仅资格

## 状态表达规则
- 同对象浏览页的 lifecycle/health/watch/qualification 表达
- 时间线用颜色圆点区分变化类型
- 超标值用红色高亮（如 P90 > 1500m）

## 下钻路径
- 所属 BS → BS 对象详情
- 所属 LAC → LAC 对象详情
- 关联异常 → 异常工作台
- 事实分布中批次 → 批次中心
- 状态历史中批次 → 批次中心

## 组件边界建议
- ObjectDetailView.vue (页面)
- ObjectHeader.vue (头部)
- ProfileSummaryCard.vue (画像)
- FactDistributionTable.vue (事实分布)
- StatusTimeline.vue (状态历史)
- QualificationReasonCard.vue (资格原因)
- RelatedAnomalyCard.vue (关联异常)
- DownstreamImpactCard.vue (下游影响)

## 读模型建议
- GET /api/objects/:key/detail (含snapshot/profile/history/facts/anomalies/impact)
- 一次请求返回全部，避免多次往返

## 空状态 / 错误状态 / 重跑中状态
- 404: "未找到该对象"
- 新对象(waiting): 画像和历史可能很少，用空状态提示"数据积累中"

## 开发注意事项
- 详情页也可作为抽屉在对象列表页侧边打开
- 状态时间线数据量可能很大，考虑分页或折叠
- P90 半径超标等需要后端标注阈值
