# 基线/画像 设计说明

## 页面目标
帮助用户理解 baseline 为什么刷新/没刷新、变化来自哪里、稳定性风险在哪里。

## 这个页面主要回答什么问题
1. 当前 baseline 版本和触发原因
2. 覆盖了多少对象，质量如何
3. 与上一版差异来自哪里（waiting晋升/异常剔除/画像漂移）
4. 稳定性风险在哪里
5. 版本演进趋势

## 用户主要操作
1. 查看当前 baseline 概况和触发原因
2. 查看与上一版差异（新增/移除/变更）
3. 点击差异对象跳转详情
4. 评估稳定性风险
5. 浏览版本历史

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 必须字段 | 是否需要 delta | 是否支持下钻 |
|---------|---------|---------|---------|---------------|-------------|
| 基线概况 | 10% | 版本+触发信息 | version/rule_set/创建时间/触发批次 | 否 | 是(→批次) |
| 覆盖与质量 | 15% | 对象覆盖+质量指标 | Cell/BS/LAC数/异常占比/GPS可信度/信号覆盖 | 是(较上版delta) | 否 |
| 刷新触发原因 | 15% | 为什么刷新 | 触发条件/类型/waiting贡献/异常贡献 | 否 | 是(→批次) |
| 版本差异 | 30% | 具体变化 | 新增/移除/变更列表 | 否 | 是(→对象详情) |
| 稳定性风险 | 15% | 风险评估 | 评分/风险因素/正面因素 | 否 | 否 |
| 版本历史 | 15% | 演进趋势 | 版本号/时间/触发批次/对象数/变化 | 是 | 是(→历史版本) |

## 状态表达规则
- 稳定性: ≥80 绿色, 60-79 黄色, <60 红色
- 差异类型: 新增=绿, 移除=红, 变更=橙
- 风险因素: ⚠黄色, ✓绿色

## 下钻路径
- 触发批次 → 批次中心
- 差异对象 → 对象详情
- 版本历史某版 → 该版详情

## 组件边界建议
- BaselineProfileView.vue, BaselineOverview.vue, CoverageMetrics.vue
- RefreshTriggerCard.vue, VersionDiffTable.vue, StabilityAssessment.vue

## 读模型建议
- GET /api/baseline/current (概况+触发+覆盖)
- GET /api/baseline/current/diff (与上版差异)
- GET /api/baseline/history (版本列表)

## 空状态 / 错误状态 / 重跑中状态
- 空: "尚未生成 baseline，请先完成初始化"
- 重跑: "重跑中，baseline 可能在下一批次后更新"

## 开发注意事项
- 稳定性评分需后端计算
- 差异列表可能很长，需分页或折叠
