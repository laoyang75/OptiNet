## Q1: 对象浏览页 vs LAC/BS/Cell 画像页的关系

**问题**：当前设计中有一个通用的"对象浏览"页（03_objects.html），也新增了 LAC 画像、BS 画像、Cell 画像三个独立页面。它们之间的导航关系如何组织？

**我的理解**：
- "对象浏览"页是治理视角（lifecycle_state、health_state、资格状态），关注对象在治理流程中的状态
- LAC/BS/Cell 画像页是数据质量视角（GPS来源构成、信号质量、空间精度），关注对象的画像属性和数据质量

**选项**：
- A) 对象浏览页保留为主入口，LAC/BS/Cell 画像页作为子页面通过 Tab 切换访问
- B) 对象浏览页和画像页并列放在侧边栏，用户自行选择进入哪个视角
- C) 合并为一个页面，用 Tab 切换"治理状态"和"画像质量"两个视角

## Q2: 画像页中的 rebuild2 字段是否保留

**问题**：rebuild2 的画像页包含一些特定字段（如 `classification_v2`、`gps_confidence`、`signal_confidence`、`centroid_span_m`），这些字段在 rebuild3 的冻结文档中没有明确定义。是否保留这些字段，还是用 rebuild3 的新模型替代？

**我的理解**：
- rebuild3 用 `health_state` + `anchorable` + `baseline_eligible` 替代了旧的 `classification_v2` + `gps_confidence`
- 但画像层面的空间精度字段（P50/P90、覆盖面积）仍然有价值

**选项**：
- A) 完全使用 rebuild3 新模型字段，不保留 rebuild2 的分类和可信度字段
- B) 保留空间精度字段（P50/P90/面积/中心点），但用 rebuild3 的状态模型替代分类和可信度
- C) 两套字段都保留，rebuild2 字段标记为"参考/历史"

## Q3: 搜索功能的范围

**问题**：全局搜索应该搜索哪些内容？

**我的理解**：用户主要需要通过主键快速找到特定对象。

**选项**：
- A) 仅搜索对象主键（LAC 编号、BS ID、Cell ID），跳转到对应对象详情
- B) 搜索对象主键 + batch_id + run_id，跳转到对应页面
- C) 全文搜索（包括异常类型、状态等），但实现复杂度较高

## Q4: 初始化页面是否应放入主导航

**问题**：初始化数据页是一个"查看初始化结果"的页面，使用频率较低。是否应该放入侧边栏主导航？

**我的理解**：初始化只跑一次或很少几次，日常使用不需要频繁访问。

**选项**：
- A) 放入侧边栏主导航，作为独立入口
- B) 放在"运行/批次中心"页面中，作为一个特殊的批次类型展示
- C) 放在侧边栏底部作为"系统"类入口，与启动器并列

## Q5: 侧边栏导航项数量

**问题**：加上新增页面，侧边栏可能有 10+ 个导航项。是否需要分组？

**当前导航项**：
1. 流转总览
2. 流转快照
3. 运行/批次中心
4. 对象浏览
5. 等待/观察工作台
6. 异常工作台
7. 基线/画像
8. 验证/对照
9. LAC 画像（新）
10. BS 画像（新）
11. Cell 画像（新）
12. 初始化（新）

**选项**：
- A) 按功能分组：治理主线（流转总览/快照/批次中心）→ 对象（对象浏览/LAC/BS/Cell）→ 工作台（等待/异常/基线）→ 验证（对照/初始化）
- B) 保持平铺，但 LAC/BS/Cell 画像放在"对象浏览"下作为二级菜单
- C) LAC/BS/Cell 画像不放入侧边栏，只能从"对象浏览"页的 Tab 进入

## Q6: delta 展示的统一性

**问题**：流转总览页已改为流程图视图，不再以 delta 对比为主。但其他页面（对象浏览、等待工作台等）仍大量使用 delta 展示。是否所有页面都需要 delta，还是仅部分页面需要？

**我的理解**：delta 是流式治理工作台的核心表达之一，但不是每个场景都需要。

**选项**：
- A) 所有页面都保留 delta 展示（当前设计）
- B) 仅首页（流转快照版）和批次中心使用 delta，其他页面只展示当前值
- C) 保留 delta，但改为可折叠/隐藏，用户自行决定是否查看
