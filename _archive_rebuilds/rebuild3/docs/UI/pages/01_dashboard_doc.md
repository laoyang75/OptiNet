# 仪表盘设计说明

## 页面目标
让用户在 5 秒内了解系统整体健康状态，快速定位需要关注的区域

## 用户主要操作
1. 扫视关键指标
2. 查看需要关注的警示项并跳转
3. 浏览最近批次执行情况
4. 确认当前版本上下文

## 页面区块说明
| 区块名称 | 面积占比 | 核心作用 | 核心字段 |
|---------|---------|---------|---------|
| 关键指标卡 | 15% | 一眼看到核心数量 | 活跃对象数、等待数、异常数、批次进度 |
| 系统健康概览 | 35% | health_state 和 lifecycle 分布 | 健康分布饼图、生命周期分布条 |
| 需要关注 | 20% | 引导用户进入问题区域 | 警示项列表，带跳转链接 |
| 最近批次 | 25% | 查看批次执行趋势 | 批次ID、窗口、四分流比例、晋升/异常 |
| 版本上下文 | 5% | 确认当前运行环境 | run_id, baseline_version, contract, rules |

## 筛选器 & 排序
- 无全局筛选器（首页为概览）
- 最近批次表格可按时间排序

## 状态表达规则
- 指标卡用颜色侧边线区分：green/amber/orange/indigo
- 健康分布用语义色饼图
- 批次状态用小圆点徽标

## 组件边界建议
- DashboardView.vue (页面)
  - MetricCardRow (4个 MetricCard)
  - HealthOverview (含 donut chart + lifecycle bar)
  - AttentionPanel (警示项列表)
  - RecentBatchTable (批次表格)
  - VersionContextBar (版本信息条)

## 读模型建议
- /api/v1/dashboard/summary — 聚合指标，可缓存 30s
- /api/v1/batch/recent?limit=5 — 最近批次
- /api/v1/alerts/active — 需要关注项
- 指标数据应为预聚合结果，不应实时 COUNT 大表

## 空状态 / 错误状态
- 首次启动无数据：显示引导卡"开始第一次初始化运行"
- API 超时：指标卡显示"--"，下方提示重试
- 部分数据缺失：已有数据正常显示，缺失区域灰色占位

## 开发注意事项
- 仪表盘是高频刷新页面，建议 30s 轮询或 SSE
- 饼图/条形图需 ECharts，注意按需引入
- 指标卡数字建议后端预计算，避免前端做大量聚合
