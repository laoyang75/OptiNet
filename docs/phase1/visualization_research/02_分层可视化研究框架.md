# 分层可视化研究框架（Layer_0 ~ Layer_5）

## 1. 总体方法
对每一层统一使用“六件套”描述，保证跨层可比：
1. 目标：这层存在的目的。
2. 输入：来源对象、行数、关键字段。
3. 处理：规则、阈值、补丁、例外。
4. 输出：产物对象、行数、主键、质量标签。
5. 变化：输入->输出的量化变化（数量、覆盖率、异常率）。
6. 风险：本层残留问题与对下游影响。

## 2. 每层必须呈现的可视化
1. 数据量漏斗图：输入行数 -> 过滤后 -> 补齐后 -> 输出。
2. 字段质量热力图：关键字段空值率/异常率前后对比。
3. 标签分布图：碰撞、严重碰撞、动态cell、bs_id异常、多运营商共建。
4. 规则命中图：每条规则命中行数与占比。
5. 样本穿透表：可点击查看 before/after/source/reason。

## 3. 每层必须回答的问题
1. 这一步让数据“更可信”还是“更完整”？
2. 哪些数据被丢弃，丢弃是否合理？
3. 哪些数据被补齐，补齐是否可解释？
4. 有没有把问题“藏起来”而不是“标记出来”？
5. 对下一层造成了什么结构性影响？

## 4. 闭环机制（发现问题 -> 修复）
1. 发现：通过门禁SQL或可视化异常触发问题单。
2. 定位：锁定 Layer + 规则 + 样本 + 影响范围。
3. 修复：Agent提交修复方案（SQL/逻辑/参数）。
4. 验证：自动产出修复前后差异报告。
5. 记录：写入补丁日志与决策日志。
6. 固化：可复用的修复策略进入下一轮基线。

## 5. 研究产出要求
1. 层级总览图（全链路）。
2. 每层研究卡片（六件套）。
3. 问题清单（P0/P1/P2）+ 证据。
4. 补丁清单 + 影响评估。
5. 下一步实施计划（优先级 + 工期 + 风险）。
