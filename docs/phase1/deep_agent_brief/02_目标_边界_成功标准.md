# 02 目标、边界、成功标准（给无代码能力Agent）

## 1. 目标（必须达成）
1. 设计一套 Layer_0~Layer_5 的可视化研究框架。
2. 设计问题发现与修复闭环（发现->修复->验证->记录->固化）。
3. 设计可视化页面方案（页面、组件、图表、交互、数据接口概念）。
4. 给出分阶段实施路线（本周 / 2-6周 / 6周+）。

## 2. 边界（必须遵守）
1. 你没有代码库访问能力，不要求写代码。
2. 你不需要重新验证数据库事实，以提供材料为准。
3. 你的输出应是“方案设计与执行路径”，不是技术实现细节。
4. 输出语言为中文，结构清晰，便于团队直接评审。

## 3. 成功标准（验收）
1. 团队成员不看SQL也能理解每层职责和数据变化。
2. 任一异常可按统一路径在短时间定位到层与规则。
3. 任一补丁都有记录：原因、修改、影响、验证、回滚。
4. 方案可直接转化为后续可执行版本（由有代码能力Agent落地）。

## 4. 输出质量要求
1. 每个建议必须说明“为什么”。
2. 方案必须有优先级（P0/P1/P2或高/中/低）。
3. 必须给出最小可行版本（MVP）和迭代计划。
4. 对信息不足处要明确写“假设前提”和“待确认项”。
