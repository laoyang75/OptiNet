# 01 项目与问题摘要（给无代码能力Agent）

## 1. 项目是什么
这是一个通信数据处理流程项目，主链路是：
- Layer_0 解析标准化
- Layer_1 合规规则
- Layer_2 可信LAC与映射
- Layer_3 可信BS主库
- Layer_4 明细纠偏与补齐（GPS + 信号）
- Layer_5 画像汇总（LAC/BS/CELL）

目标是把原始明细处理成可用、可解释、可追溯的数据资产。

## 2. 当前核心痛点
1. 研究过程主要靠分析师经验，很多思路是隐性的。
2. 逻辑分散在多层与多脚本中，其他人难以快速看懂。
3. 发现问题后缺少统一闭环：定位、修复、验证、记录不标准。
4. 结果可见，但“每一步是怎么变化的”不可视化，不利于判断合理性。

## 3. 你要解决的本质
不是做一份静态报告，而是设计一套“分层可视化研究方法 + 问题修复闭环机制”，让团队能够：
- 看清每一层输入输出变化
- 看清规则和补丁影响
- 快速发现并修正问题
- 沉淀为下一阶段规模化执行基础

## 4. 期望风格
- 结果必须结构化、可执行
- 强调“可解释、可追溯、可复盘”
- 明确区分“事实”和“推断”
